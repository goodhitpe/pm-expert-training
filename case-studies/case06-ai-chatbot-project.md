# Case Study 06: AI 챗봇 개발 프로젝트

> **난이도**: 중급  
> **프로젝트 유형**: AI/ML 소프트웨어 개발  
> **산업**: 핀테크  
> **기간**: 6개월  
> **예산**: ₩500,000,000

---

## 📋 프로젝트 개요

### 배경
**FinTech Korea**는 중소기업 대상 금융 서비스를 제공하는 핀테크 스타트업입니다. 고객 문의가 급증하면서 고객 지원팀의 업무 부담이 커졌고, 24/7 자동화된 고객 지원이 필요해졌습니다.

### 프로젝트 목표
**AI 기반 고객 지원 챗봇** 개발
- 금융 상품 문의 자동 응답
- 계좌 조회, 거래 내역 확인 등 기본 업무 자동화
- 고객 지원팀 업무량 50% 감소
- 고객 만족도 향상 (응답 시간 단축)

### 성공 기준
1. **정확도**: 챗봇 응답 정확도 85% 이상
2. **자동화율**: 전체 문의의 60% 자동 처리
3. **응답 시간**: 평균 응답 시간 5초 이내
4. **고객 만족도**: NPS 점수 70점 이상
5. **비용 절감**: 고객 지원 인건비 월 ₩30,000,000 절감

## 👥 주요 이해관계자

### 프로젝트 스폰서
- **김철수 (CEO)**: 프로젝트 최종 승인, 예산 지원
- **기대**: 비용 절감 및 고객 경험 개선

### 프로젝트 팀
- **이영희 (프로젝트 매니저)**: 전체 프로젝트 관리
- **박지훈 (AI/ML 리드)**: 모델 개발 및 학습
- **최민수 (백엔드 개발자)**: API 및 시스템 통합
- **정수진 (프론트엔드 개발자)**: 챗봇 UI 구현
- **강동욱 (데이터 엔지니어)**: 데이터 파이프라인 구축
- **한소라 (QA 엔지니어)**: 테스트 및 품질 보증

### 외부 이해관계자
- **고객 지원팀 (팀장: 오민지)**: 최종 사용자, 요구사항 제공
- **IT 보안팀 (팀장: 서준호)**: 보안 요구사항, 개인정보 보호
- **컴플라이언스팀 (팀장: 안지민)**: 금융 규제 준수
- **고객들**: 최종 사용자

## 📊 프로젝트 일정 (6개월)

### Phase 1: 착수 및 계획 (Month 1)

**Week 1-2: 착수**
- 프로젝트 킥오프 미팅
- 이해관계자 인터뷰 및 요구사항 수집
- 기존 고객 문의 데이터 분석 (6개월치)

**Week 3-4: 계획**
- 프로젝트 헌장 작성 및 승인
- 상세 WBS 작성
- 일정 및 예산 수립
- 리스크 식별 및 대응 계획

**주요 산출물**:
- 프로젝트 헌장
- 요구사항 명세서
- 데이터 분석 보고서 (문의 유형, 빈도, 패턴)

---

### Phase 2: 데이터 준비 및 모델 설계 (Month 2)

**Week 5-6: 데이터 준비**
- 고객 문의 데이터 수집 (과거 2년치)
- 데이터 정제 및 레이블링
  - 문의 유형 분류 (20개 카테고리)
  - 개인정보 마스킹
- 학습/검증/테스트 데이터셋 분리 (70:15:15)

**Week 7-8: 모델 설계**
- NLP 모델 선정 (KoBERT 기반)
- 챗봇 아키텍처 설계
- 대화 흐름 설계 (Dialogue Flow)
- API 명세 작성

**주요 산출물**:
- 정제된 데이터셋 (50,000건)
- 모델 아키텍처 문서
- 대화 흐름도

**도전 과제 #1: 데이터 품질**
- **문제**: 기존 문의 데이터에 오타, 비속어, 불완전한 문장 많음
- **해결**: 데이터 정제 자동화 스크립트 개발, 수동 검토 병행
- **교훈**: 데이터 품질이 모델 성능의 80%를 결정

---

### Phase 3: AI 모델 개발 (Month 3)

**Week 9-10: 모델 학습**
- KoBERT 파인 튜닝
- 의도 분류 모델 (Intent Classification)
- 개체명 인식 모델 (Named Entity Recognition)
- 초기 모델 평가 (정확도 72%)

**Week 11-12: 모델 개선**
- 하이퍼파라미터 튜닝
- 데이터 증강 (Data Augmentation)
- 앙상블 모델 시도
- 개선된 모델 평가 (정확도 83%)

**주요 산출물**:
- 학습된 AI 모델 (v1.0)
- 모델 평가 보고서
- 성능 벤치마크 결과

**도전 과제 #2: 목표 정확도 미달**
- **문제**: Week 10에 정확도 72%, 목표 85%에 미달
- **대응**: 
  - 크래싱 적용 (데이터 사이언티스트 1명 추가 투입)
  - 데이터 증강으로 학습 데이터 30% 증가
  - 앙상블 모델로 정확도 11% 향상
- **비용 증가**: ₩20,000,000 (추가 인력 + 클라우드 비용)

---

### Phase 4: 시스템 통합 (Month 4)

**Week 13-14: 백엔드 개발**
- FastAPI 기반 REST API 개발
- 모델 서빙 인프라 (AWS SageMaker)
- 데이터베이스 연동 (고객 정보, 거래 내역)
- 보안 인증 (OAuth 2.0)

**Week 15-16: 프론트엔드 개발**
- 웹 챗봇 UI (React)
- 모바일 앱 통합 (React Native)
- 관리자 대시보드 (성능 모니터링)

**주요 산출물**:
- 챗봇 API (v1.0)
- 웹/모바일 UI
- 관리자 대시보드

**도전 과제 #3: 보안 요구사항**
- **문제**: IT 보안팀이 개인정보 보호 우려 제기
- **대응**:
  - 데이터 암호화 (전송 중: TLS, 저장: AES-256)
  - 대화 로그 30일 후 자동 삭제
  - 민감 정보 마스킹 (계좌번호 뒤 4자리만 표시)
- **일정 영향**: 2주 지연 → 패스트 트래킹으로 1주 회복

---

### Phase 5: 테스트 및 최적화 (Month 5)

**Week 17-18: 테스트**
- 단위 테스트 (Unit Testing)
- 통합 테스트 (Integration Testing)
- 성능 테스트 (Load Testing)
  - 동시 사용자 1,000명 처리 가능 확인
- UAT (사용자 수용 테스트)
  - 고객 지원팀 10명 파일럿 테스트

**Week 19-20: 최적화**
- 응답 시간 최적화 (10초 → 3초)
- 모델 경량화 (모델 크기 50% 감소)
- 캐싱 전략 (자주 묻는 질문)
- 예외 처리 개선

**주요 산출물**:
- 테스트 케이스 및 결과 보고서
- 성능 최적화 보고서
- UAT 피드백 보고서

**도전 과제 #4: 성능 문제**
- **문제**: 동시 사용자 500명에서 응답 지연 (15초)
- **해결**:
  - 모델 경량화 (Distillation)
  - Redis 캐싱 도입
  - AWS Auto Scaling 설정
- **결과**: 응답 시간 3초로 단축, 1,000명 동시 처리 가능

---

### Phase 6: 배포 및 종료 (Month 6)

**Week 21-22: 파일럿 배포**

- 고객 지원팀 교육 (챗봇 관리 방법)

**Week 24: 프로젝트 종료**
- 최종 보고서 작성
- 교훈 정리 (Lessons Learned)
- 프로젝트 문서 아카이빙
- 팀 해산 및 축하

**주요 산출물**:
- 배포 완료 보고서

## 🎓 주요 교훈 (Lessons Learned)

### 잘된 점 (Successes)

1. **조기 이해관계자 참여**
   - 고객 지원팀, IT 보안팀 초기부터 참여
   - 요구사항 명확화, 저항 최소화

2. **애자일 방법론 적용**
   - 2주 Sprint로 빠른 피드백
   - 유연한 요구사항 변경 수용

3. **데이터 품질 집중**
   - 초기 데이터 정제에 시간 투자
   - 모델 성능에 직접적 기여

4. **파일럿 테스트**
   - 베타 사용자 100명으로 리스크 완화
   - 긴급 버그 조기 발견 및 수정

### 개선할 점 (Improvements)

1. **초기 예산 산정 부족**
   - AI/ML 프로젝트의 불확실성 과소평가
   - 예비 비용 10% → 20% 권장

2. **성능 테스트 늦음**
   - Month 5에서야 성능 문제 발견
   - Month 3부터 지속적 성능 테스트 필요

3. **보안 요구사항 늦은 반영**
   - Month 4에서 보안 우려 제기
   - 착수 단계부터 보안팀 참여 필요

4. **모델 설명 가능성 부족**
   - 블랙박스 모델로 신뢰도 문제
   - Explainable AI (XAI) 고려 필요

### 다음 프로젝트에 적용할 사항

1. **더 많은 예비 비용** (20%)
2. **조기 성능 테스트** (Month 3부터)
3. **보안/컴플라이언스 조기 참여** (착수 단계)
4. **모델 설명 가능성 고려** (설계 단계)
5. **운영 인수인계 계획** (프로젝트 초기부터)

## 💡 PM 실무 인사이트

### AI/ML 프로젝트 특성

**전통적 소프트웨어 개발과의 차이**:

| 구분 | 전통적 개발 | AI/ML 프로젝트 |
|-----|-----------|--------------|
| **요구사항** | 명확 | 불명확 (성능 목표) |
| **일정 예측** | 비교적 정확 | 불확실성 높음 |
| **품질 측정** | 기능 구현 여부 | 모델 정확도 (확률적) |
| **변경 관리** | 코드 수정 | 데이터 + 모델 재학습 |
| **테스트** | 결정론적 | 확률적 |

### AI/ML 프로젝트 PM의 역할

1. **불확실성 관리**
   - 모델 성능은 데이터에 의존 → 보장 불가
   - 충분한 버퍼 및 예비 계획

2. **이해관계자 기대 관리**
   - "AI는 마법이 아님" 교육
   - 현실적 목표 설정 (정확도 100%는 불가능)

3. **데이터 품질 강조**
   - "Garbage In, Garbage Out"
   - 데이터 수집/정제에 충분한 시간과 예산

4. **윤리 및 규제 고려**
   - AI 편향성 (Bias)
   - 개인정보 보호
   - 설명 가능성 (Explainability)

5. **지속적 개선 계획**
   - 배포 후에도 모델 재학습 필요
   - 운영팀 인수인계 중요

## 📝 실습 문제

### 문제 1: 리스크 관리 (30점)

**시나리오**: Month 3에 새로운 리스크 발견
- "경쟁사가 유사한 챗봇을 먼저 출시할 예정"
- 확률: 60%, 영향: 높음

**질문**:
1. 이 리스크의 대응 전략은? (회피, 전가, 완화, 수용)
2. 구체적 대응 계획 3가지를 제시하시오.
3. 일정을 2주 앞당기기 위한 크래싱/패스트 트래킹 계획을 수립하시오.

---

### 문제 2: 예산 통제 (30점)

**시나리오**: Month 4 종료 시점
- PV = ₩333,333,333 (4/6)
- EV = ₩300,000,000
- AC = ₩350,000,000

**질문**:
1. CPI, SPI 계산
2. EAC (Estimate at Completion) 계산
3. 프로젝트 상태 평가 (일정, 비용)
4. 대응 방안 3가지 제시

---

### 문제 3: 이해관계자 관리 (20점)

**시나리오**: 고객 지원팀장이 "챗봇 도입으로 일자리 위협" 우려 제기

**질문**:
1. 이해관계자 참여 수준 평가 (모름/저항/중립/지원/주도)
2. 현재 상태에서 목표 상태로 전환 전략 수립
3. 커뮤니케이션 계획 작성

---

### 문제 4: 품질 관리 (20점)

**시나리오**: UAT에서 사용자들이 "챗봇 응답이 부정확하다" 피드백

**질문**:
1. 품질 문제의 근본 원인 분석 (Fishbone Diagram)
2. 시정 조치 계획
3. 예방 조치 (향후 프로젝트)
