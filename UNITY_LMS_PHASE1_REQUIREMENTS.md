# 🎓 Unity 기반 LMS 및 프로젝트 시뮬레이터 - 1차 요구사항 정의서

> **버전**: 1.0  
> **작성일**: 2025년 2월 3일  
> **목적**: Unity 기반 학습 관리 시스템(LMS)과 프로젝트 시뮬레이터 개발을 위한 Phase 1 요구사항 정의

---

## 📌 1. 프로젝트 개요

### 1.1 배경
- 현재 PM 전문가 양성 프로그램은 16주 커리큘럼으로 구성된 문서 기반 교육 자료로 제공됨
- 학습자의 몰입도와 실무 적용 능력 향상을 위해 인터랙티브한 학습 환경 필요
- Unity 기반 3D/2D 환경에서 실습과 시뮬레이션을 통한 학습 효과 극대화

### 1.2 목표
1. **LMS 기능**: 기존 커리큘럼 콘텐츠를 Unity 환경에서 제공하는 학습 관리 시스템
2. **프로젝트 시뮬레이터**: 실제 프로젝트 관리 상황을 시뮬레이션하여 실무 감각 익히기
3. **통합 학습 경험**: LMS에서 학습 후 시뮬레이터에서 즉시 실습 가능한 연계 구조

### 1.3 대상 사용자
- IT 비전공자
- PM으로 전환을 희망하는 실무자
- 신입/주니어 PM
- 프로젝트 관리 역량을 강화하려는 학습자

## 📋 2. 기능 요구사항

### 2.1 LMS (Learning Management System) 기능

#### 2.1.1 학습 콘텐츠 관리
**FR-LMS-001: 주차별 커리큘럼 네비게이션**
- 16주 커리큘럼을 시각적으로 표현하는 학습 맵 제공
- Phase별 (기초/핵심/실전/IT기술) 구분 표시
- 현재 진행 중인 주차 하이라이트
- 잠금/해제 메커니즘 (순차적 학습 진행)

**FR-LMS-002: 강의 콘텐츠 뷰어**
- 마크다운 기반 강의 자료 렌더링
- 이미지, 다이어그램, 표 등 멀티미디어 지원
- 텍스트 하이라이트 및 북마크 기능
- 학습 노트 작성 및 저장 기능

**FR-LMS-003: 사전 학습 자료(Prerequisite) 제공**
- 각 주차별 사전 학습 자료 접근
- 선수 지식 확인 체크리스트
- 관련 용어집 링크 연동

**FR-LMS-004: 실습 가이드 통합**
- Jira, GitHub, SQL, AWS 등 도구별 실습 가이드 제공
- 단계별 인터랙티브 튜토리얼
- 실습 완료 체크 및 진행률 추적

#### 2.1.2 평가 및 퀴즈 시스템
**FR-LMS-005: 주차별 퀴즈**
- 각 주차 완료 후 평가 퀴즈 제공
- 객관식, 주관식, 시나리오 기반 문제
- 즉시 피드백 및 해설 제공
- 재시험 기능 (최대 3회)

**FR-LMS-006: 평가 루브릭 기반 과제 제출**
- 주차별 과제 제출 인터페이스
- 평가 루브릭 확인 기능
- 자가 평가 체크리스트
- 제출 기록 및 피드백 조회

**FR-LMS-007: 학습 진행률 추적**
- 전체 커리큘럼 진행률 대시보드
- 주차별 완료 상태 (미시작/진행중/완료)
- 퀴즈 점수 및 평균 표시
- 학습 시간 통계

#### 2.1.3 케이스 스터디
**FR-LMS-008: 케이스 스터디 라이브러리**
- 6개 산업별 케이스 스터디 제공
- 케이스 상세 내용 뷰어
- 케이스별 질문 및 토론 포인트
- 모범 답안 및 해설

### 2.2 프로젝트 시뮬레이터 기능

#### 2.2.1 3D/2D 가상 오피스 환경
**FR-SIM-001: 가상 오피스 공간**
- 3D/2D 오피스 환경 (프로젝트 룸, 회의실, 개인 작업 공간)
- 캐릭터 이동 및 NPC와 상호작용
- 환경 오브젝트: 화이트보드, 모니터, 문서함 등

**FR-SIM-002: 프로젝트 팀 시뮬레이션**
- 가상 팀원 NPC (개발자, 디자이너, QA, 고객 등)
- 팀원별 특성 및 성격 설정
- 팀원과의 대화 인터페이스
- 팀원 만족도 및 생산성 지표

#### 2.2.2 프로젝트 관리 시뮬레이션
**FR-SIM-003: 프로젝트 시나리오 실행**
- 6개 케이스 기반 시뮬레이션 시나리오 제공
  - 모바일 앱 개발
  - 마케팅 캠페인
  - 사무실 이전
  - ERP 구축
  - 제품 런칭
  - AI 챗봇 개발
- 시나리오별 초기 조건 설정 (예산, 일정, 팀 구성)
- 단계별 이벤트 발생 (예: 범위 변경, 리스크 발생, 리소스 이슈)

**FR-SIM-004: 의사결정 및 액션 시스템**
- 프로젝트 관리 의사결정 선택지 제공
- 의사결정에 따른 결과 시뮬레이션
- 리소스, 시간, 예산 관리 인터페이스
- WBS, 간트 차트, 리스크 레지스터 작성 도구

**FR-SIM-005: 회의 및 커뮤니케이션 시뮬레이션**
- 가상 회의 진행 (킥오프, 스탠드업, 스프린트 리뷰 등)
- 이해관계자와의 대화 시뮬레이션
- 보고서 작성 및 발표 연습
- 갈등 상황 대처 시뮬레이션

**FR-SIM-006: 실시간 프로젝트 지표 모니터링**
- 프로젝트 상태 대시보드
- 일정 준수율 (SPI, Schedule Performance Index)
- 비용 준수율 (CPI, Cost Performance Index)
- 팀 사기 및 생산성 지표
- 리스크 발생 현황

#### 2.2.3 도구 통합 시뮬레이션
**FR-SIM-007: PM 도구 시뮬레이션**
- Jira/Trello 스타일 태스크 보드
- 간트 차트 생성 및 수정
- WBS 작성 도구
- 리스크 매트릭스

**FR-SIM-008: 애자일/스크럼 시뮬레이션**
- 스프린트 계획 및 실행
- 데일리 스탠드업 시뮬레이션
- 스프린트 리뷰 및 회고
- 백로그 관리

#### 2.2.4 평가 및 피드백
**FR-SIM-009: 성과 평가 시스템**
- 프로젝트 완료 후 종합 평가
- 의사결정 품질 분석
- 팀 관리 능력 평가
- 재무 성과 분석

**FR-SIM-010: 멘토 AI 피드백**
- 실시간 힌트 및 가이드 제공
- 실수 시 즉각적인 피드백
- 우수 사례 및 개선 제안
- 학습 포인트 요약

### 2.3 통합 기능

#### 2.3.1 사용자 관리
**FR-INT-001: 사용자 프로필**
- 사용자 등록 및 로그인
- 학습 이력 저장
- 시뮬레이터 기록 저장
- 성취 배지 및 레벨 시스템

**FR-INT-002: 진행률 동기화**
- LMS 학습 진행률과 시뮬레이터 잠금 해제 연동
- 주차별 학습 완료 시 해당 시뮬레이션 활성화
- 크로스 플랫폼 진행률 동기화

#### 2.3.2 데이터 연동
**FR-INT-003: 커리큘럼 데이터 임포트**
- 기존 마크다운 파일 자동 파싱
- 템플릿 파일 로딩 및 표시
- 케이스 스터디 데이터 변환
- 평가 문항 및 루브릭 데이터 로딩

**FR-INT-004: 콘텐츠 업데이트 시스템**
- 외부 콘텐츠 파일 핫 리로드
- 버전 관리 및 업데이트 알림
- 오프라인 모드 지원 (로컬 캐싱)

## 🛠️ 4. Unity 기술 스택

### 4.1 Unity 버전 및 패키지
**Unity 버전**: Unity 2022 LTS (Long Term Support)

**핵심 패키지**:
- **TextMesh Pro**: 텍스트 렌더링 및 다국어 지원
- **Unity UI**: UI 시스템
- **Cinemachine**: 카메라 관리
- **Input System**: 입력 처리
- **Addressables**: 에셋 관리 및 동적 로딩
- **JSON .NET**: 데이터 직렬화

**추가 패키지 검토**:
- **Markdown Renderer**: 마크다운 렌더링 (에셋 스토어 또는 커스텀)
- **DOTween**: 애니메이션 및 트윈
- **UniTask**: 비동기 처리
- **VContainer**: 의존성 주입

### 4.2 아키텍처 패턴
**디자인 패턴**:
- **MVC (Model-View-Controller)**: UI 및 로직 분리
- **Observer Pattern**: 이벤트 기반 통신
- **State Machine**: 시뮬레이터 상태 관리
- **Factory Pattern**: 오브젝트 생성 관리

**코드 구조**:
```
Assets/
├── Scripts/
│   ├── Core/              # 핵심 시스템 (데이터 관리, 네비게이션)
│   ├── LMS/               # LMS 관련 스크립트
│   │   ├── ContentViewer/ # 콘텐츠 뷰어
│   │   ├── Quiz/          # 퀴즈 시스템
│   │   ├── Progress/      # 진행률 관리
│   │   └── Assessment/    # 평가 시스템
│   ├── Simulator/         # 시뮬레이터 관련 스크립트
│   │   ├── Environment/   # 3D 환경 관리
│   │   ├── NPC/           # NPC AI 및 대화
│   │   ├── Project/       # 프로젝트 관리 로직
│   │   ├── Events/        # 이벤트 시스템
│   │   └── Tools/         # PM 도구 시뮬레이션
│   ├── UI/                # UI 컴포넌트
│   ├── Data/              # 데이터 모델
│   └── Utils/             # 유틸리티
├── Resources/
│   └── Curriculum/        # 커리큘럼 데이터 (JSON)
├── Prefabs/
├── Scenes/
│   ├── MainMenu/
│   ├── LMS/
│   └── Simulator/
└── Materials/
```

## 📅 5. Phase 1 개발 범위 및 우선순위

### 5.1 Phase 1 (MVP) - 2개월
**목표**: 핵심 LMS 기능 + 1개 시뮬레이션 시나리오

**포함 항목**:
- ✅ 주차별 커리큘럼 네비게이션
- ✅ 강의 콘텐츠 뷰어 (마크다운 렌더링)
- ✅ 학습 진행률 추적 기본 기능
- ✅ 1개 케이스 스터디 시뮬레이션 (모바일 앱 개발)
- ✅ 기본 3D 오피스 환경
- ✅ 간단한 NPC 상호작용
- ✅ 의사결정 시스템 프로토타입
- ✅ 사용자 프로필 및 저장 시스템

**제외 항목** (Phase 2로 이연):
- 퀴즈 시스템 (일부만 구현)
- 평가 루브릭 통합
- 6개 전체 시뮬레이션
- 멘토 AI 피드백
- 멀티플레이어 기능

### 5.2 개발 우선순위

**P0 (필수)**:
1. 커리큘럼 데이터 파싱 및 로딩 시스템
2. LMS 메인 화면 및 네비게이션
3. 마크다운 뷰어
4. 진행률 저장/로딩
5. 기본 시뮬레이터 환경

**P1 (중요)**:

## 🔄 6. 기존 자료와의 통합 방안

### 6.1 커리큘럼 콘텐츠 통합
**방법**: 
- 기존 마크다운 파일을 Unity Resources 또는 Addressables로 관리
- 빌드 시 JSON으로 변환하여 메타데이터 생성
- 실행 시 동적 로딩

**경로 매핑**:
```
curriculum/week01-pm-introduction/
  → LMS > Week 1 > 콘텐츠
```

### 6.2 템플릿 통합
**방법**:
- 템플릿 파일을 시뮬레이터 내 작성 가능한 폼으로 변환
- 사용자가 작성한 내용을 템플릿 형식으로 저장 가능

**예시**:
- Project Charter 템플릿 → 시뮬레이터 내 프로젝트 헌장 작성 UI

### 6.3 케이스 스터디 통합
**방법**:
- 각 케이스 스터디를 시뮬레이션 시나리오로 변환
- 케이스 스터디의 배경, 상황, 질문을 이벤트 및 의사결정으로 구현

### 6.4 평가 및 퀴즈 통합
**방법**:
- assessment/quizzes/ 데이터를 JSON으로 변환
- assessment/rubrics/ 데이터를 평가 기준으로 활용
