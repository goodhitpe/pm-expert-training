## 1. AWS란?

### 1.1 AWS (Amazon Web Services) 개요
**AWS**는 아마존이 제공하는 클라우드 컴퓨팅 플랫폼입니다.

### 1.2 클라우드 컴퓨팅이란?
- **정의**: 인터넷을 통해 IT 리소스를 온디맨드로 제공
- **장점**:
  - 초기 투자 비용 감소
  - 필요할 때 즉시 확장
  - 사용한 만큼만 과금
  - 글로벌 인프라 활용

### 1.3 AWS의 장점
- ✅ **시장 점유율 1위**: 가장 많이 사용되는 클라우드
- ✅ **다양한 서비스**: 200개 이상의 서비스
- ✅ **글로벌 인프라**: 전 세계 33개 리전
- ✅ **보안 및 컴플라이언스**: 엔터프라이즈급 보안
- ✅ **커뮤니티 및 자료**: 풍부한 학습 자료

### 1.4 AWS vs Azure vs GCP

| 항목 | AWS | Azure | GCP |
|-----|-----|-------|-----|
| 시장 점유율 | 1위 (32%) | 2위 (23%) | 3위 (10%) |
| 서비스 수 | 200+ | 100+ | 100+ |
| 강점 | 범용성, 생태계 | MS 제품 통합 | 빅데이터, AI |
| 주요 고객 | Netflix, Airbnb | Adobe, BMW | Spotify, Twitter |
| 학습 난이도 | 중간 | 중간 | 쉬움 |

### 1.5 PM이 클라우드를 알아야 하는 이유
- 📊 **인프라 비용 산정**: 프로젝트 예산 수립
- 🗣️ **기술팀 커뮤니케이션**: 아키텍처 이해
- ⚡ **빠른 의사결정**: 클라우드 vs 온프레미스 선택
- 🔒 **보안 및 컴플라이언스**: 데이터 보호 요구사항 이해

## 2. AWS Free Tier 개요

### 2.1 Free Tier 유형

#### 🎉 12개월 무료
**신규 가입 후 12개월간 무료**
- EC2: 750시간/월 (t2.micro 또는 t3.micro)
- S3: 5GB 스토리지
- RDS: 750시간/월 (db.t2.micro, db.t3.micro)
- CloudFront: 50GB 아웃바운드 데이터 전송

#### ♾️ 항상 무료
**기간 제한 없이 무료**
- Lambda: 100만 요청/월
- DynamoDB: 25GB 스토리지
- SNS: 100만 게시/월
- CloudWatch: 10개 사용자 지정 지표

#### 🧪 평가판
**특정 기간 동안 무료 체험**
- SageMaker: 250시간/2개월
- Redshift: 750시간/2개월

### 2.2 주의사항

#### ⚠️ 과금 방지 필수 체크사항
1. **12개월 제한**: 가입일 기준 12개월 후 자동 과금
2. **사용량 제한**: 무료 한도 초과 시 과금
3. **리전 제한**: 일부 서비스는 특정 리전만 무료
4. **자동 갱신**: 인스턴스 자동 시작 방지
5. **데이터 전송**: 아웃바운드 데이터 전송 비용

#### 💡 비용 절약 팁
- ✅ 사용하지 않는 리소스 즉시 삭제
- ✅ CloudWatch 비용 알람 설정
- ✅ 태그로 리소스 관리
- ✅ 정기적으로 Billing Dashboard 확인

## 3. AWS 계정 생성

### 3.1 필요한 것
- 📧 이메일 주소
- 💳 신용카드 또는 체크카드 (검증용, 과금 없음)
- 📱 휴대폰 (SMS 인증)

### 3.2 회원가입 단계

#### Step 1: AWS 웹사이트 접속
1. [https://aws.amazon.com/ko/free/](https://aws.amazon.com/ko/free/) 접속
2. **"무료 계정 만들기"** 버튼 클릭

#### Step 2: 계정 정보 입력
1. **이메일 주소** 입력
2. **AWS 계정 이름** 입력 (예: "김철수-개인용")
3. **비밀번호** 설정 (최소 8자)
4. **비밀번호 확인**
5. **"계속"** 버튼 클릭

#### Step 3: 연락처 정보
1. **계정 유형** 선택:
   - **개인**: 학습 및 개인 프로젝트 (권장)
   - **비즈니스**: 회사 업무용
2. **이름** 입력
3. **전화번호** 입력 (국가 코드 포함: +82)
4. **주소** 입력
5. **AWS 고객 계약 동의** 체크
6. **"계속"** 버튼 클릭

#### Step 4: 결제 정보
1. **신용카드 정보** 입력
   - 카드 번호
   - 유효 기간
   - 카드 소유자 이름
2. **청구지 주소** 입력
3. **"인증 및 계속"** 클릭

**⚠️ 주의**: $1 임시 승인 후 취소됨 (실제 과금 아님)

#### Step 5: 본인 확인
1. **전화번호** 확인
2. **보안 문자** 입력
3. **"SMS 전송"** 또는 **"전화 받기"** 선택
4. 받은 **인증 코드** 입력
5. **"계속"** 버튼 클릭

#### Step 6: 지원 플랜 선택
- **기본 지원 - 무료** 선택 (권장)
- **"무료 가입 완료"** 클릭

#### Step 7: 계정 생성 완료
- 이메일로 확인 메일 수신
- 계정 활성화 (최대 24시간 소요 가능)

### 3.3 AWS Management Console 로그인

#### 로그인 방법
1. [https://console.aws.amazon.com](https://console.aws.amazon.com) 접속
2. **"Root user"** 선택
3. 이메일 주소 입력
4. 비밀번호 입력
5. **"로그인"** 클릭

### 3.4 MFA (다중 인증) 설정 (권장)

#### MFA 설정 이유
- 계정 보안 강화
- 해킹 방지
- AWS 모범 사례

#### MFA 설정 방법
1. Console 우측 상단 **계정 이름** 클릭
2. **"보안 자격 증명"** 클릭
3. **"MFA"** 섹션에서 **"MFA 디바이스 할당"** 클릭
4. **"가상 MFA 디바이스"** 선택
5. 스마트폰에 **Google Authenticator** 또는 **Authy** 앱 설치
6. QR 코드 스캔
7. 연속된 두 개의 MFA 코드 입력
8. **"MFA 할당"** 클릭

## 4. AWS 주요 서비스 소개

### 4.1 컴퓨팅 서비스

#### EC2 (Elastic Compute Cloud)
- **설명**: 가상 서버
- **용도**: 웹 서버, 애플리케이션 서버
- **Free Tier**: 750시간/월 (t2.micro)

#### Lambda
- **설명**: 서버리스 컴퓨팅
- **용도**: 이벤트 기반 함수 실행
- **Free Tier**: 100만 요청/월 (항상 무료)

### 4.2 스토리지 서비스

#### S3 (Simple Storage Service)
- **설명**: 객체 스토리지 (파일 저장)
- **용도**: 정적 파일, 백업, 로그
- **Free Tier**: 5GB 스토리지, 20,000 GET 요청

#### EBS (Elastic Block Store)
- **설명**: EC2용 블록 스토리지
- **용도**: EC2 디스크
- **Free Tier**: 30GB/월

### 4.3 데이터베이스 서비스

#### RDS (Relational Database Service)
- **설명**: 관리형 관계형 DB
- **지원 DB**: MySQL, PostgreSQL, MariaDB, Oracle, SQL Server
- **Free Tier**: 750시간/월 (db.t2.micro)

#### DynamoDB
- **설명**: NoSQL 데이터베이스
- **용도**: 대규모 키-값 데이터
- **Free Tier**: 25GB 스토리지 (항상 무료)

### 4.4 네트워킹 서비스

#### VPC (Virtual Private Cloud)
- **설명**: 가상 네트워크
- **용도**: 네트워크 격리 및 보안
- **Free Tier**: 무료

#### CloudFront
- **설명**: CDN (Content Delivery Network)
- **용도**: 콘텐츠 전송 가속
- **Free Tier**: 50GB 아웃바운드 데이터 전송

### 4.5 보안 및 관리

#### IAM (Identity and Access Management)
- **설명**: 사용자 및 권한 관리
- **용도**: 보안 정책 설정
- **Free Tier**: 무료

#### CloudWatch
- **설명**: 모니터링 및 로그
- **용도**: 리소스 모니터링, 알람
- **Free Tier**: 10개 지표 (항상 무료)

## 5. 실습 1: EC2 (가상 서버)

### 5.1 EC2란?
- 클라우드에서 실행되는 가상 서버
- 원하는 OS 및 소프트웨어 설치 가능
- 필요에 따라 크기 조정 가능

### 5.2 EC2 인스턴스 생성

#### Step 1: EC2 대시보드 접속
1. AWS Console에서 **"EC2"** 검색
2. **"EC2 Dashboard"** 클릭

#### Step 2: 인스턴스 시작
1. **"인스턴스 시작"** 버튼 클릭

#### Step 3: 이름 및 태그
- **Name**: `my-first-server` 입력

#### Step 4: AMI (Amazon Machine Image) 선택
- **Amazon Linux 2023 AMI** 선택 (권장)
- **프리 티어 사용 가능** 표시 확인

#### Step 5: 인스턴스 유형 선택
- **t2.micro** 선택 (프리 티어)
  - vCPU: 1개
  - 메모리: 1GB
  - 네트워크: 낮음-보통

#### Step 6: 키 페어 생성
1. **"새 키 페어 생성"** 클릭
2. 키 페어 이름: `my-keypair` 입력
3. 키 페어 유형: **RSA**
4. 프라이빗 키 파일 형식:
   - Windows: **.ppk** (PuTTY용)
   - Mac/Linux: **.pem** (OpenSSH용)
5. **"키 페어 생성"** 클릭
6. **자동으로 다운로드됨** (안전한 곳에 보관!)

#### Step 7: 네트워크 설정
- **"기본 VPC"** 선택 (자동 생성됨)
- **"퍼블릭 IP 자동 할당"**: 활성화
- **보안 그룹** (방화벽 규칙):
  - ✅ SSH (포트 22): 내 IP만 허용 (권장)
  - ⬜ HTTP (포트 80): 필요 시 추가

#### Step 8: 스토리지 구성
- **크기**: 8GB (기본값, 프리 티어 30GB까지 무료)
- **볼륨 유형**: gp3 (기본값)

#### Step 9: 고급 세부 정보 (선택)
- **종료 방지**: 활성화 (실수로 삭제 방지)

#### Step 10: 인스턴스 시작
- **"인스턴스 시작"** 버튼 클릭
- **"인스턴스 보기"** 클릭

### 5.3 EC2 인스턴스 연결

#### 연결 방법 1: EC2 Instance Connect (간단)
1. 인스턴스 선택
2. **"연결"** 버튼 클릭
3. **"EC2 Instance Connect"** 탭 선택
4. **"연결"** 버튼 클릭
5. 브라우저에서 터미널 열림

#### 연결 방법 2: SSH (Mac/Linux)
```bash
# 키 파일 권한 설정
chmod 400 my-keypair.pem

# SSH 연결
ssh -i my-keypair.pem ec2-user@<퍼블릭-IP>
```

#### 연결 방법 3: PuTTY (Windows)
1. PuTTY 실행
2. Host Name: `ec2-user@<퍼블릭-IP>`
3. Connection → SSH → Auth → Credentials
4. Private key file: `.ppk` 파일 선택
5. **Open** 클릭

### 5.4 간단한 웹 서버 구축

```bash
# Apache 웹 서버 설치
sudo yum update -y
sudo yum install -y httpd

# 웹 서버 시작
sudo systemctl start httpd
sudo systemctl enable httpd

# 간단한 웹 페이지 생성
echo "<h1>Hello from AWS EC2!</h1>" | sudo tee /var/www/html/index.html

# 방화벽 규칙 추가 (보안 그룹에서 HTTP 허용 필요)
```

**보안 그룹에 HTTP 규칙 추가**:
1. EC2 → 보안 그룹 → 해당 보안 그룹 선택
2. **"인바운드 규칙 편집"** 클릭
3. **"규칙 추가"** 클릭
4. 유형: **HTTP**, 소스: **0.0.0.0/0** (모든 IP)
5. **"규칙 저장"** 클릭

**브라우저에서 확인**:
```
http://<퍼블릭-IP>
```

### 5.5 인스턴스 중지 및 종료

#### 중지 (Stop) vs 종료 (Terminate)
- **중지**: 인스턴스 일시 정지, 스토리지 유지, 나중에 재시작 가능
- **종료**: 인스턴스 완전 삭제, 데이터 손실

#### 중지 방법
1. 인스턴스 선택
2. **"인스턴스 상태"** → **"인스턴스 중지"** 클릭

#### 종료 방법
1. 인스턴스 선택
2. **"인스턴스 상태"** → **"인스턴스 종료"** 클릭
3. **확인**

**⚠️ 중요**: 사용하지 않을 때는 반드시 중지 또는 종료하여 과금 방지!

## 6. 실습 2: S3 (파일 저장소)

### 6.1 S3란?
- 객체 스토리지 서비스
- 무제한 용량
- 99.999999999% (11개의 9) 내구성
- 정적 웹사이트 호스팅 가능

### 6.2 S3 버킷 생성

#### Step 1: S3 대시보드 접속
1. AWS Console에서 **"S3"** 검색
2. **"S3"** 클릭

#### Step 2: 버킷 생성
1. **"버킷 만들기"** 버튼 클릭

#### Step 3: 버킷 이름 및 리전
- **버킷 이름**: `my-first-bucket-20260201` (전 세계에서 고유해야 함)
- **AWS 리전**: **아시아 태평양(서울) ap-northeast-2** 선택

#### Step 4: 객체 소유권
- **ACL 비활성화됨** 선택 (권장)

#### Step 5: 퍼블릭 액세스 차단 설정
- **"모든 퍼블릭 액세스 차단"** 체크 (기본값, 보안 권장)
- 정적 웹사이트 호스팅 시에만 해제

#### Step 6: 버킷 버전 관리
- **비활성화** (기본값, 필요 시 활성화)

#### Step 7: 버킷 생성 완료
- **"버킷 만들기"** 클릭

### 6.3 파일 업로드

#### Step 1: 버킷 선택
- 생성한 버킷 클릭

#### Step 2: 파일 업로드
1. **"업로드"** 버튼 클릭
2. **"파일 추가"** 또는 드래그 앤 드롭
3. 파일 선택 (예: 이미지, PDF 등)
4. **"업로드"** 버튼 클릭

#### Step 3: 업로드 확인
- 파일 목록에서 확인
- 파일 클릭 → **"객체 URL"** 복사

### 6.4 정적 웹사이트 호스팅 (선택)

#### Step 1: 버킷 퍼블릭 액세스 허용
1. 버킷 선택 → **"권한"** 탭
2. **"퍼블릭 액세스 차단 편집"** 클릭
3. **"모든 퍼블릭 액세스 차단"** 체크 해제
4. **"변경 사항 저장"** 클릭

#### Step 2: 버킷 정책 설정
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-first-bucket-20260201/*"
    }
  ]
}
```
- **"권한"** 탭 → **"버킷 정책"** → **"편집"**
- 위 JSON 붙여넣기 (버킷 이름 수정)
- **"변경 사항 저장"** 클릭

#### Step 3: 정적 웹사이트 호스팅 활성화
1. **"속성"** 탭 스크롤 다운
2. **"정적 웹 사이트 호스팅"** → **"편집"** 클릭
3. **"활성화"** 선택
4. **인덱스 문서**: `index.html` 입력
5. **오류 문서**: `error.html` 입력 (선택)
6. **"변경 사항 저장"** 클릭

#### Step 4: index.html 업로드
```html
<!DOCTYPE html>
<html>
<head>
    <title>My First S3 Website</title>
</head>
<body>
    <h1>Hello from S3!</h1>
    <p>This is my first static website hosted on AWS S3.</p>
</body>
</html>
```
- 위 내용을 `index.html`로 저장
- S3 버킷에 업로드

#### Step 5: 웹사이트 접속
- **"속성"** 탭 → **"정적 웹 사이트 호스팅"** 섹션
- **"버킷 웹사이트 엔드포인트"** 복사
- 브라우저에서 접속

## 7. 실습 3: RDS (데이터베이스)

### 7.1 RDS란?
- 관리형 관계형 데이터베이스
- 백업, 패치, 모니터링 자동화
- 고가용성 및 읽기 복제본 지원

### 7.2 RDS 인스턴스 생성

#### Step 1: RDS 대시보드 접속
1. AWS Console에서 **"RDS"** 검색
2. **"RDS"** 클릭

#### Step 2: 데이터베이스 생성
1. **"데이터베이스 생성"** 버튼 클릭

#### Step 3: 생성 방법 선택
- **"표준 생성"** 선택

#### Step 4: 엔진 옵션
- **MySQL** 선택 (권장)
- 버전: 최신 버전 (예: MySQL 8.0.35)

#### Step 5: 템플릿
- **"프리 티어"** 선택 ⭐

#### Step 6: 설정
- **DB 인스턴스 식별자**: `my-first-database`
- **마스터 사용자 이름**: `admin`
- **마스터 암호**: 강력한 암호 설정 (기억하세요!)

#### Step 7: 인스턴스 구성
- **db.t3.micro** 또는 **db.t2.micro** (프리 티어)

#### Step 8: 스토리지
- **범용 SSD (gp2)**
- **할당된 스토리지**: 20GB (프리 티어 최대)
- **스토리지 자동 조정**: 비활성화

#### Step 9: 연결
- **VPC**: 기본 VPC 선택
- **퍼블릭 액세스**: **예** 선택 (외부 접속 허용)
- **VPC 보안 그룹**: **새로 생성**
- **가용 영역**: 기본 설정

#### Step 10: 추가 구성
- **초기 데이터베이스 이름**: `mydb` (선택)
- **백업**: 자동 백업 활성화 (기본 7일)
- **암호화**: 비활성화 (프리 티어 제한)
- **모니터링**: 기본 모니터링

#### Step 11: 데이터베이스 생성
- **"데이터베이스 생성"** 클릭
- 생성 완료까지 5-10분 소요

### 7.3 RDS 연결

#### Step 1: 엔드포인트 확인
1. RDS 대시보드에서 생성한 DB 클릭
2. **"연결 & 보안"** 탭
3. **"엔드포인트"** 복사 (예: `my-first-database.xxxxx.ap-northeast-2.rds.amazonaws.com`)

#### Step 2: 보안 그룹 수정
1. **"VPC 보안 그룹"** 클릭
2. **"인바운드 규칙 편집"** 클릭
3. **"규칙 추가"**:
   - 유형: **MySQL/Aurora**
   - 포트: **3306**
   - 소스: **내 IP** (보안 강화) 또는 **0.0.0.0/0** (모든 IP)
4. **"규칙 저장"** 클릭

#### Step 3: MySQL 클라이언트로 연결

**MySQL Workbench (GUI, 권장)**
1. [MySQL Workbench](https://www.mysql.com/products/workbench/) 다운로드 및 설치
2. **"+"** 버튼 클릭 (새 연결)
3. 연결 정보 입력:
   - Connection Name: `AWS RDS`
   - Hostname: RDS 엔드포인트
   - Port: `3306`
   - Username: `admin`
   - Password: 저장한 암호
4. **"Test Connection"** → **"OK"** 클릭

**명령줄 (CLI)**
```bash
mysql -h my-first-database.xxxxx.ap-northeast-2.rds.amazonaws.com -P 3306 -u admin -p
```

#### Step 4: 데이터베이스 사용
```sql
-- 데이터베이스 목록 확인
SHOW DATABASES;

-- 데이터베이스 선택
USE mydb;

-- 테이블 생성
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

-- 데이터 삽입
INSERT INTO users (name, email) VALUES ('김철수', 'kim@example.com');

-- 데이터 조회
SELECT * FROM users;
```

### 7.4 RDS 삭제 (과금 방지)

#### Step 1: 데이터베이스 선택
1. RDS 대시보드에서 DB 선택
2. **"작업"** → **"삭제"** 클릭

#### Step 2: 삭제 확인
- **"최종 스냅샷 생성"**: 필요 시 체크 (보통은 해제)
- **"자동 백업 보관"**: 해제
- **"DB 인스턴스 식별자 입력"**: `delete me` 입력
- **"삭제"** 클릭

## 10. 자주 묻는 질문 (FAQ)

### Q1: 프리 티어 기간이 지나면 자동으로 과금되나요?
**A**: 네, 12개월 후에는 정상 요금이 부과됩니다. 사용하지 않는 리소스는 미리 삭제하세요.

### Q2: 실수로 과금되면 어떻게 하나요?
**A**: AWS 지원팀에 연락하면 첫 과금은 면제해주는 경우가 많습니다.

### Q3: EC2를 중지하면 요금이 부과되나요?
**A**: 컴퓨팅 비용은 청구되지 않지만, EBS 스토리지 비용은 계속 부과됩니다.

### Q4: S3는 무제한 무료인가요?
**A**: 아니요, 12개월간 5GB까지만 무료입니다.

### Q5: RDS는 왜 생성에 시간이 오래 걸리나요?
**A**: 데이터베이스 인스턴스를 프로비저닝하고 초기화하는 데 시간이 필요합니다.

### Q6: 리전은 어디를 선택해야 하나요?
**A**: 한국에서는 **아시아 태평양(서울)** 리전을 선택하면 지연 시간이 최소화됩니다.
