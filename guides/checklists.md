## 1. 프로젝트 착수 체크리스트

### 1.1 프로젝트 승인 전

#### 비즈니스 케이스 검토
- [ ] 프로젝트의 비즈니스 필요성 명확히 정의됨
- [ ] 예상되는 비즈니스 가치와 혜택 정량화됨
- [ ] 투자 수익률(ROI) 또는 비용-편익 분석 완료
- [ ] 대안 분석 및 권고안 제시됨
- [ ] 리스크 및 제약 조건 식별됨
- [ ] 재무 실행 가능성 검증됨

#### 실행 가능성 평가
- [ ] 기술적 실행 가능성 평가 완료
- [ ] 자원 가용성 확인 (인력, 예산, 장비)
- [ ] 조직 역량 평가 완료
- [ ] 법적/규제 요구사항 검토 완료
- [ ] 시장 및 경쟁 환경 분석 완료

#### 이해관계자 초기 식별
- [ ] 주요 이해관계자 목록 작성
- [ ] 프로젝트 스폰서 식별 및 약속 확보
- [ ] 최종 의사결정권자 명확화
- [ ] 주요 이해관계자의 기대사항 수집
- [ ] 잠재적 반대자 식별 및 대응 계획

### 1.2 프로젝트 헌장 개발

#### 프로젝트 정의
- [ ] 프로젝트 목적 명확히 기술
- [ ] 프로젝트 목표 정의 (SMART 기준)
- [ ] 프로젝트 범위 개요 작성
- [ ] 주요 인도물 목록 작성
- [ ] 성공 기준 정의
- [ ] 측정 가능한 성과 지표(KPI) 수립

#### 제약 조건 및 가정
- [ ] 시간 제약 식별 (마감일, 마일스톤)
- [ ] 예산 제약 명시
- [ ] 자원 제약 파악
- [ ] 기술적 제약 문서화
- [ ] 주요 가정 사항 기록
- [ ] 의존성 식별

#### 고위험 요소
- [ ] 주요 리스크 초기 식별
- [ ] 리스크 대응 전략 개요
- [ ] 리스크 소유자 배정

#### 프로젝트 관리자 지정
- [ ] PM 선정 및 배정
- [ ] PM 권한 수준 명시
- [ ] PM 책임 범위 정의
- [ ] 예산 승인 권한 명시

#### 승인 및 서명
- [ ] 프로젝트 스폰서 검토 및 승인
- [ ] 주요 이해관계자 동의 확보
- [ ] 프로젝트 헌장 공식 서명
- [ ] 프로젝트 헌장 배포

### 1.3 킥오프 준비

#### 킥오프 미팅 계획
- [ ] 킥오프 미팅 일시 및 장소 확정
- [ ] 참석자 명단 작성 및 초대
- [ ] 킥오프 아젠다 준비
- [ ] 프레젠테이션 자료 작성
- [ ] 프로젝트 개요 문서 준비
- [ ] 팀 소개 자료 준비

#### 초기 커뮤니케이션
- [ ] 프로젝트 공지 이메일 발송
- [ ] 프로젝트 포털/공간 설정
- [ ] 문서 저장소 구축
- [ ] 커뮤니케이션 채널 확립

## 2. 계획 수립 체크리스트

### 2.1 범위 관리 계획

#### 요구사항 수집
- [ ] 이해관계자 인터뷰 수행
- [ ] 요구사항 워크숍 개최
- [ ] 브레인스토밍 세션 진행
- [ ] 설문조사 실시
- [ ] 프로토타입 개발 (필요시)
- [ ] 기존 문서 검토
- [ ] 벤치마킹 수행

#### 요구사항 문서화
- [ ] 비즈니스 요구사항 문서화
- [ ] 기능 요구사항 정의
- [ ] 비기능 요구사항 정의
- [ ] 기술적 요구사항 명시
- [ ] 품질 요구사항 정의
- [ ] 요구사항 우선순위 지정
- [ ] 인수 기준 정의

#### 요구사항 추적성
- [ ] 요구사항 추적 매트릭스 작성
- [ ] 각 요구사항에 고유 ID 부여
- [ ] 요구사항과 인도물 매핑
- [ ] 요구사항 소유자 배정

#### 범위 정의
- [ ] 프로젝트 범위 기술서 작성
- [ ] 제품 범위 상세 설명
- [ ] 프로젝트 경계 명확화
- [ ] 포함 사항 명시
- [ ] 제외 사항 명시
- [ ] 인도물 상세 설명

#### WBS 작성
- [ ] WBS 작성 방법론 선택 (하향식/상향식)
- [ ] 프로젝트를 주요 인도물로 분해
- [ ] 인도물을 작업 패키지로 세분화
- [ ] WBS 사전 작성 (각 요소 설명)
- [ ] 작업 패키지 ID 부여
- [ ] 책임자 배정
- [ ] WBS 검토 및 승인

### 2.2 일정 관리 계획

#### 활동 정의
- [ ] WBS 작업 패키지를 활동으로 분해
- [ ] 활동 목록 작성
- [ ] 활동 속성 정의
- [ ] 마일스톤 목록 작성

#### 활동 순서 배열
- [ ] 선후행 관계 식별
- [ ] 의존성 유형 결정 (FS, SS, FF, SF)
- [ ] 네트워크 다이어그램 작성
- [ ] 리드와 래그 적용
- [ ] 외부 의존성 식별

#### 활동 기간 산정
- [ ] 산정 기법 선택 (유사/모수/상향식)
- [ ] 각 활동의 기간 산정
- [ ] 3점 추정 수행 (필요시)
- [ ] 산정 근거 문서화
- [ ] 예비 시간 고려
- [ ] 전문가 검토

#### 일정 개발
- [ ] 주공정 분석 (CPM)
- [ ] 자원 가용성 고려
- [ ] 자원 평준화 수행
- [ ] 일정 압축 검토 (크래싱/패스트트래킹)
- [ ] 간트 차트 작성
- [ ] 마일스톤 일정 확정
- [ ] 일정 기준선 설정
- [ ] 버퍼 시간 확보

#### 일정 검증
- [ ] 일정의 현실성 검토
- [ ] 자원 과부하 확인
- [ ] 제약 조건 준수 확인
- [ ] 이해관계자 검토 및 승인

### 2.3 비용 관리 계획

#### 비용 산정
- [ ] 활동별 비용 산정
- [ ] 자원 비용 산정 (인건비)
- [ ] 장비 및 재료 비용 산정
- [ ] 외부 조달 비용 산정
- [ ] 간접비 산정
- [ ] 예비비 산정 (우발사태 예비비)
- [ ] 관리 예비비 산정
- [ ] 비용 산정 근거 문서화

#### 예산 수립
- [ ] 활동별 비용을 시간에 따라 집계
- [ ] S-커브 작성
- [ ] 단계별 예산 배분
- [ ] 비용 기준선 설정
- [ ] 자금 조달 계획 수립
- [ ] 예산 승인 확보

#### 비용 통제 계획
- [ ] EVM 측정 방법 정의
- [ ] 성과 측정 기준 설정
- [ ] 보고 주기 결정
- [ ] 임계값 설정 (경보 기준)

### 2.4 품질 관리 계획

#### 품질 기준 정의
- [ ] 조직 품질 정책 검토
- [ ] 산업 표준 및 규제 식별

- [ ] 품질 감사 계획 수립
- [ ] 프로세스 개선 방법 정의
- [ ] 품질 도구 및 기법 선택
- [ ] 품질 역할 및 책임 배정

#### 품질 통제 계획
- [ ] 검사 계획 수립
- [ ] 테스트 전략 수립
- [ ] 품질 메트릭 정의
- [ ] 샘플링 방법 결정
- [ ] 결함 추적 방법 정의

### 2.5 자원 관리 계획

#### 자원 요구사항 식별
- [ ] 활동별 필요 자원 식별

- [ ] 프로젝트 일정 및 마일스톤 공유
- [ ] 의사소통 규칙 및 도구 안내
- [ ] 기대사항 및 우려사항 청취
- [ ] 질의응답 시간
- [ ] 회의록 작성 및 배포

### 3.2 팀 구성 및 온보딩

- [ ] 팀원 배정 확정
- [ ] 외부 자원 계약 체결
- [ ] 대체 자원 확보 (필요시)

#### 온보딩 프로세스
- [ ] 프로젝트 오리엔테이션 제공
- [ ] 도구 및 시스템 접근 권한 부여
- [ ] 프로젝트 문서 공유
- [ ] 역할 및 책임 명확화
- [ ] 팀 규범 및 규칙 설명
- [ ] 멘토 또는 버디 배정 (필요시)

#### 팀 개발
- [ ] 팀 빌딩 활동 수행
- [ ] 교육 훈련 제공

- [ ] 갈등 예방 전략 실행

### 3.3 작업 실행

#### 작업 배정 및 관리
- [ ] 작업 패키지 배정
- [ ] 작업 지시 명확히 전달

- [ ] 일일/주간 스탠드업 미팅
- [ ] 작업 진행률 업데이트
- [ ] 완료된 작업 검증
- [ ] 지연 요인 식별 및 제거

#### 이슈 및 장애물 관리
- [ ] 이슈 식별 및 로그 기록

## 체크리스트 사용 팁

### 효과적인 활용 방법
1. **맞춤화**: 프로젝트 특성에 맞게 체크리스트를 조정
2. **정기 검토**: 체크리스트를 정기적으로 검토하여 누락 방지
3. **팀 공유**: 팀원들과 체크리스트를 공유하여 공동 책임
4. **지속 개선**: 프로젝트 경험을 바탕으로 체크리스트 개선
5. **도구 활용**: 체크리스트를 프로젝트 관리 도구에 통합

### 주의사항
- 체크리스트는 가이드이며, 프로젝트 상황에 따라 유연하게 적용
- 체크박스만 체크하는 것이 아닌, 실질적인 활동 수행이 중요
- 모든 항목이 모든 프로젝트에 적용되는 것은 아님
- 팀의 피드백을 반영하여 지속적으로 개선
