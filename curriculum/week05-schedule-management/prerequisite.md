## 1️⃣ 일정(Schedule)이란?

### 정의
**일정**은 "언제 무엇을 할 것인가?"를 시간 순서대로 정리한 계획입니다.

### 실생활 비유
```
아침 출근 준비 일정:
- 06:00 기상
- 06:05 세면
- 06:15 아침 식사
- 06:30 옷 입기
- 06:40 출발
- 07:00 회사 도착
```

### PM 관점
프로젝트 일정은:
- **무엇을** (작업)
- **언제** (시작일, 종료일)
- **얼마나** (소요 기간)
- **누가** (담당자)
를 명확히 합니다.

## 2️⃣ 의존 관계 (Dependencies)

### 정의
작업 간의 **선후 관계** - "A가 끝나야 B를 시작할 수 있다"

### 4가지 유형

#### FS (Finish-to-Start) - 가장 일반적
**"A가 끝나야 B 시작"**

**예시:**
```
결혼식 준비:
A: 예식장 계약 완료 (끝)
→ B: 청첩장 인쇄 시작
```

#### SS (Start-to-Start)
**"A가 시작하면 B도 시작"**

**예시:**
```
집 짓기:
A: 기초 공사 시작
→ B: 자재 납품 시작 (동시에)
```

#### FF (Finish-to-Finish)
**"A가 끝나면 B도 끝"**

**예시:**
```
책 출판:
A: 원고 작성 완료
→ B: 편집 작업 완료 (같이)
```

#### SF (Start-to-Finish) - 드물게 사용
**"A가 시작하면 B 종료"**

**예시:**
```
야간 경비:
A: 아침 경비 시작
→ B: 야간 경비 종료
```

### 쉬운 비유
```
🍳 계란 프라이 만들기 (FS):
1. 달걀 깨기 → 2. 프라이팬에 붓기 → 3. 익히기 → 4. 접시에 담기

🏃 마라톤 훈련 (SS):
달리기 시작 + 음악 듣기 시작 (동시)

📚 시험 공부 (FF):
교재 읽기 완료 + 문제 풀기 완료 (같이 끝)
```

## 3️⃣ 리드(Lead)와 래그(Lag)

### 리드 (Lead) - 앞당기기
**선행 작업이 끝나기 전에 후속 작업 시작**

**예시:**
```
집 페인트칠:
A: 1층 페인트칠 (3일 소요)
B: 2층 페인트칠

리드 없음: A 완료 후 B 시작 (총 6일)
리드 1일: A 2일 완료 후 B 시작 (총 5일)
→ 1일 절약! ✅
```

### 래그 (Lag) - 대기 시간
**선행 작업이 끝난 후 대기 후 후속 작업 시작**

**예시:**
```
잔디 심기:
A: 잔디 심기 완료
[3일 대기 - 뿌리 내리는 시간]
B: 물 주기 시작

래그 3일 설정 필요
```

### 실생활 비유
```
🎂 케이크 만들기:
- 리드: 굽기 완료 전에 크림 만들기 시작 (시간 단축)
- 래그: 케이크 구운 후 2시간 식히기 대기 (필수 대기)
```

## 4️⃣ 간트 차트 (Gantt Chart)

### 정의
시간을 가로축으로, 작업을 세로축으로 하여 **막대 형태**로 일정을 표시한 차트

### 구성 요소
```
작업      │1월  2월  3월  4월
─────────┼────────────────────
기획      │████
디자인    │    ████
개발      │      ████████
테스트    │            ████
배포      │                █
```

### 실생활 비유
```
여름 휴가 계획 간트 차트:
작업          │5월 6월 7월 8월
─────────────┼──────────────
여행지 검색   │██
항공권 예약   │  ██
숙소 예약     │    ██
짐 준비       │        ██
여행          │          ████
```

### 왜 유용한가?
- 한눈에 전체 일정 파악
- 작업 간 의존 관계 표시
- 진행 상황 추적 용이
- 이해관계자와 공유 쉬움

## 5️⃣ 크리티컬 패스 (Critical Path)

### 정의
**프로젝트의 가장 긴 경로** - 이 경로가 지연되면 전체 프로젝트가 지연됨

### 왜 중요한가?
- 전체 프로젝트 기간을 결정
- 여유 시간이 전혀 없음 (Float = 0)
- 집중 관리 필요

### 실생활 비유
```
결혼식 준비 (3개월 프로젝트):

경로 1:
예식장 계약(1주) → 드레스 맞춤(4주) → 리허설(1주) = 6주

경로 2:
초청장 디자인(2주) → 인쇄(1주) → 발송(1주) = 4주

경로 3:
혼수 쇼핑(2주) → 집 정리(2주) = 4주

→ 크리티컬 패스: 경로 1 (6주) ⚠️
→ 드레스 맞춤이 지연되면 결혼식 날짜를 미뤄야 함!
```

### PM 관점
```
웹사이트 구축:

경로 A (크리티컬):
기획(2주) → 디자인(3주) → 프론트 개발(4주) → 테스트(2주) = 11주

경로 B:
기획(2주) → 백엔드 개발(3주) → 테스트(2주) = 7주

→ 크리티컬 패스: 경로 A (11주)
→ 프론트 개발에 자원 집중 필요!
```

## 6️⃣ 여유 시간 (Float/Slack)

### 정의
작업을 **지연시켜도 전체 프로젝트에 영향 없는 시간**

### 종류

#### 총 여유 시간 (Total Float)
**전체 프로젝트 지연 없이 지연 가능한 최대 시간**

**예시:**
```
작업 A: 3일 소요, 총 여유 2일
→ 5일까지 써도 프로젝트 전체에 영향 없음
```

#### 자유 여유 시간 (Free Float)
**다음 작업에 영향 없이 지연 가능한 시간**

**예시:**
```
작업 A → 작업 B
A의 자유 여유 1일
→ 1일 지연해도 B의 시작일에 영향 없음
```

### 실생활 비유
```
🚗 자동차 여행:
A 지점 → B 지점 (200km, 2시간 소요)
도착 마감 시간: 오후 5시
출발 시간: 오후 1시

총 여유 시간: 2시간
- 오후 3시까지 출발해도 5시 도착 가능
- 1시에 출발하면 2시간 여유
```

## 7️⃣ 일정 압축 기법

### 왜 필요한가?
- 고객이 일정 단축 요구
- 경쟁사보다 빨리 출시 필요
- 프로젝트 지연 발생 시 회복

### 2가지 기법

#### 크래싱 (Crashing)
**자원을 추가 투입하여 기간 단축**

**예시:**
```
웹 개발:
원래 계획: 개발자 2명, 8주
크래싱: 개발자 4명, 5주

✅ 장점: 일정 단축 확실
❌ 단점: 비용 증가
```

**실생활 비유:**
```
🏠 이사 짐 싸기:
혼자: 8시간
가족 4명: 3시간 (크래싱)
→ 시간은 줄지만, 저녁 식사 비용 증가 (피자 시킴)
```

#### 패스트 트래킹 (Fast Tracking)
**순차 작업을 병렬로 수행**

**예시:**
```
원래: 설계 완료 → 개발 시작
패스트 트래킹: 설계 80% 완료 → 개발 시작

✅ 장점: 비용 증가 없음
❌ 단점: 재작업 리스크 (설계 변경 시)
```

**실생활 비유:**
```
🍳 저녁 식사 준비:
순차: 밥 짓기(30분) → 반찬 만들기(20분) = 50분
병렬: 밥 짓는 동안 반찬 동시에 만들기 = 30분

위험: 두 가지 동시에 하다 실수할 수 있음
```

## 💬 자주 묻는 질문 (FAQ)

**Q1: 크리티컬 패스는 항상 하나인가요?**  
A: 아닙니다! 여러 개의 크리티컬 패스가 존재할 수 있습니다. 모두 같은 길이로 가장 긴 경로이면 모두 크리티컬 패스입니다.

**Q2: 크래싱을 하면 항상 기간이 단축되나요?**  
A: 아닙니다! "임신은 9개월"처럼 사람을 아무리 많이 투입해도 단축 안 되는 작업도 있습니다. (예: 시멘트 굳는 시간)

**Q3: 간트 차트를 꼭 전문 도구로 만들어야 하나요?**  
A: 아닙니다! 작은 프로젝트는 Excel이나 Google Sheets로도 충분합니다. 복잡한 프로젝트는 MS Project, Jira 등 전문 도구를 권장합니다.

**Q4: 일정이 지연되면 어떻게 하나요?**  
A: 먼저 크리티컬 패스의 지연인지 확인하세요. 크리티컬이 아니면 여유 시간 내에서 해결 가능합니다. 크리티컬 패스 지연이면 크래싱이나 패스트 트래킹을 고려하세요.

**Q5: 모든 작업에 의존 관계를 설정해야 하나요?**  
A: 주요 작업 간 의존 관계는 반드시 설정해야 합니다. 너무 세세하게 하면 관리가 어려우니 적절한 수준에서 설정하세요.
