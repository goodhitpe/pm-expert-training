- 리스크 대응 전략 수립

## 📚 주요 내용

### 1. 품질 관리

#### 1.1 품질의 정의
- **품질**: 요구사항 충족 정도
- **등급 (Grade)** vs **품질 (Quality)**
  - 낮은 등급, 높은 품질: 가능 (기본형 제품이지만 결함 없음)
  - 높은 등급, 낮은 품질: 불가 (고급 제품이지만 결함 많음)

#### 1.2 품질 관리 프로세스
- **품질 관리 계획**: 품질 요구사항 및 표준 정의
- **품질 보증 (QA)**: 프로세스 준수 확인
- **품질 통제 (QC)**: 산출물 검증

#### 1.3 품질 비용 (Cost of Quality)
**적합 비용 (Cost of Conformance):**
- 예방 비용: 교육, 프로세스 문서화
- 평가 비용: 검사, 테스트

**부적합 비용 (Cost of Non-Conformance):**
- 내부 실패 비용: 재작업, 폐기
- 외부 실패 비용: 보증 비용, 고객 이탈

#### 1.4 품질 관리 기법

**7가지 기본 품질 도구:**
1. **원인-결과 다이어그램 (Fishbone/Ishikawa)**: 문제의 근본 원인 분석
2. **파레토 차트**: 80/20 법칙, 주요 원인 식별
3. **히스토그램**: 데이터 분포 시각화
4. **관리도 (Control Chart)**: 프로세스 안정성 모니터링
5. **산점도 (Scatter Diagram)**: 변수 간 상관관계
6. **체크시트**: 데이터 수집
7. **흐름도 (Flowchart)**: 프로세스 시각화

**추가 기법:**
- 통계적 샘플링
- 벤치마킹
- 설계 검토
- 품질 감사

#### 1.5 품질 메트릭
- 결함률 (Defect Rate)
- 테스트 커버리지
- 고객 만족도
- 재작업 비율

### 2. 리스크 관리

#### 2.1 리스크의 정의
- **리스크**: 발생할 수 있는 불확실한 사건
- **긍정적 리스크 (기회)**: 프로젝트에 유리한 영향
- **부정적 리스크 (위협)**: 프로젝트에 불리한 영향

#### 2.2 리스크 관리 프로세스

**1단계: 리스크 관리 계획 수립**
- 리스크 관리 접근 방법 정의
- 역할 및 책임 할당
- 리스크 카테고리 정의

**2단계: 리스크 식별**
- 브레인스토밍
- 체크리스트
- 델파이 기법
- SWOT 분석
- 전문가 인터뷰

**3단계: 정성적 리스크 분석**
- 발생 확률 평가
- 영향도 평가
- 우선순위 매트릭스 작성

**4단계: 정량적 리스크 분석**
- 민감도 분석
- EMV (Expected Monetary Value) 분석
- 의사결정 트리
- 시뮬레이션 (Monte Carlo)

**5단계: 리스크 대응 계획**
- 대응 전략 수립
- 우발 계획 (Contingency Plan)
- 대체 계획 (Fallback Plan)

**6단계: 리스크 모니터링**
- 리스크 재평가
- 트리거 모니터링
- 새로운 리스크 식별

#### 2.3 리스크 확률 및 영향도 매트릭스

```
영향도
높음  │  중간   높음   매우높음
중간  │  낮음   중간   높음
낮음  │  매우낮음 낮음   중간
      └──────────────────
       낮음  중간  높음  
            확률
```

#### 2.4 리스크 대응 전략

**부정적 리스크 (위협) 전략:**
1. **회피 (Avoid)**: 리스크 원인 제거
   - 예: 신기술 사용 대신 검증된 기술 사용

2. **전가 (Transfer)**: 제3자에게 리스크 이전
   - 예: 보험 가입, 외주

3. **완화 (Mitigate)**: 확률 또는 영향도 감소
   - 예: 프로토타입 제작, 추가 테스트

4. **수용 (Accept)**: 리스크 인정하고 대응 계획 수립
   - 적극적 수용: 우발 계획 준비
   - 소극적 수용: 발생 시 대응

**긍정적 리스크 (기회) 전략:**
1. **활용 (Exploit)**: 기회 확실하게 실현
2. **공유 (Share)**: 파트너와 기회 공유
3. **강화 (Enhance)**: 확률 또는 영향도 증대
4. **수용 (Accept)**: 기회 활용하되 적극 추구 안 함

#### 2.5 리스크 등록부 (Risk Register)

**포함 내용:**
- 리스크 ID
- 리스크 설명
- 카테고리
- 원인
- 발생 확률 (1-5)
- 영향도 (1-5)
- 점수 (확률 × 영향도)
- 대응 전략
- 담당자
- 우발 계획
- 트리거 (조짐)
- 상태

#### 2.6 EMV (Expected Monetary Value) 분석

**공식:**
EMV = 확률 × 영향

**예제:**
- 리스크 A: 30% 확률, -50만원 영향 → EMV = -15만원
- 기회 B: 40% 확률, +30만원 영향 → EMV = +12만원
- 전체 EMV = -3만원

**의사결정 트리:**
```
          대안 A (70% 성공)
         /      → 성공: +100만원
프로젝트
         \대안 B (50% 성공)
                → 성공: +150만원

EMV(A) = 0.7 × 100 = 70만원
EMV(B) = 0.5 × 150 = 75만원
→ 대안 B 선택
```

### 3. 통합 품질-리스크 관리

#### 3.1 품질 리스크
- 품질 미달 리스크 식별
- 품질 통제 실패의 영향 평가
- 예방 조치 강화

#### 3.2 리스크 기반 테스팅
- 고위험 영역 집중 테스트
- 테스트 우선순위 설정
- 리스크 완화를 위한 품질 활동

### 과제 2: 리스크 등록부 작성

## 📝 핵심 용어
- **품질 보증 (QA)**: 프로세스 중심의 품질 관리
- **품질 통제 (QC)**: 산출물 중심의 품질 관리
- **리스크 (Risk)**: 불확실한 사건 또는 조건
- **EMV (Expected Monetary Value)**: 기대 금전 가치
- **리스크 등록부 (Risk Register)**: 리스크 목록 및 분석 문서
- **파레토 법칙 (Pareto Principle)**: 80/20 법칙
- **우발 계획 (Contingency Plan)**: 리스크 발생 시 대응 계획
