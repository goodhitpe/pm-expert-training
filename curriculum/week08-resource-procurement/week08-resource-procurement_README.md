## 📚 주요 내용

### 1. 자원 관리

#### 1.1 자원의 유형
- **인적 자원 (Human Resources)**: 팀원, 전문가
- **물적 자원 (Physical Resources)**: 장비, 자재, 시설
- **금전적 자원 (Financial Resources)**: 예산
- **정보 자원 (Information Resources)**: 지식, 데이터

#### 1.2 자원 관리 프로세스

**1단계: 자원 관리 계획**
- 자원 필요량 산정
- 역할 및 책임 정의 (RACI 차트)
- 조직 구조 설계

**2단계: 활동 자원 산정**
- WBS 활동별 필요 자원 식별
- 자원 캘린더 확인
- 자원 가용성 평가

**3단계: 팀 확보**
- 팀원 선발
- 협상 (다른 부서, 외부)
- 가상 팀 구성

**4단계: 팀 개발**
- 팀 빌딩 활동
- 교육 훈련
- 성과 향상

**5단계: 팀 관리**
- 갈등 해결
- 성과 평가
- 동기 부여

**6단계: 자원 통제**
- 자원 활용도 모니터링
- 자원 재배치
- 자원 평준화

#### 1.3 RACI 매트릭스

```
활동      │ PM │ 개발자 │ 디자이너 │ 고객
─────────┼────┼────────┼──────────┼──────
요구사항  │ A  │   C    │    I     │  R
설계      │ A  │   R    │    R     │  C
개발      │ A  │   R    │    I     │  I
테스트    │ A  │   R    │    C     │  I
```

- **R (Responsible)**: 실행 책임
- **A (Accountable)**: 최종 책임 (1명만 가능)
- **C (Consulted)**: 사전 협의
- **I (Informed)**: 사후 통보

#### 1.4 자원 최적화 기법

**자원 평준화 (Resource Leveling):**
- 자원 제약 조건 우선
- 일정 연장 가능
- 자원 과다 배정 해소

**자원 평탄화 (Resource Smoothing):**
- 일정 제약 조건 우선
- 일정 변경 없음
- 여유 시간 활용

### 2. 팀 관리

#### 2.1 팀 개발 단계 (Tuckman 모델)

1. **형성 (Forming)**: 팀 구성, 서로 알아가기
2. **혼란 (Storming)**: 갈등 발생, 역할 경쟁
3. **규범 (Norming)**: 규칙 확립, 협력 시작
4. **수행 (Performing)**: 높은 성과 창출
5. **해산 (Adjourning)**: 프로젝트 종료

#### 2.2 동기 부여 이론

**매슬로우 욕구 5단계:**
```
    자아실현
    존중 욕구
   사회적 욕구
   안전 욕구
   생리적 욕구
```

**허츠버그 2요인 이론:**
- **위생 요인**: 불만족 방지 (급여, 근무 환경)
- **동기 요인**: 만족 증진 (인정, 성취감)

**맥그리거 X-Y 이론:**
- **X 이론**: 사람은 본질적으로 일을 싫어함 → 통제 필요
- **Y 이론**: 사람은 자기주도적 → 권한 부여

#### 2.3 갈등 관리

**갈등의 원인:**
- 자원 경쟁
- 일정 압박
- 우선순위 차이
- 의사소통 문제
- 성격 차이

**갈등 해결 기법:**
1. **협력/문제 해결 (Collaborate/Problem Solve)**: 양측 모두 만족 (Win-Win)
2. **타협 (Compromise)**: 중간점 찾기 (부분 Win-Win)
3. **강제/지시 (Force/Direct)**: PM이 결정 (Win-Lose)
4. **완화/수용 (Smooth/Accommodate)**: 공통점 강조, 차이 최소화
5. **회피 (Withdraw/Avoid)**: 갈등 회피 또는 연기

**효과적 순서:** 협력 > 타협 > 강제 > 완화 > 회피

#### 2.4 성과 평가
- 정기적 1:1 미팅
- 360도 피드백
- KPI 기반 평가
- 건설적 피드백 제공

### 3. 조달 관리

#### 3.1 조달 관리 프로세스

**1단계: 조달 관리 계획**
- Make vs Buy 결정
- 계약 유형 선정
- 공급업체 선정 기준

**2단계: 조달 수행**
- 입찰 공고 (RFI, RFP, RFQ)
- 제안서 평가
- 공급업체 선정
- 계약 체결

**3단계: 조달 통제**
- 계약 이행 모니터링
- 변경 관리
- 대금 지급 승인
- 클레임 관리

#### 3.2 계약 유형

**고정가 계약 (Fixed Price, FP):**
- 총액 고정
- 판매자 리스크 높음
- 구매자에게 유리

**원가 정산 계약 (Cost Reimbursable, CR):**
- 실비 + 수수료
- 구매자 리스크 높음
- 판매자에게 유리

**T&M (Time and Material):**
- 시간 × 단가 + 자재비
- 중간 리스크
- 범위 불명확 시 유용

#### 3.3 조달 문서

**RFI (Request for Information):**
- 정보 요청서
- 시장 조사 목적

**RFP (Request for Proposal):**
- 제안 요청서
- 상세한 제안서 필요

**RFQ (Request for Quotation):**
- 견적 요청서
- 가격 중심

#### 3.4 공급업체 선정 기준
- 기술 역량
- 가격 경쟁력
- 납기 준수 능력
- 품질 수준
- 재무 안정성
- 참조 프로젝트

#### 3.5 협상 기법
- Win-Win 추구
- BATNA (Best Alternative to Negotiated Agreement) 준비
- 양보 전략
- 적극적 경청
- 객관적 기준 활용

### 4. 가상 팀 관리

#### 4.1 가상 팀의 특징
- 지리적 분산
- 시차
- 문화 차이
- 대면 접촉 제한

#### 4.2 가상 팀 관리 전략
- 명확한 커뮤니케이션 규칙
- 협업 도구 활용 (Slack, Zoom, Teams)
- 정기적 화상 회의
- 문서화 강화
- 신뢰 구축
- 시차 고려한 회의 시간

## 💡 실습 과제

## 📝 핵심 용어
- **RACI 매트릭스**: 역할 책임 배정표
- **자원 평준화 (Resource Leveling)**: 자원 제약 기반 일정 조정
- **자원 평탄화 (Resource Smoothing)**: 일정 유지하며 자원 조정
- **고정가 계약 (Fixed Price)**: 총액 고정 계약
- **원가 정산 계약 (Cost Reimbursable)**: 실비 보상 계약
- **RFP (Request for Proposal)**: 제안 요청서
- **BATNA**: 협상 결렬 시 최선의 대안
