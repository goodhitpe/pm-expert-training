Week 3에서는 프로젝트 이해관계자 식별, 분석, 참여 전략, 효과적인 커뮤니케이션 방법을 다룹니다.

## 1️⃣ 이해관계자(Stakeholder)란?

### 정의
**이해관계자**는 프로젝트에 영향을 주거나 프로젝트로부터 영향을 받는 모든 사람, 그룹, 조직을 말합니다.

### 실생활 비유
```
결혼식 준비를 프로젝트라고 가정:
- 신랑, 신부: 핵심 이해관계자 (직접 당사자)
- 양가 부모님: 주요 의사결정권자 (예산 승인)
- 하객들: 최종 사용자 (서비스 이용자)
- 웨딩플래너: 프로젝트 팀
- 예식장, 스드메 업체: 공급업체
- 주례 선생님: 외부 전문가
```

### PM 관점
프로젝트가 성공하려면 **모든 이해관계자를 만족**시켜야 합니다. 한 명이라도 불만족하면 프로젝트가 실패할 수 있습니다.

## 2️⃣ 이해관계자의 권력(Power)과 관심(Interest)

### 권력이란?
프로젝트에 **영향을 미칠 수 있는 능력**

**예시:**
- 높은 권력: CEO, 프로젝트 스폰서, 예산 담당자
- 낮은 권력: 일반 직원, 간접 사용자

### 관심이란?
프로젝트 결과에 대한 **관심도와 참여 의지**

**예시:**
- 높은 관심: 프로젝트로 업무가 바뀌는 직원
- 낮은 관심: 간접적으로만 영향 받는 타 부서

### 쉬운 비유
```
아파트 엘리베이터 교체 프로젝트:
- 높은 권력, 높은 관심: 입주자 대표 (결정권 있고, 매일 사용)
- 높은 권력, 낮은 관심: 건물주 (결정권 있지만, 거주하지 않음)
- 낮은 권력, 높은 관심: 1층 거주자 (사용은 하지만, 결정권 없음)
- 낮은 권력, 낮은 관심: 방문객 (가끔 사용, 결정권 없음)
```

### PM 전략
- **높은 권력 + 높은 관심**: 긴밀히 관리 (정기 보고, 의사결정 참여)
- **높은 권력 + 낮은 관심**: 만족시키기 (요약 보고, 주요 사안만)
- **낮은 권력 + 높은 관심**: 정보 제공 (상세 업데이트, 의견 수렴)
- **낮은 권력 + 낮은 관심**: 모니터링만 (필요 시 정보 제공)

## 3️⃣ 커뮤니케이션의 기본 원칙

### 커뮤니케이션이란?
정보, 의견, 감정을 주고받는 과정

### 5가지 핵심 원칙

#### 1. 명확성 (Clarity)
**의미**: 모호하지 않고 분명하게 전달

**나쁜 예**: "빨리 해주세요"  
**좋은 예**: "금요일 오후 5시까지 보고서를 이메일로 보내주세요"

#### 2. 일관성 (Consistency)
**의미**: 같은 내용을 동일하게 전달

**실생활 비유**:
```
부모님이 아이에게:
- 엄마: "게임은 하루 30분만"
- 아빠: "게임은 1시간까지 괜찮아"
→ 일관성 없음, 혼란 발생 ❌

일관되게 전달:
- 부모 모두: "게임은 하루 30분만"
→ 명확한 메시지 ✅
```

#### 3. 적시성 (Timeliness)
**의미**: 적절한 시기에 전달

**예시**:
- 나쁨: 회의 5분 전에 자료 공유
- 좋음: 회의 1일 전에 자료 공유

#### 4. 양방향 소통 (Feedback)
**의미**: 일방적 전달이 아닌, 피드백 주고받기

**예시**:
- 발신자: "제 설명이 이해되셨나요?"
- 수신자: "네, 이해했습니다. 단, A 부분은 재확인 필요합니다."

#### 5. 문서화 (Documentation)
**의미**: 중요한 내용은 기록으로 남기기

**실생활 비유**:
```
카카오톡으로 친구와 약속:
- 구두만: "다음 주에 만나자" → 날짜 혼동 가능
- 문서화: "2025년 2월 15일(토) 오후 3시 강남역 2번 출구" → 명확
```

## 4️⃣ 회의의 종류와 목적

### 회의란?
특정 목적을 달성하기 위해 여러 사람이 모여 논의하는 것

### 주요 회의 유형

#### 킥오프 미팅 (Kick-off Meeting)
- **목적**: 프로젝트 시작 선언, 팀 소개
- **비유**: 신학기 첫날 OT

#### 주간 진행 회의
- **목적**: 진행 상황 공유, 이슈 해결
- **비유**: 가족 주간 회의 (이번 주 일정 공유)

#### 이슈 해결 회의
- **목적**: 특정 문제 해결
- **비유**: 자동차 고장 시 정비소 방문

#### 회고 회의 (Retrospective)
- **목적**: 프로젝트 종료 후 교훈 정리
- **비유**: 여행 후 사진 보며 추억 회상

## 5️⃣ 효과적인 회의 진행 방법

### 회의 전 준비
1. **목적 명확히**: "오늘 회의에서 무엇을 결정할 것인가?"
2. **의제 작성**: 논의할 항목 리스트업
3. **참석자 선정**: 꼭 필요한 사람만
4. **자료 준비**: 사전에 공유

### 실생활 비유
```
가족 여행 계획 회의:
❌ 준비 없이:
  - "어디 갈까?" (목적지 정하느라 30분 소요)
  - "언제 갈까?" (일정 조율 30분)
  - 결론: 결정 못 함

✅ 준비하고:
  - 의제: 1) 목적지 선정(후보 3곳), 2) 일정, 3) 예산
  - 사전 자료: 각 후보지 정보 공유
  - 결론: 30분 만에 모든 결정 완료
```

### 회의 중
- **시간 엄수**: 시작과 종료 시간 준수
- **의제 준수**: 주제에서 벗어나지 않기
- **액션 아이템 기록**: 누가, 무엇을, 언제까지

### 회의 후
- **회의록 작성**: 결정 사항 정리
- **공유**: 참석자 및 관련자에게 전송
- **후속 조치**: 액션 아이템 추적

## 6️⃣ 보고의 기본

### 보고란?
상급자나 이해관계자에게 진행 상황을 알리는 것

### 효과적인 보고의 3가지 원칙

#### 1. 결론부터 (Bottom-line Up Front)
**나쁜 예**: "지난주에 A를 했고, B도 했고, C도 했는데..."  
**좋은 예**: "프로젝트는 계획대로 진행 중입니다. 주요 성과는..."

#### 2. 간결하게 (Keep It Simple)
**나쁜 예**: A4 10장짜리 상세 보고서  
**좋은 예**: A4 1장 요약 + 상세는 첨부

#### 3. 시각화 (Visualization)
**나쁜 예**: "예산의 70%를 사용했습니다"  
**좋은 예**: [파이 차트 또는 진행바로 표시]

### 실생활 비유
```
부모님께 학교 성적 보고:
❌ 비효율적:
  "국어 시험은 어려웠는데, 수학은 쉬웠고, 영어는 보통이었고..."

✅ 효율적:
  "이번 학기 평균은 A입니다. (결론)
   국어 A, 수학 A+, 영어 B+ (간결)
   [성적표 사진 첨부] (시각화)"
```

## 7️⃣ RAG 상태 표시

### RAG란?
신호등 색상으로 프로젝트 상태를 표시하는 방법

### 색상 의미
- **🟢 Green (녹색)**: 정상, 문제 없음
- **🟡 Amber (황색)**: 주의 필요, 잠재적 위험
- **🔴 Red (적색)**: 위험, 즉시 조치 필요

### 실생활 비유
```
건강 검진 결과:
- 🟢 혈압: 정상 범위
- 🟡 콜레스테롤: 경계 수치 (식단 조절 필요)
- 🔴 혈당: 위험 수치 (치료 필요)
```

### PM 활용
```
프로젝트 대시보드:
- 일정: 🟢 (계획대로)
- 예산: 🟡 (10% 초과 우려)
- 품질: 🟢 (이슈 없음)
- 리스크: 🔴 (핵심 인력 퇴사)
```

## 💬 자주 묻는 질문 (FAQ)

**Q1: 이해관계자가 너무 많으면 어떻게 하나요?**  
A: 모든 이해관계자를 똑같이 관리할 필요는 없습니다. 권력/관심 매트릭스로 우선순위를 정하고, 핵심 이해관계자에 집중하세요.

**Q2: 이해관계자가 서로 상충되는 요구를 하면?**  
A: 이는 매우 흔한 상황입니다. Week 3에서 이해관계자 간 균형을 맞추는 전략을 배우게 됩니다. (예: 협상, 우선순위 조정)

**Q3: 회의가 너무 길어지는데 어떻게 하나요?**  
A: 회의 전 명확한 의제와 시간 배분이 핵심입니다. 타이머를 사용하고, 주제에서 벗어나면 "주차"(parking lot)에 기록 후 나중에 논의하세요.

**Q4: 비대면 회의에서도 같은 원칙이 적용되나요?**  
A: 네! 오히려 비대면에서는 사전 준비(의제, 자료)와 명확한 커뮤니케이션이 더욱 중요합니다.

**Q5: 보고를 자주 해야 하나요?**  
A: 이해관계자마다 다릅니다. CEO는 월 1회 요약 보고, 직속 상사는 주 1회 상세 보고가 일반적입니다. Week 3에서 커뮤니케이션 계획을 세우는 방법을 배웁니다.
