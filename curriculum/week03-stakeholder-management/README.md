## 📚 주요 내용

### 1. 이해관계자 관리

#### 1.1 이해관계자란?
- 프로젝트에 영향을 주거나 받는 개인, 그룹, 조직
- 내부/외부 이해관계자 구분

#### 1.2 이해관계자 식별
**주요 이해관계자 유형:**
- 프로젝트 스폰서 (Project Sponsor)
- 프로젝트 팀원
- 고객 (Customer)
- 최종 사용자 (End User)
- 공급업체 (Vendor)
- 경영진
- 규제 기관

#### 1.3 이해관계자 분석

**권력/관심 매트릭스 (Power/Interest Grid)**
```
높은 권력
    ┌──────────┬──────────┐
    │  관리    │  긴밀히  │
    │  만족    │  관리    │
    ├──────────┼──────────┤ 높은 관심
    │  모니터  │  정보    │
    │          │  제공    │
    └──────────┴──────────┘
낮은 권력
```

**영향력/관심 매트릭스 활용법:**
- **긴밀히 관리 (High Power, High Interest)**: 주요 의사결정자
- **정보 제공 (Low Power, High Interest)**: 정기적인 커뮤니케이션
- **관리 만족 (High Power, Low Interest)**: 적절한 정보 제공
- **모니터 (Low Power, Low Interest)**: 최소한의 관리

#### 1.4 이해관계자 참여 전략
- **비협조 (Unaware)** → **저항 (Resistant)** → **중립 (Neutral)** → **지원 (Supportive)** → **주도 (Leading)**
- 각 이해관계자의 현재 상태와 목표 상태 정의
- 참여 전략 수립

### 2. 커뮤니케이션 관리

#### 2.1 커뮤니케이션 계획
- **누구에게 (Who)**: 대상 식별
- **무엇을 (What)**: 전달 내용
- **언제 (When)**: 빈도 및 시기
- **어떻게 (How)**: 방법 및 채널

#### 2.2 커뮤니케이션 방법
- **구두 (Verbal)**: 회의, 전화, 발표
- **서면 (Written)**: 이메일, 보고서, 문서
- **비공식 (Informal)**: 대화, 메신저
- **공식 (Formal)**: 공식 회의, 보고서

#### 2.3 효과적인 커뮤니케이션 기법
1. **명확성**: 간결하고 분명한 표현
2. **일관성**: 동일한 메시지 유지
3. **적시성**: 적절한 시기에 전달
4. **피드백**: 양방향 소통
5. **문서화**: 중요 내용 기록

### 3. 회의 운영

#### 3.1 회의 유형
- 킥오프 미팅 (Kick-off Meeting)
- 주간 진행 회의 (Weekly Status Meeting)
- 이슈 해결 회의
- 검토 회의 (Review Meeting)
- 회고 회의 (Retrospective)

#### 3.2 효과적인 회의 진행
**회의 전:**
- 명확한 목적 설정
- 의제 사전 배포
- 참석자 선정
- 자료 준비

**회의 중:**
- 시간 엄수
- 의제 준수
- 액션 아이템 기록
- 결론 도출

**회의 후:**
- 회의록 작성 및 배포
- 액션 아이템 추적
- 후속 조치

#### 3.3 회의록 작성 요령
- 회의 기본 정보 (일시, 장소, 참석자)
- 주요 논의 사항
- 결정 사항
- 액션 아이템 (담당자, 기한)
- 다음 회의 일정

### 4. 보고서 작성

#### 4.1 프로젝트 진행 보고서 (Status Report)
**포함 내용:**
- 프로젝트 개요
- 주요 성과 (Accomplishments)
- 이슈 및 리스크
- 다음 단계 계획
- 지표 (일정, 예산, 품질)

#### 4.2 대시보드 활용
- RAG 상태 표시 (Red/Amber/Green)
- 주요 지표 시각화
- 트렌드 분석

#### 4.3 경영진 보고 (Executive Summary)
- 한 페이지 요약
- 핵심 메시지 강조
- 의사결정 필요 사항 명시
- 시각 자료 활용

- **이해관계자 (Stakeholder)**: 프로젝트에 관련된 모든 당사자
