## 1️⃣ PM 도구란 무엇인가?

### 쉬운 설명
**PM 도구**는 프로젝트 관리를 쉽고 효율적으로 할 수 있게 도와주는 소프트웨어입니다.

### 실생활 비유
**가계부 앱**을 생각해보세요:
- 종이 가계부 = 수동 프로젝트 관리 (엑셀, 문서)
- 가계부 앱 = PM 도구 (Jira, MS Project)

앱을 사용하면:
- ✅ 자동 계산 (수입/지출 집계)
- ✅ 시각화 (그래프, 차트)
- ✅ 알림 기능 (납부일 알림)
- ✅ 데이터 백업 및 공유

PM 도구도 마찬가지로 **자동화, 시각화, 협업**을 지원합니다!

### PM 도구를 쓰는 이유
- **가시성**: 프로젝트 상황을 한눈에 파악
- **협업**: 팀원들과 실시간 정보 공유
- **자동화**: 반복 작업 줄이고 효율 향상
- **추적**: 누가 무엇을 언제 했는지 기록
- **보고**: 자동 리포트 생성

## 2️⃣ 칸반 보드(Kanban Board)

### 정의
작업을 **카드 형태**로 만들어 진행 상태별로 이동시키는 시각적 관리 방법입니다.

### 실생활 비유
**세탁물 관리**를 생각해보세요:

```
┌───────────┬───────────┬───────────┐
│  세탁 전  │  세탁 중  │  세탁 완료 │
├───────────┼───────────┼───────────┤
│ [셔츠]    │ [청바지]  │ [양말]    │
│ [수건]    │           │ [속옷]    │
└───────────┴───────────┴───────────┘
```

옷이 세탁 과정을 거치며 칸을 이동합니다. 
프로젝트 작업도 마찬가지로 "할 일 → 진행 중 → 완료"로 이동합니다.

### 기본 구조
```
┌──────────┬──────────┬──────────┬──────────┐
│  To Do   │ In Progress │ Review  │  Done   │
├──────────┼──────────┼──────────┼──────────┤
│ 작업 A   │  작업 B  │  작업 D  │  작업 E  │
│ 작업 C   │          │          │  작업 F  │
└──────────┴──────────┴──────────┴──────────┘
```

### 칸반의 장점
- **직관적**: 누구나 쉽게 이해
- **유연함**: 언제든 우선순위 조정
- **투명성**: 누가 무엇을 하는지 명확
- **병목 발견**: 한 칸에 작업이 쌓이면 문제 파악

### 실무 활용
- **Trello**: 가장 단순한 칸반 도구
- **Jira Kanban**: 애자일 팀에서 사용
- **Asana**: 다양한 뷰 제공

## 3️⃣ 스프린트(Sprint)

### 정의
정해진 기간(보통 1-4주) 동안 특정 목표를 달성하는 **짧은 개발 주기**입니다.

### 실생활 비유
**학교 방학 계획**을 생각해보세요:

```
2주 방학 = 1개 스프린트
목표: 영어 실력 향상

주간 계획:
- 1주차: 단어 100개 암기, 문법 2개 챕터
- 2주차: 모의 시험 3회, 오답 정리

2주 후 점검: 얼마나 달성했나?
```

스프린트도 마찬가지로:
1. 목표 설정
2. 계획 수립
3. 실행
4. 회고

### 스프린트 구조
```
스프린트 1 (2주)
  ├─ Day 1: 스프린트 계획 회의
  ├─ Day 2-9: 개발 작업
  │   └─ 매일 데일리 스탠드업
  ├─ Day 10: 스프린트 리뷰
  └─ Day 10: 스프린트 회고
      ↓
스프린트 2 (2주) 시작
```

### 핵심 원칙
- **고정된 기간**: 연장하지 않음
- **완료 가능한 목표**: 무리하지 않게 설정
- **변경 금지**: 스프린트 중에는 새 작업 추가 안 함
- **검토와 개선**: 매 스프린트 후 회고

### PM이 알아야 하는 이유
- Jira 스크럼 프로젝트는 스프린트 기반
- 애자일 방법론의 핵심 개념
- 일정 관리를 짧은 주기로 반복

## 4️⃣ 이슈(Issue) & 티켓(Ticket)

### 정의
**이슈**는 프로젝트에서 처리해야 할 작업, 버그, 요청 등을 하나의 단위로 표현한 것입니다.

### 실생활 비유
**병원 접수증**을 생각해보세요:
- 접수번호 = 이슈 번호 (PROJ-123)
- 환자 이름 = 담당자
- 증상 = 이슈 설명
- 진료과 = 이슈 유형 (버그, 기능, 개선)
- 대기 시간 = 우선순위

### 이슈 유형

#### 1. **Epic (에픽)** - 큰 단위의 작업
- **비유**: 여행 전체 (유럽 여행)
- **예시**: "사용자 인증 기능 구현"
- **기간**: 수 주 ~ 수 개월

#### 2. **Story (스토리)** - 사용자 관점의 기능
- **비유**: 여행 중 한 도시 (파리 관광)
- **예시**: "사용자는 이메일로 로그인할 수 있다"
- **기간**: 수 일 ~ 1주

#### 3. **Task (태스크)** - 구체적인 작업
- **비유**: 여행 중 한 활동 (에펠탑 방문)
- **예시**: "로그인 UI 디자인 작성"
- **기간**: 수 시간 ~ 1일

#### 4. **Bug (버그)** - 오류 수정
- **비유**: 여행 중 문제 해결 (짐 분실 대응)
- **예시**: "로그인 버튼 클릭 시 오류 발생"
- **기간**: 상황에 따라 다름

### 이슈 관계
```
Epic: 온라인 쇼핑몰 구축
  ├─ Story: 상품 검색 기능
  │   ├─ Task: 검색 UI 디자인
  │   ├─ Task: 검색 API 개발
  │   └─ Task: 검색 결과 페이지
  └─ Story: 장바구니 기능
      ├─ Task: 장바구니 UI
      └─ Task: 장바구니 API
```

## 5️⃣ 간트 차트(Gantt Chart)

### 정의
프로젝트 일정을 **막대 형태**로 표현하여 시작일, 종료일, 기간, 의존 관계를 보여주는 차트입니다.

### 실생활 비유
**여행 일정표**를 생각해보세요:

```
날짜:    1일  2일  3일  4일  5일  6일  7일
항공편:  [==]
호텔:         [====================]
관광:              [====]    [====]
쇼핑:                   [==]
귀국:                             [==]
```

각 항목이 언제 시작하고 끝나는지 한눈에 보입니다!

### 간트 차트 예시
```
작업명         기간    |1주 |2주 |3주 |4주 |5주 |6주 |
기획          2주     [====]
디자인        2주          [====]
개발          3주               [======]
테스트        1주                    [==]
배포          1주                       [==]
```

### 간트 차트의 구성 요소
- **작업(Task)**: 각 행
- **기간(Duration)**: 막대 길이
- **의존 관계(Dependency)**: 화살표로 연결
- **마일스톤(Milestone)**: 중요 시점 (◆ 표시)
- **크리티컬 패스**: 빨간색으로 강조

### 간트 차트의 장점
- ✅ 전체 일정을 한눈에 파악
- ✅ 작업 간 의존 관계 명확
- ✅ 진척도 시각화
- ✅ 자원 배분 계획

### 도구
- **MS Project**: 가장 전문적인 간트 차트 도구
- **Excel**: 간단한 간트 차트 가능
- **Jira**: 로드맵 기능으로 간트 차트 제공
- **Monday.com**: 시각적 타임라인

## 6️⃣ 클라우드 기반 협업

### 정의
인터넷을 통해 여러 사람이 동시에 작업하고 정보를 공유하는 방식입니다.

### 실생활 비유
**Google 문서** vs **MS Word**:

| 방식 | 전통적(Word) | 클라우드(Google 문서) |
|------|--------------|----------------------|
| 저장 | 내 컴퓨터 | 클라우드 서버 |
| 협업 | 파일 이메일로 주고받기 | 링크 공유로 동시 작업 |
| 버전 | 파일명에 날짜 추가 | 자동 버전 관리 |
| 접근 | 해당 컴퓨터에서만 | 어디서나 인터넷으로 |

### 클라우드 PM 도구의 장점
- 🌐 **어디서나 접근**: 사무실, 집, 카페에서 동일하게 작업
- 👥 **실시간 협업**: 팀원이 동시에 업데이트
- 💾 **자동 저장**: 작업 내용 자동 백업
- 🔄 **동기화**: 모든 사람이 최신 정보 확인
- 📱 **다양한 기기**: PC, 태블릿, 스마트폰 모두 가능

### 대표 클라우드 PM 도구
- **Jira Cloud**: 애자일 프로젝트 관리
- **Trello**: 간단한 칸반 보드
- **Asana**: 작업 관리 및 협업
- **Monday.com**: 시각적 프로젝트 관리
- **Notion**: 문서화 및 작업 관리

### 전통적 도구 vs 클라우드 도구
| 특징 | 전통적(MS Project) | 클라우드(Jira, Trello) |
|------|-------------------|----------------------|
| 설치 | 개별 PC에 설치 | 웹 브라우저 접속 |
| 가격 | 라이선스 구매 | 월 구독료 |
| 업데이트 | 수동 설치 | 자동 업데이트 |
| 협업 | 파일 공유 | 실시간 동기화 |
| 접근성 | 설치된 PC만 | 인터넷 연결 시 어디서나 |

## 7️⃣ 통합(Integration)과 자동화(Automation)

### 정의
여러 도구를 연결하고 반복 작업을 자동으로 처리하는 것입니다.

### 실생활 비유
**스마트홈**을 생각해보세요:

```
문 열림 감지 → 자동으로 불 켜짐 → 에어컨 작동
```

PM 도구도 마찬가지로:
```
Jira 이슈 생성 → Slack 알림 → GitHub 이슈 자동 생성
```

### 통합 사례

#### 1. **Jira + Slack**
- 이슈 생성/업데이트 시 Slack에 자동 알림
- Slack에서 간단한 명령으로 Jira 이슈 생성

#### 2. **Jira + GitHub**
- 코드 커밋 시 Jira 이슈에 자동 연결
- Pull Request 정보 Jira에 표시

#### 3. **Jira + Confluence**
- Jira 이슈를 Confluence 문서에 삽입
- 프로젝트 현황을 문서로 자동 생성

#### 4. **MS Project + Teams**
- 프로젝트 업데이트를 Teams 채널에 공유
- Teams에서 작업 상태 확인

### 자동화 예시

#### **Jira Automation**
- **규칙**: 이슈가 "Done"으로 변경되면
- **동작**: 담당자에게 완료 알림 + 다음 이슈 자동 생성

#### **Trello Butler**
- **규칙**: 카드가 "Doing"으로 이동하면
- **동작**: 시작일 자동 기록 + 담당자에게 알림

### PM이 알아야 하는 이유
- ⏱️ 반복 작업 시간 절약
- 📧 수동 알림/보고 불필요
- 🔗 데이터 일관성 유지
- 🚀 생산성 향상

## 💬 자주 묻는 질문 (FAQ)

### Q1: 도구가 너무 많은데 어떤 걸 배워야 하나요?
**A**: Week 10에서는 **Jira**와 **MS Project** 중심으로 배웁니다. 두 도구를 이해하면 다른 도구도 쉽게 배울 수 있습니다. 실무에서는 조직의 도구를 따르면 되고, 개인 프로젝트는 **Trello**(간단함)나 **Jira Free**(무료)를 추천합니다.

### Q2: IT 지식이 없어도 도구를 사용할 수 있나요?
**A**: 네! 최신 PM 도구들은 **사용자 친화적**으로 설계되었습니다. 특히 Trello는 드래그 앤 드롭만으로 사용 가능하고, Jira도 기본 기능은 쉽습니다. Week 10에서 실습하면서 자연스럽게 익히게 됩니다.

### Q3: 칸반 보드와 스프린트 중 어느 것이 더 좋나요?
**A**: 상황에 따라 다릅니다:
- **칸반**: 지속적인 업무, 작은 팀, 유연한 일정
- **스프린트**: 정해진 목표, 큰 프로젝트, 정기 배포

애자일 팀은 주로 스프린트를 사용하지만, 유지보수 팀은 칸반을 선호합니다.

### Q4: 무료 도구와 유료 도구의 차이는?
**A**: 주요 차이점:
- **무료**: 기본 기능, 인원 제한, 저장 용량 제한
- **유료**: 고급 기능(자동화, 통합, 리포트), 무제한 인원, 우선 지원

소규모 팀이나 학습 목적이라면 **무료로 충분**합니다!

### Q5: MS Project는 배우기 어렵다는데 꼭 배워야 하나요?
**A**: MS Project는 **전통적 방법론**에서 표준이고, 건설/제조 등 업계에서 필수입니다. 학습 곡선이 있지만, Week 10에서 핵심 기능만 배웁니다. 대체 도구로는 **ProjectLibre**(무료) 또는 **Smartsheet**(웹 기반)이 있습니다.

### Q6: Jira와 Trello의 차이는 뭔가요?
**A**: 
- **Trello**: 단순 칸반 보드, 초보자 친화적, 가벼운 프로젝트
- **Jira**: 애자일 전문 도구, 복잡한 프로젝트, 개발팀 특화

**비유**: Trello = 메모장, Jira = 워드프로세서

### Q7: 도구를 바꾸면 데이터를 옮길 수 있나요?
**A**: 대부분 도구는 **내보내기/가져오기** 기능을 제공합니다:
- CSV, Excel 형식으로 내보내기
- 다른 도구에서 가져오기
- 일부 도구는 직접 마이그레이션 지원

단, 완벽하지 않으니 도구 선택 시 신중해야 합니다.

### Q8: 팀원들이 도구 사용을 꺼려하면 어떻게 하나요?
**A**: 
1. **간단한 것부터**: Trello처럼 쉬운 것부터 시작
2. **실익 강조**: "이메일 보고 대신 클릭 한 번"
3. **교육 제공**: 30분 튜토리얼로 충분
4. **서서히 도입**: 일부 프로젝트부터 파일럿

변화 관리도 PM의 중요한 역할입니다!
