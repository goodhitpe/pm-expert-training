# Week 11-12 사전 학습: 모의 프로젝트 준비

> **학습 시간**: 30분  
> **목적**: 2주간 모의 프로젝트를 성공적으로 수행하기 위해 필요한 준비 사항과 핵심 개념 총정리

---

## 📚 이 주차를 위한 준비

Week 11-12는 지난 10주간 배운 모든 내용을 **실전에 적용**하는 종합 실습입니다. 
이론을 실제 프로젝트에 어떻게 적용하는지 경험하고, 팀 협업과 문제 해결 능력을 키우는 중요한 시간입니다.

---

## 1️⃣ 모의 프로젝트란?

### 정의
**가상의 프로젝트를 실제처럼 수행**하여 PM 역량을 키우는 실습

### 왜 중요한가?
- 이론만 아는 것 ≠ 실전 적용 가능
- 실수할 수 있는 안전한 환경
- 팀 협업 경험
- 포트폴리오 제작

### 실생활 비유
```
🚗 운전 면허 시험:
- Week 1-10: 교통 법규, 운전 이론 학습 (필기시험)
- Week 11-12: 실제 도로 주행 (실기시험)

이론만 알아서는 운전 못 함!
실제로 운전해봐야 익숙해짐 ✅
```

### 2주간 흐름
```
Week 11 (계획):
- Day 1: 착수, 범위 정의
- Day 2: 일정, 예산 계획
- Day 3: 품질, 리스크, 자원 계획
- Day 4: 통합 및 킥오프 발표

Week 12 (실행/통제/종료):
- Day 5-6: 실행 시뮬레이션 (상황 카드 대응)
- Day 7: 성과 측정 및 통제
- Day 8: 종료 및 최종 발표
```

## 2️⃣ 프로젝트 수명주기 복습

### 5가지 프로세스 그룹

#### 1. 착수 (Initiating)
**"프로젝트 시작 선언"**

**주요 산출물:**
- 프로젝트 헌장
- 이해관계자 등록부

**실생활 비유:**
```
🎓 졸업 여행 계획:
- 여행 가기로 결정
- 참여자 확정
- 예산 대략 정하기
```

#### 2. 계획 (Planning)
**"상세 계획 수립"**

**주요 산출물:**
- WBS
- 간트 차트
- 예산 계획서
- 리스크 등록부
- RACI 매트릭스

**실생활 비유:**
```
🎓 졸업 여행 계획:
- 일정표 작성 (Day 1: 제주도, Day 2: 한라산...)
- 예산표 작성 (항공권 30만원, 숙박 20만원...)
- 역할 분담 (A: 숙박 예약, B: 교통, C: 음식...)
```

#### 3. 실행 (Executing)
**"계획대로 수행"**

**주요 활동:**
- 작업 수행
- 팀 관리
- 품질 보증
- 커뮤니케이션

**실생활 비유:**
```
🎓 졸업 여행:
- 실제 여행 진행
- 예약한 숙소 체크인
- 계획한 관광지 방문
```

#### 4. 감시 및 통제 (Monitoring & Controlling)
**"진행 상황 확인 및 조정"**

**주요 활동:**
- 진척도 측정
- 변경 관리
- 이슈 해결
- 리스크 모니터링

**실생활 비유:**
```
🎓 졸업 여행 중:
- 예산 사용 현황 체크
- 비 오면 실내 활동으로 변경
- 교통 지연 시 일정 조정
```

#### 5. 종료 (Closing)
**"프로젝트 마무리"**

**주요 활동:**
- 최종 인도물 확인
- 교훈 정리
- 종료 보고서 작성
- 축하 🎉

**실생활 비유:**
```
🎓 졸업 여행 후:
- 사진 정리
- 비용 정산
- 회고 ("제주도 좋았어", "다음엔 더 여유롭게")
```

## 4️⃣ 성공적인 팀워크 원칙

### 명확한 역할 분담
**누가 무엇을 하는지 명확히**

**예시:**
```
팀 구성 (5명):
- PM: 전체 조율, 일정 관리
- 범위/일정 담당: WBS, 간트 차트
- 원가/품질 담당: 예산, 품질 계획
- 리스크 담당: 리스크 등록부
- 자원/커뮤니케이션 담당: RACI, 보고서
```

### 정기적인 소통
**매일 짧게, 주기적으로**

**일일 스탠드업 미팅 (10분):**
```
각자 3가지 공유:
1. 어제 한 일
2. 오늘 할 일
3. 어려운 점 (도움 필요)
```

### 문서화 습관
**모든 결정과 변경 사항을 기록**

**왜 중요한가:**
- 나중에 "누가 그렇게 결정했어?"를 방지
- 평가 시 근거 자료
- 다음 프로젝트에 참고

### 긍정적인 분위기
**서로 격려하고 존중**

**❌ 피해야 할 행동:**
- "내 생각이 맞아, 네가 틀렸어"
- "이건 내 파트가 아니에요"
- "시간 없으니 대충 하자"

**✅ 해야 할 행동:**
- "좋은 의견이네요, 이렇게 보완하면 어떨까요?"
- "제가 도와드릴게요"
- "조금 더 시간을 들여 제대로 합시다"

## 5️⃣ 상황 카드 대응 전략

### 상황 카드란?
강사가 **예상치 못한 상황을 부여**하여 대응 능력 테스트

### 예시 상황들

#### 상황 1: 인력 이슈
**"핵심 개발자가 갑자기 3일 병가"**

**대응 방법:**
1. 영향도 분석 (크리티컬 패스?)
2. 대안 검토 (다른 개발자 투입? 일정 조정?)
3. 변경 요청서 작성
4. 이해관계자에게 통보

#### 상황 2: 범위 변경
**"고객이 새로운 기능 추가 요청"**

**대응 방법:**
1. 요구사항 명확화
2. 영향도 분석 (시간, 비용)
3. MoSCoW로 우선순위 평가
4. 고객과 협상 (추가 비용/시간 or 다음 버전)

#### 상황 3: 예산 삭감
**"예산 10% 삭감 통보"**

**대응 방법:**
1. 필수 항목 vs 선택 항목 구분
2. 범위 축소 방안 검토
3. 자원 재배치
4. 수정된 계획 수립

#### 상황 4: 외주 지연
**"외주 업체가 납기를 1주 지연"**

**대응 방법:**
1. 크리티컬 패스 확인
2. 일정 압축 검토 (크래싱? 패스트 트래킹?)
3. 병렬 작업 가능한 항목 찾기
4. 대체 업체 검토

### 핵심 원칙
1. **당황하지 말기** - 팀이 함께 해결
2. **체계적 분석** - 영향도 먼저 파악
3. **문서화** - 모든 결정 기록
4. **이해관계자 소통** - 투명하게 공유

**구성 (20분):**

## 7️⃣ 자주 하는 실수 & 예방법

### 실수 1: 계획에만 시간 소비
**문제:** Week 11 전체를 계획에만 쓰고, 실행 시간 부족

**예방:**
- Day 4까지 계획 완료 (시간 엄수)
- 80% 완성도면 충분 (100% 완벽 추구 금지)

### 실수 2: 문서화 소홀
**문제:** 결정 사항을 기록 안 해서 나중에 혼란

**예방:**
- 회의록 필수 작성
- 변경 사항 즉시 기록
- 문서 담당자 지정

### 실수 3: 팀원 간 소통 부족
**문제:** 각자 작업만 하고, 통합 안 됨

**예방:**
- 매일 스탠드업 미팅
- 공유 문서 사용 (Google Docs)
- 진행 상황 시각화 (칸반 보드)

### 실수 4: PM이 모든 일 혼자
**문제:** PM이 과부하, 팀원은 할 일 없음

**예방:**
- 명확한 역할 분담 (RACI)
- 권한 위임
- PM은 조율자, 실무자 아님

### 실수 5: 리스크 관리 무시
**문제:** 상황 카드에 당황, 대응 못 함

**예방:**
- 리스크 등록부에 상황 카드 예상 시나리오 미리 작성
- 우발 계획 준비
- 정기적 리스크 검토

## 💬 자주 묻는 질문 (FAQ)

**Q1: 프로젝트 시나리오를 직접 만들어도 되나요?**  
A: 네! 단, 강사 승인이 필요합니다. 너무 간단하거나 복잡하지 않은 적절한 수준이어야 합니다.

**Q2: PM은 누가 하나요?**  
A: 팀 내에서 자발적으로 정하거나, 돌아가면서 할 수 있습니다. PM 경험이 중요하니 가능하면 해보세요!

**Q3: 실제로 개발이나 작업을 하나요?**  
A: 아닙니다! "관리" 문서만 작성합니다. 실제 개발은 시뮬레이션으로 대체합니다.

**Q4: 평가 비중은?**  
A: 산출물(40%) + 팀워크/프로세스(30%) + 최종 발표(30%). 완벽한 산출물보다 프로세스와 협업이 더 중요합니다.

**Q5: Week 12에 정말 위기 상황이 오나요?**  
A: 네! 상황 카드는 반드시 제공됩니다. 당황하지 말고, 배운 대로 체계적으로 대응하세요. 이것이 진짜 배움입니다!
