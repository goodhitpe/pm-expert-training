## 👥 Part 2: 팀 구성 및 역할

### 2.1 팀 구성 원칙

#### 적정 팀 규모
- **권장**: 4-5명
- **최소**: 3명
- **최대**: 6명

#### 팀 구성 방법

**방법 1: 자유 구성**
- 장점: 친한 사람들과 편안한 협업
- 단점: 다양성 부족, 친분 관계로 갈등 회피

**방법 2: 강사 배정**
- 장점: 다양한 배경 믹스
- 단점: 초기 어색함

**방법 3: 역할 기반 매칭**
- 장점: 균형 잡힌 팀
- 단점: 선호 역할 경쟁

**추천**: 방법 3 (역할 기반) + 방법 1 (일부 자유)

### 2.2 팀 역할 분담

#### 필수 역할

**1. 프로젝트 매니저 (PM)**
```
책임:
- 전체 프로젝트 총괄
- 계획 수립 및 진행 관리
- 이해관계자 커뮤니케이션
- 리스크 관리
- 최종 발표 진행

필요 역량:
- 리더십
- 커뮤니케이션
- 문제 해결
- 시간 관리
```

**2. 기획자 / 분석가 (BA)**
```
책임:
- 요구사항 수집 및 분석
- 사용자 스토리 작성
- 프로세스 설계
- 문서화

필요 역량:
- 분석력
- 문서 작성
- 논리적 사고
```

**3. 개발 리드 / 기술 리드**
```
책임:
- 기술 아키텍처 설계
- 기술 스택 선정
- 개발 방법론 제안
- 기술 리스크 파악

필요 역량:
- 기술 지식
- 설계 능력
- 문제 해결
```

**4. 품질 관리자 (QA)**
```
책임:
- 품질 계획 수립
- 테스트 전략
- 품질 기준 정의
- 리뷰 및 검증

필요 역량:
- 꼼꼼함
- 분석력
- 비판적 사고
```

**5. 운영 / 지원**
```
책임:
- 회의록 작성
- 문서 정리
- 일정 추적
- 커뮤니케이션 지원

필요 역량:
- 조직력
- 문서 작성
- 세심함
```

### 2.3 팀 빌딩 활동

#### 아이스브레이킹 (15분)
```
활동: "Two Truths and a Lie"
- 각자 자신에 대한 3가지 진술 (2개 진실, 1개 거짓)
- 팀원들이 거짓 찾기
- 자연스러운 자기소개
```

#### 팀 규범 수립 (30분)
```
토론 주제:
1. 회의 시간 및 빈도
2. 커뮤니케이션 방법 (Slack, 카톡 등)
3. 응답 시간 기대치 (24시간 내 등)
4. 의사결정 방법 (합의 vs 다수결)
5. 갈등 해결 방법
6. 작업 분담 원칙

문서화: 팀 규범 문서 작성
```

#### RACI 차트 작성 (20분)
```
주요 활동별 역할 명확화:
- 프로젝트 헌장 작성
- WBS 개발
- 일정 계획
- 예산 산정
- 리스크 관리
- 최종 발표
```

## 📊 Part 5: 평가 기준

### 5.1 평가 항목 및 배점

| 평가 항목 | 배점 | 세부 내용 |
|---------|------|----------|
| **프로젝트 산출물** | 40점 | 헌장, WBS, 일정, 예산, 리스크 |
| **프로세스 준수** | 30점 | PM 방법론 적용, 도구 활용 |
| **팀워크 및 협업** | 20점 | 역할 분담, 의사소통, 협력 |
| **발표 품질** | 10점 | 논리성, 명확성, 시간 준수 |
| **합계** | 100점 | |

### 5.2 상세 평가 루브릭

#### 프로젝트 산출물 (40점)

**프로젝트 헌장 (10점)**:
```
탁월 (9-10): 모든 섹션 완벽, SMART 목표 명확
우수 (7-8): 대부분 완성, 일부 개선 필요
보통 (5-6): 기본 내용 포함, 세부 부족
미흡 (0-4): 불완전하거나 부정확
```

**WBS 및 일정 (10점)**:
```
탁월 (9-10): 3-4레벨 상세 분해, 주공정 식별
우수 (7-8): 2-3레벨 분해, 일정 적절
보통 (5-6): 기본 WBS, 일정 있음
미흡 (0-4): 불완전하거나 비현실적
```

**예산 및 자원 (10점)**:
```
탁월 (9-10): 상세 항목별 산정, RACI 명확
우수 (7-8): 주요 항목 산정, 자원 배정
보통 (5-6): 대략적 예산, 기본 자원
미흡 (0-4): 근거 없거나 비현실적
```

**리스크 관리 (10점)**:
```
탁월 (9-10): 5개+ 리스크, 대응 전략 구체적
우수 (7-8): 3-4개 리스크, 기본 대응
보통 (5-6): 2-3개 리스크, 간단한 대응
미흡 (0-4): 불충분하거나 비현실적
```

#### 프로세스 준수 (30점)

**PM 방법론 적용 (15점)**:
```
- PMBOK 프로세스 준수
- 문서 템플릿 활용
- 용어 정확성
- 논리적 흐름
```

**PM 도구 활용 (15점)**:
```
- Jira/GitHub/Trello 사용
- 이슈 관리
- 칸반 보드 운영
- 자동화 설정
```

#### 팀워크 및 협업 (20점)

**역할 분담 (10점)**:
```
- RACI 차트 작성 및 준수
- 모든 팀원 기여
- 역할 명확성
```

**협업 및 커뮤니케이션 (10점)**:
```
- 회의록 작성
- 진행 상황 공유
- 갈등 해결
- 팀 규범 준수
```

#### 발표 품질 (10점)

**내용 (5점)**:
```
- 논리적 구성
- 핵심 전달
- 시간 준수
```

**전달 (5점)**:
```
- 명확한 발음
- 적절한 속도
- 시선 처리
- 질의 응답
```
