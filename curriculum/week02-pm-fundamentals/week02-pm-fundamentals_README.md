## 📚 주요 내용

### 1. 5대 프로세스 그룹

#### 1.1 착수 프로세스 그룹 (Initiating)
- 프로젝트 헌장 개발
- 이해관계자 식별
- 프로젝트 목표 및 범위 정의

#### 1.2 기획 프로세스 그룹 (Planning)
- 프로젝트 관리 계획서 개발
- 요구사항 수집
- 범위, 일정, 예산 상세 계획
- 리스크 관리 계획

#### 1.3 실행 프로세스 그룹 (Executing)
- 프로젝트 작업 지휘 및 관리
- 프로젝트 팀 관리
- 품질 관리
- 커뮤니케이션 관리

#### 1.4 감시 및 통제 프로세스 그룹 (Monitoring & Controlling)
- 프로젝트 작업 감시 및 통제
- 통합 변경 통제 수행
- 범위, 일정, 원가 통제
- 품질 통제

#### 1.5 종료 프로세스 그룹 (Closing)
- 프로젝트 종료 또는 단계 종료
- 교훈 정리
- 최종 보고서 작성

### 2. 10대 지식영역

#### 2.1 통합 관리 (Integration Management)
- 프로젝트의 다양한 요소들을 조율
- 프로젝트 헌장 개발
- 프로젝트 관리 계획서 개발
- 변경 통제

#### 2.2 범위 관리 (Scope Management)
- 요구사항 수집 및 정의
- WBS(작업 분해 구조) 작성
- 범위 검증 및 통제

#### 2.3 일정 관리 (Schedule Management)
- 활동 정의 및 순서 배열
- 소요 기간 산정
- 일정 개발 및 통제

#### 2.4 원가 관리 (Cost Management)
- 원가 산정
- 예산 결정
- 원가 통제 (EVM)

#### 2.5 품질 관리 (Quality Management)
- 품질 계획 수립
- 품질 보증 수행
- 품질 통제

#### 2.6 자원 관리 (Resource Management)
- 팀 구성 및 관리
- 자원 확보 및 배분
- 팀 개발 및 관리

#### 2.7 의사소통 관리 (Communications Management)
- 커뮤니케이션 계획
- 정보 배포
- 보고서 작성 및 공유

#### 2.8 리스크 관리 (Risk Management)
- 리스크 식별
- 리스크 분석 (정성적/정량적)
- 리스크 대응 계획
- 리스크 감시

#### 2.9 조달 관리 (Procurement Management)
- 조달 계획
- 공급업체 선정
- 계약 관리

#### 2.10 이해관계자 관리 (Stakeholder Management)
- 이해관계자 식별 및 분석
- 이해관계자 참여 관리
- 이해관계자 기대 관리

### 3. 프로세스 그룹과 지식영역의 관계

프로세스 그룹과 지식영역은 매트릭스 구조로 연결됩니다:
- 각 지식영역은 여러 프로세스 그룹에 걸쳐 프로세스를 포함
- 총 49개의 프로세스가 존재 (PMBOK 6th Edition 기준)

## 🎯 실무 적용 예시: IT 프로젝트에서의 PMBOK 활용

### 예시 프로젝트: 모바일 뱅킹 앱 개발

#### 1. 5대 프로세스 그룹 적용

**착수 (Initiating):**
```
프로젝트 헌장:
- 목표: 고객 만족도 30% 향상
- 예산: $500,000
- 기간: 6개월
- 주요 기능: 계좌 조회, 이체, 대출 신청

기술적 고려사항 (IT 지식 연계):
- 보안: 금융 보안 규정 준수 (전자금융감독규정)
- 인프라: AWS 클라우드 사용
- 플랫폼: iOS, Android 동시 개발
```

**기획 (Planning):**
```
범위 관리:
- WBS 작성 시 SDLC 단계 반영
  └─ 1.0 요구사항 분석 (2주)
  └─ 2.0 아키텍처 설계 (2주)
  └─ 3.0 개발 (12주)
      └─ 3.1 백엔드 API (6주)
      └─ 3.2 iOS 앱 (6주)
      └─ 3.3 Android 앱 (6주)
  └─ 4.0 테스트 (4주)
  └─ 5.0 배포 및 전환 (2주)

일정 관리:
- 크리티컬 패스: 보안 인증 획득 (3주 소요)
- 기술 의존성: API 완료 후 앱 개발 가능

비용 관리:
- 개발 비용: $300,000
- 클라우드 비용: $50,000 (6개월)
- 보안 인증: $30,000
- 예비비: $120,000 (24%)

리스크 관리:
- 기술 리스크: 생체 인증 API 호환성 이슈
  → 대응: 2개 백업 솔루션 준비
- 보안 리스크: 해킹 취약점
  → 대응: 외부 보안 전문가 침투 테스트
```

**실행 (Executing):**
```
팀 관리:
- 백엔드: 3명 (Java Spring)
- 프론트엔드: 4명 (React Native)
- QA: 2명
- DevOps: 1명
- 보안 전문가: 1명 (컨설턴트)

품질 보증:
- 코드 리뷰 100%
- 자동화 테스트 커버리지 80% 이상
- 보안 스캔 주 1회

커뮤니케이션:
- 일일 스탠드업 (개발팀)
- 주간 상태 보고 (경영진)
- 격주 스프린트 리뷰 (이해관계자)
```

**감시 및 통제 (Monitoring & Controlling):**
```
진척도 추적:
- Jira 번다운 차트로 스프린트 진행 상황 모니터링
- EVM 지표:
  * CPI (비용 성과 지수) = 1.05 (5% 절감)
  * SPI (일정 성과 지수) = 0.95 (5% 지연)

변경 관리:
- 변경 요청: "Apple Pay 통합 추가"
  → 영향도 분석:
     - 기간: +2주
     - 비용: +$20,000
     - 리스크: Apple 승인 프로세스 3주 소요
  → 결정: Phase 2로 이연

품질 통제:
- 보안 취약점 발견: SQL Injection 1건
  → 즉시 수정, 전체 코드 재검토
```

**종료 (Closing):**
```
최종 인도물:
- iOS/Android 앱 (App Store, Play Store 출시)
- API 문서
- 운영 매뉴얼
- 소스 코드 (GitHub)

교훈 정리:
✅ 잘한 점:
- CI/CD 파이프라인 초기 구축으로 배포 시간 단축
- 보안 전문가 조기 참여로 취약점 사전 예방

❌ 개선 필요:
- 생체 인증 호환성 테스트 부족 (1주 지연 원인)
- 클라우드 비용 모니터링 미흡 ($5,000 초과)

프로젝트 성과:
- 고객 만족도 35% 향상 (목표 초과)
- 일일 활성 사용자 10만 명
- 앱 스토어 평점 4.7/5.0
```

#### 2. 10대 지식영역의 기술 연계

| 지식영역 | PMBOK 프로세스 | IT 기술 연계 | 실무 적용 |
|---------|--------------|------------|----------|
| **범위 관리** | WBS 작성 | 소프트웨어 공학 (SDLC) | 개발 단계별 작업 분해 |
| **일정 관리** | 일정 개발 | Git 브랜치 전략 | 기능별 병렬 개발 계획 |
| **비용 관리** | 예산 결정 | 클라우드 요금 체계 | AWS 비용 계산기 활용 |
| **품질 관리** | 품질 보증 | 테스트 자동화 | Jest, Selenium 도입 |
| **자원 관리** | 팀 개발 | DevOps 문화 | Cross-functional 팀 구성 |
| **의사소통 관리** | 정보 배포 | Confluence, Slack | 실시간 협업 도구 |
| **리스크 관리** | 리스크 대응 | 보안 취약점 스캔 | SonarQube, OWASP ZAP |
| **조달 관리** | 공급업체 선정 | SaaS 선택 | AWS vs Azure 비교 평가 |
| **이해관계자 관리** | 참여 관리 | 애자일 데모 | 2주마다 작동하는 SW 시연 |
| **통합 관리** | 변경 통제 | CI/CD 파이프라인 | Jenkins 자동 빌드/배포 |

### 실전 시나리오: 변경 요청 처리

**상황:**
개발 중반에 고객이 "얼굴 인식 로그인 추가" 요청

**❌ PMBOK 지식만 있는 PM:**
```
PM: "변경 요청서 작성하세요."
(양식만 작성, 기술적 분석 없음)

결과:
- 영향도 파악 안 됨
- 일정/비용 추정 부정확
- 팀 혼란
```

**✅ PMBOK + IT 기술 이해하는 PM:**
```
PM의 분석:

1. 통합 관리 (변경 영향도 분석)
   - 현재: ID/PW + OTP 인증
   - 추가: 얼굴 인식 (Face ID/Face Recognition)

2. 범위 관리 (기술 실현 가능성)
   - iOS: Face ID API 사용 (네이티브)
   - Android: ML Kit 사용 (Google)
   - 백엔드: 생체 데이터 저장 불필요 (디바이스 내)
   - 보안: 생체 정보 보호법 준수

3. 일정 관리 (개발 소요 기간)
   - iOS 구현: 3일 (Face ID API 간단)
   - Android 구현: 5일 (ML Kit 통합)
   - 테스트: 2일 (다양한 디바이스)
   - 보안 검토: 1일
   총 11일 (버퍼 포함 14일 = 2주)

4. 비용 관리
   - 개발 비용: $8,000 (2주 × 2명)
   - 추가 라이선스: $0 (SDK 무료)
   - 보안 검토: $2,000
   총 $10,000

5. 품질 관리
   - 정확도 테스트: 다양한 조명 환경
   - 보안 테스트: 사진 공격 방어
   - 호환성: iPhone X 이상, Android 8.0 이상

6. 리스크 관리
   - 리스크 1: 오래된 디바이스 미지원
     → 대응: 기존 인증 방식 유지 (옵션)
   - 리스크 2: 인식률 저하
     → 대응: 3회 실패 시 대체 인증

7. 이해관계자 관리
   - 고객: 편의성 증가
   - 개발팀: 2주 추가 작업
   - 보안팀: 검토 필요
   - 법무팀: 개인정보 처리 방침 업데이트

변경 통제 위원회(CCB) 제안:
- 영향도: 중간 (기능 추가, 기존 기능 유지)
- 이득: 사용자 경험 향상
- 비용: $10,000 / 2주 지연
- 권고: 승인 (Phase 2 대신 현재 포함)
```

## 💡 실습 과제

### 과제 1: 프로세스 매핑 (실무 사례)
다음 프로젝트 사례를 선정하고, 5대 프로세스 그룹에 따른 주요 활동을 **기술적 고려사항 포함**하여 나열하시오:
- 온라인 쇼핑몰 구축
- 사내 업무 시스템 개발
- 클라우드 마이그레이션 프로젝트

### 과제 2: 지식영역 × IT 기술 매트릭스
10대 지식영역 중 3가지를 선택하고, 각각에 연계되는 IT 기술과 실무 적용 예시를 작성하시오.

**예시:**
```
지식영역: 비용 관리
IT 기술: AWS Cost Explorer
실무 적용: 월별 클라우드 비용을 Cost Explorer로 모니터링하고,
           예산 초과 시 알람 설정하여 EVM의 AC(실제 비용) 추적
```

## 📝 핵심 용어
- **PMBOK (Project Management Body of Knowledge)**: PMI의 프로젝트 관리 표준
- **프로세스 그룹 (Process Group)**: 프로젝트 생명주기의 논리적 단계
- **지식영역 (Knowledge Area)**: 프로젝트 관리의 전문 분야
- **WBS (Work Breakdown Structure)**: 작업 분해 구조
- **EVM (Earned Value Management)**: 성과 측정 기법
