## 1️⃣ 원가 (Cost)와 예산 (Budget)

### 원가란?
**원가**는 어떤 것을 만들거나 서비스를 제공하기 위해 소비된 자원의 금전적 가치입니다.

### 예산이란?
**예산**은 특정 기간 동안 사용할 수 있는 돈의 계획입니다.

### 실생활 비유
```
집을 리모델링한다고 가정:
- 원가: 실제로 지출한 돈 (재료비 300만원, 인건비 200만원 = 총 500만원)
- 예산: 계획한 돈 (최대 600만원까지 쓰기로 계획)
```

### PM 관점
- **원가**: 프로젝트 수행에 실제 사용된 비용
- **예산**: 프로젝트에 할당된 총 금액
- **원가 관리 목표**: 예산 범위 내에서 프로젝트 완료

---

## 2️⃣ 직접 원가 vs 간접 원가

### 직접 원가 (Direct Cost)
프로젝트에 **직접 귀속**되는 비용

**예시**:
- 프로젝트 전담 인력 급여
- 프로젝트 전용 장비
- 프로젝트 전용 소프트웨어 라이선스

### 간접 원가 (Indirect Cost)
여러 프로젝트에 **공통으로 발생**하여 배분되는 비용

**예시**:
- 사무실 임대료
- 전기, 인터넷 등 공과금
- 관리 부서(인사, 재무) 비용

### 쉬운 비유
```
회사 식당에서 점심 식사:
- 직접 원가: 내가 먹은 음식 재료비
- 간접 원가: 식당 운영비(인건비, 임대료 등)를 전체 직원에게 나눈 금액
```

---

## 3️⃣ 고정 원가 vs 변동 원가

### 고정 원가 (Fixed Cost)
작업량과 **무관**하게 일정한 비용

**예시**:
- 월 고정 계약금
- 장비 리스 비용
- 소프트웨어 연간 라이선스

### 변동 원가 (Variable Cost)
작업량에 **비례**하여 변동하는 비용

**예시**:
- 시간당 인건비
- 사용량 기반 클라우드 비용
- 소모품 비용

### 실제 예시
```
웹 서비스 운영:
- 고정 원가: 서버 월 임대료 100만원 (사용자 수와 무관)
- 변동 원가: 데이터 전송량 비용 (사용자 많을수록 증가)
```

## 4️⃣ EVM 기초 개념

### EVM이란?
**EVM (Earned Value Management, 획득가치관리)**는 프로젝트의 진행 상황과 비용을 통합적으로 측정하는 방법입니다.

### 왜 필요한가?
단순히 "50% 완료"라고 말하는 것보다, "계획 대비 얼마나 진행되었고, 비용은 얼마나 사용했는지"를 정량적으로 파악할 수 있습니다.

### 3가지 핵심 지표

#### PV (Planned Value, 계획 가치)
- **의미**: 이 시점까지 **계획상** 완료했어야 하는 작업의 가치
- **비유**: 계획표에 "오늘까지 100페이지 읽기"

#### EV (Earned Value, 획득 가치)
- **의미**: 이 시점까지 **실제로** 완료한 작업의 가치
- **비유**: 실제로 읽은 페이지 "80페이지"

#### AC (Actual Cost, 실제 원가)
- **의미**: 이 시점까지 **실제로** 사용한 비용
- **비유**: 실제로 쓴 돈 "120만원"

### 쉬운 예시
```
프로젝트 예산: 1,000만원
프로젝트 기간: 10주

Week 5 종료 시점:
- PV (계획 가치): 500만원 (50% 완료 계획)
- EV (획득 가치): 400만원 (실제 40% 완료)
- AC (실제 원가): 450만원 (실제 사용 금액)

해석:
- 계획보다 작업이 뒤처짐 (EV < PV)
- 완료한 작업 대비 비용이 더 들어감 (AC > EV)
```

---

## 5️⃣ 차이 분석 기초

### CV (Cost Variance, 원가 차이)
- **공식**: CV = EV - AC
- **의미**: 
  - CV > 0: 예산보다 적게 씀 (좋음 ✅)
  - CV < 0: 예산 초과 (나쁨 ⚠️)

### SV (Schedule Variance, 일정 차이)
- **공식**: SV = EV - PV
- **의미**:
  - SV > 0: 일정보다 빠름 (좋음 ✅)
  - SV < 0: 일정 지연 (나쁨 ⚠️)

### 예시
```
위의 Week 5 예시 계속:

CV = EV - AC = 400만원 - 450만원 = -50만원
→ 예산을 50만원 초과 사용 ⚠️

SV = EV - PV = 400만원 - 500만원 = -100만원
→ 계획보다 100만원어치 작업이 뒤처짐 ⚠️
```

---

## 6️⃣ 성과 지수

### CPI (Cost Performance Index, 원가 성과 지수)
- **공식**: CPI = EV / AC
- **의미**: 투입한 비용 대비 얼마나 효율적으로 작업했는가?
- **해석**:
  - CPI = 1.0: 계획대로 (정상)
  - CPI > 1.0: 예산보다 효율적 (좋음 ✅)
  - CPI < 1.0: 예산 비효율적 (나쁨 ⚠️)

### SPI (Schedule Performance Index, 일정 성과 지수)
- **공식**: SPI = EV / PV
- **의미**: 계획 대비 얼마나 빠르게 진행되고 있는가?
- **해석**:
  - SPI = 1.0: 계획대로 (정상)
  - SPI > 1.0: 계획보다 빠름 (좋음 ✅)
  - SPI < 1.0: 계획보다 느림 (나쁨 ⚠️)

### 예시
```
위의 Week 5 예시 계속:

CPI = EV / AC = 400만원 / 450만원 = 0.89
→ 1원 쓸 때마다 0.89원어치 작업 (비효율적) ⚠️

SPI = EV / PV = 400만원 / 500만원 = 0.80
→ 계획의 80%만 완료 (지연) ⚠️
```

## 7️⃣ 회계 vs 재무

많은 분들이 혼동하는 개념입니다.

### 회계 (Accounting)
- **초점**: 과거 (이미 발생한 거래 기록)
- **목적**: 정확한 재무 상태 파악
- **예시**: 지난달 지출 내역 정리

### 재무 (Finance)
- **초점**: 미래 (돈을 어떻게 조달하고 사용할지)
- **목적**: 자금 운용 및 투자 결정
- **예시**: 내년 프로젝트 예산 계획

### PM 관점
- PM은 주로 **재무** 관점 (미래 예산 계획 및 통제)
- 하지만 **회계** 용어를 알아야 재무팀과 소통 가능

---

## 8️⃣ ROI (Return on Investment, 투자 수익률)

### 정의
투자한 돈 대비 얼마나 수익을 얻었는가?

### 공식
```
ROI = (수익 - 투자) / 투자 × 100%
```

### 예시
```
프로젝트 투자: 5,000만원
프로젝트로 인한 수익: 8,000만원

ROI = (8,000만원 - 5,000만원) / 5,000만원 × 100%
    = 3,000만원 / 5,000만원 × 100%
    = 60%
```

### PM 관점
- 프로젝트 정당성 확보에 중요
- 경영진에게 프로젝트 승인 받을 때 사용
- ROI가 높을수록 프로젝트 우선순위 높아짐

---

## 9️⃣ 감가상각 (Depreciation)

### 정의
자산의 가치가 시간이 지남에 따라 감소하는 것

### 왜 중요한가?
장비나 소프트웨어를 구매할 때 한 번에 비용 처리하지 않고, 사용 기간에 걸쳐 나누어 처리합니다.

### 예시
```
서버 구매: 6,000만원 (5년 사용 예정)

감가상각 방법 (정액법):
- 연간 감가상각비: 6,000만원 / 5년 = 1,200만원
- 프로젝트가 1년 사용: 1,200만원만 원가로 계산
```

### PM 관점
- 장비 구매 vs 임대 비교 시 고려
- 프로젝트 원가 계산 시 감가상각비 포함

## 🔟 주요 회계 용어 정리

| 용어 | 영문 | 의미 | PM 관점 |
|-----|------|------|---------|
| **원가** | Cost | 자원 소비의 금전적 가치 | 프로젝트 실제 지출 |
| **예산** | Budget | 계획된 금액 | 프로젝트 할당 금액 |
| **수익** | Revenue | 벌어들인 돈 | 프로젝트로 인한 이익 |
| **이익** | Profit | 수익 - 원가 | 프로젝트 성과 지표 |
| **현금흐름** | Cash Flow | 돈의 입출금 흐름 | 프로젝트 자금 관리 |
| **손익분기점** | Break-Even Point | 수익 = 원가인 시점 | 투자 회수 시기 |

### 자주 묻는 질문 (FAQ)

**Q1: 회계를 전혀 몰라도 괜찮을까요?**  
A: 네! Week 6는 PM 관점에서 설명합니다. 전문 회계사 수준은 필요 없고, 기본 개념만 이해하면 됩니다.

**Q2: EVM이 왜 필요한가요? 진행률만 보면 안 되나요?**  
A: 진행률만으로는 예산 효율성을 알 수 없습니다. EVM을 사용하면 "계획보다 지연되었는지", "예산을 효율적으로 쓰고 있는지"를 정량적으로 파악할 수 있습니다.

**Q3: 수학을 못하는데 계산이 어려울까요?**  
A: 기본적인 사칙연산만 할 수 있으면 됩니다. 복잡한 계산은 Excel이나 PM 도구가 자동으로 해줍니다.
