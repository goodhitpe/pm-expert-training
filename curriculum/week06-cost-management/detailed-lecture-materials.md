## 1. 원가 관리 (Cost Management)

### 용어
**원가 관리 (Cost Management)**

### 설명
프로젝트를 승인된 예산 내에서 완료하기 위해 원가를 계획, 산정, 예산화, 자금 조달, 관리 및 통제하는 프로세스입니다.

### 세부 요소 및 설명

#### 1.1 원가 관리의 4가지 프로세스
- **원가 관리 계획 수립**: 원가를 관리하는 방법 정의
- **원가 산정**: 프로젝트 활동 완료에 필요한 비용 예측
- **예산 결정**: 산정된 원가를 시간별로 배분하여 원가 베이스라인 수립
- **원가 통제**: 프로젝트 원가 상태 모니터링 및 변경 관리

#### 1.2 원가 관리의 중요성
- 프로젝트 성공의 3대 제약조건 중 하나 (범위, 일정, 원가)
- 조직의 수익성 직접 영향
- 이해관계자 신뢰 확보
- 자원 효율성 극대화

#### 1.3 원가 관리 계획서 (Cost Management Plan)
원가를 관리하는 방법을 정의하는 문서:
- **측정 단위**: 시간, 인원, 금액 단위
- **정확도 수준**: ±10%, ±5% 등
- **정밀도 수준**: 반올림 기준 (천 원, 만 원 단위)
- **통제 임계치**: 허용 가능한 편차 범위 (±10%)
- **성과 측정 규칙**: EVM 적용 방법
- **보고 형식 및 빈도**: 월간, 주간 보고

### 추가 설명

원가 관리 성공 요인:
1. **정확한 초기 산정**: 과거 데이터 기반 추정
2. **예비비 확보**: 불확실성 대비 (10-20%)
3. **지속적 모니터링**: 주간/월간 원가 리뷰
4. **조기 경보**: 편차 발생 즉시 대응
5. **변경 통제**: 범위 변경 시 예산 조정

원가 관리 실패의 주요 원인:
- 불완전한 범위 정의 (40%)
- 낙관적 추정 (25%)
- 숨겨진 비용 간과 (20%)
- 변경 관리 부재 (15%)

### 별첨

#### 원가 관리 베스트 프랙티스
1. **상향식 산정**: 세부 활동별 비용 산정
2. **예비비 분리 관리**: 우발 예비비, 관리 예비비 구분
3. **비용 편익 분석**: ROI, NPV, IRR 계산
4. **정기 리뷰**: 월간 원가 회의
5. **추세 분석**: EAC, VAC 산출 및 예측

## 2. EVM (Earned Value Management)

### 용어
**EVM (Earned Value Management, 성과 가치 관리)**

### 설명
프로젝트 범위, 일정, 원가를 통합하여 프로젝트 성과와 진척도를 객관적으로 측정하는 기법입니다.

### 세부 요소 및 설명

#### 2.1 EVM의 3대 핵심 지표

**2.1.1 PV (Planned Value, 계획 가치)**
- **정의**: 특정 시점까지 완료할 것으로 계획된 작업의 예산
- **별칭**: BCWS (Budgeted Cost of Work Scheduled)
- **계산**: 계획된 작업량 × 예산 단가
- **의미**: "얼마의 작업을 할 계획인가?"

**2.1.2 EV (Earned Value, 획득 가치)**
- **정의**: 특정 시점까지 실제 완료된 작업의 예산 가치
- **별칭**: BCWP (Budgeted Cost of Work Performed)
- **계산**: 실제 완료된 작업량 × 예산 단가
- **의미**: "얼마의 작업을 실제로 했는가?"

**2.1.3 AC (Actual Cost, 실제 원가)**
- **정의**: 특정 시점까지 실제 수행된 작업에 지출된 비용
- **별칭**: ACWP (Actual Cost of Work Performed)
- **계산**: 실제 지출 금액 합계
- **의미**: "얼마를 실제로 썼는가?"

#### 2.2 EVM 성과 분석 지표

**2.2.1 원가 편차 (CV, Cost Variance)**
- **공식**: CV = EV - AC
- **해석**:
  - CV > 0: 예산 대비 절감 (Good)
  - CV < 0: 예산 초과 (Bad)
  - CV = 0: 예산 준수

**2.2.2 일정 편차 (SV, Schedule Variance)**
- **공식**: SV = EV - PV
- **해석**:
  - SV > 0: 일정 앞섬 (Good)
  - SV < 0: 일정 지연 (Bad)
  - SV = 0: 일정 준수

**2.2.3 원가 성과 지수 (CPI, Cost Performance Index)**
- **공식**: CPI = EV / AC
- **해석**:
  - CPI > 1.0: 효율적 (1원 투입 시 1원 이상 가치 창출)
  - CPI < 1.0: 비효율적 (1원 투입 시 1원 미만 가치)
  - CPI = 1.0: 예산대로 집행

**2.2.4 일정 성과 지수 (SPI, Schedule Performance Index)**
- **공식**: SPI = EV / PV
- **해석**:
  - SPI > 1.0: 일정 앞섬
  - SPI < 1.0: 일정 지연
  - SPI = 1.0: 일정 준수

#### 2.3 EVM 예측 지표

**2.3.1 완료 시 예상 원가 (EAC, Estimate at Completion)**
현재 성과를 기반으로 프로젝트 완료 시 총 원가 예측

**방법 1: 현재 추세 지속 (가장 일반적)**
- **공식**: EAC = BAC / CPI
- **사용**: 현재 CPI가 계속 유지될 것으로 예상

**방법 2: 비정상적 편차 (일회성)**
- **공식**: EAC = AC + (BAC - EV)
- **사용**: 초기 편차가 일회성이며 향후 정상

**방법 3: CPI와 SPI 모두 고려**
- **공식**: EAC = AC + [(BAC - EV) / (CPI × SPI)]
- **사용**: 원가와 일정 모두 고려

**방법 4: 새로운 추정**
- **공식**: EAC = AC + 새로운 ETC
- **사용**: 기존 계획이 더 이상 유효하지 않음

**2.3.2 완료 시 예상 편차 (VAC, Variance at Completion)**
- **공식**: VAC = BAC - EAC
- **해석**:
  - VAC > 0: 예산 대비 절감 예상
  - VAC < 0: 예산 초과 예상

**2.3.3 완료를 위한 잔여 원가 (ETC, Estimate to Complete)**
- **공식**: ETC = EAC - AC
- **의미**: 프로젝트 완료까지 필요한 추가 비용

**2.3.4 완료 성과 지수 (TCPI, To-Complete Performance Index)**
남은 예산 내에서 프로젝트를 완료하기 위해 필요한 원가 효율

**BAC 기준:**
- **공식**: TCPI = (BAC - EV) / (BAC - AC)
- **사용**: 원래 예산(BAC) 준수 목표

**EAC 기준:**
- **공식**: TCPI = (BAC - EV) / (EAC - AC)
- **사용**: 수정된 예산(EAC) 기준

**해석**:
- TCPI > 1.0: 향후 더 효율적으로 작업 필요
- TCPI < 1.0: 향후 덜 효율적이어도 됨

#### 2.4 EVM 적용 절차
1. **WBS 작성**: 프로젝트를 작업 패키지로 분해
2. **예산 배정**: 각 작업 패키지에 예산 할당
3. **계획 가치 산정**: 시간별 PV 곡선 작성
4. **진척도 측정 규칙 수립**: 0/100, 50/50, 0/100 등
5. **실제 진척 측정**: 주간/월간 EV 계산
6. **실제 비용 집계**: AC 데이터 수집
7. **성과 분석**: CV, SV, CPI, SPI 계산
8. **예측 및 보고**: EAC, VAC, TCPI 산출

### 추가 설명

**EVM의 장점:**
1. 조기 경보: 문제를 조기에 발견
2. 객관적 측정: 주관적 추정 배제
3. 추세 분석: 미래 예측 가능
4. 통합 관리: 범위, 일정, 원가 동시 관리
5. 의사소통: 이해관계자에게 명확한 정보 제공

**EVM의 한계:**
1. 초기 설정 복잡: WBS, 예산 배정 시간 소요
2. 데이터 품질 의존: 정확한 진척도 측정 필요
3. 품질 미반영: 품질은 별도 측정 필요
4. 교육 필요: 팀원의 EVM 이해 필요

**EVM 적용 예시:**
```
프로젝트: 웹사이트 개발
BAC: 5,000만원
기간: 5개월

3개월 차 현황:
- PV: 3,000만원 (계획: 60% 완료)
- EV: 2,500만원 (실제: 50% 완료)
- AC: 2,800만원 (지출: 2,800만원)

분석:
- CV = 2,500 - 2,800 = -300만원 (예산 초과)
- SV = 2,500 - 3,000 = -500만원 (일정 지연)
- CPI = 2,500 / 2,800 = 0.89 (비효율)
- SPI = 2,500 / 3,000 = 0.83 (지연)

예측:
- EAC = 5,000 / 0.89 = 5,618만원
- VAC = 5,000 - 5,618 = -618만원 (초과 예상)
- TCPI = (5,000 - 2,500) / (5,000 - 2,800) = 1.14

결론: 프로젝트가 예산 초과 및 일정 지연 상태.
향후 CPI 1.14 이상으로 작업해야 원래 예산 준수 가능.
```

### 별첨

#### EVM 측정 규칙
| 규칙 | 시작 시 | 완료 시 | 적용 대상 |
|------|---------|---------|-----------|
| 0/100 | 0% | 100% | 단기 활동 (< 2주) |
| 50/50 | 50% | 50% | 중기 활동 (2-4주) |
| 20/80 | 20% | 80% | 리스크 높은 활동 |
| Milestone | 정의된 % | 정의된 % | 마일스톤 기반 |
| % Complete | 실제 % | 실제 % | 장기 활동 (> 1개월) |

## 3. 원가 산정 (Cost Estimating)

### 용어
**원가 산정 (Cost Estimating)**

### 설명
프로젝트 활동을 완료하는 데 필요한 금전적 자원을 예측하는 프로세스입니다.

### 세부 요소 및 설명

#### 3.1 원가 산정 기법

**3.1.1 유사 산정 (Analogous Estimating)**
- **정의**: 과거 유사 프로젝트의 실제 원가 활용
- **정확도**: ±25% (개략 산정)
- **장점**: 빠르고 비용 효율적
- **단점**: 정확도 낮음, 프로젝트 유사성 필수
- **활용**: 프로젝트 초기, 정보 부족 시

**예시:**
- 과거 모바일 앱 개발: 1억 원
- 현재 유사 앱: 1억 원 ± 25% = 7,500만~1억 2,500만 원

**3.1.2 모수 산정 (Parametric Estimating)**
- **정의**: 통계적 관계와 단위당 원가 활용
- **정확도**: ±10% (확정 산정)
- **장점**: 객관적, 정확도 높음
- **단점**: 신뢰 데이터 필요
- **활용**: 반복적 작업, 데이터 축적 영역

**예시:**
- 단위당 원가: 개발자 일당 50만 원
- 필요 인원: 5명 × 60일
- 총 원가: 5 × 60 × 50만 = 1억 5천만 원

**3.1.3 상향식 산정 (Bottom-Up Estimating)**
- **정의**: 세부 작업별 원가 산정 후 합산
- **정확도**: ±5% (확정 산정)
- **장점**: 가장 정확
- **단점**: 시간/비용 많이 소요
- **활용**: 실행 단계, 정확성 중요 시

**예시:**
```
WBS 기반 산정:
1. 요구사항 분석: 1,000만 원
2. 설계: 1,500만 원
3. 개발: 5,000만 원
4. 테스트: 2,000만 원
5. 배포: 500만 원
총계: 1억 원
```

**3.1.4 3점 산정 (Three-Point Estimating)**
- **정의**: 낙관, 가능성, 비관 시나리오 고려
- **공식**: cE = (cO + 4cM + cP) / 6
- **장점**: 불확실성 반영
- **활용**: 리스크 높은 활동

**예시:**
- 낙관적 (cO): 8,000만 원
- 가능성 (cM): 1억 원
- 비관적 (cP): 1억 4,000만 원
- 기대값: (8,000 + 40,000 + 14,000) / 6 = 1억 333만 원

#### 3.2 원가 유형

**3.2.1 직접 원가 vs 간접 원가**
- **직접 원가**: 프로젝트에만 귀속 (팀원 급여, 전용 장비)
- **간접 원가**: 여러 프로젝트 공유 (임대료, 관리비)

**3.2.2 고정 원가 vs 변동 원가**
- **고정 원가**: 작업량 무관 (계약 컨설턴트)
- **변동 원가**: 작업량 비례 (시간당 인건비)

**3.2.3 매몰 원가 (Sunk Cost)**
- **정의**: 이미 지출되어 회수 불가능한 비용
- **원칙**: 의사결정 시 고려 제외

**3.2.4 기회 원가 (Opportunity Cost)**
- **정의**: 선택하지 않은 대안의 가치
- **예시**: 프로젝트 A 선택 시 포기한 프로젝트 B의 수익

#### 3.3 원가 산정 시 고려사항
- **인플레이션**: 장기 프로젝트는 물가 상승 반영
- **환율**: 국제 프로젝트는 환율 변동 고려
- **학습 곡선**: 반복 작업 시 효율 향상
- **규모의 경제**: 대량 구매 시 할인
- **자원 가용성**: 시장 수급 상황
- **품질 수준**: 높은 품질은 비용 증가

### 추가 설명

**산정 정확도 발전 단계:**
1. **개념 단계**: ±50% (ROM, Rough Order of Magnitude)
2. **계획 초기**: ±25% (예산 산정)
3. **계획 후기**: ±10% (확정 산정)
4. **실행 중**: ±5% (통제 예산)

**원가 산정 체크리스트:**
- [ ] WBS 완성 여부
- [ ] 자원 요구사항 식별
- [ ] 과거 데이터 참조
- [ ] 전문가 의견 수렴
- [ ] 예비비 포함
- [ ] 간접비 고려
- [ ] 인플레이션 반영
- [ ] 가정사항 문서화

### 별첨

#### 원가 산정 정확도 비교
| 단계 | 기법 | 정확도 | 소요 시간 |
|------|------|--------|-----------|
| 개념 | 유사 산정 | ±50% | 1-2일 |
| 초기 | 모수 산정 | ±25% | 1주 |
| 계획 | 3점 산정 | ±10% | 2-3주 |
| 확정 | 상향식 산정 | ±5% | 1-2개월 |

- "Software Estimation" - Steve McConnell

## 4. 예산 결정 (Determine Budget)

### 용어
**예산 결정 (Determine Budget)**

### 설명
개별 활동이나 작업 패키지의 산정 원가를 종합하여 원가 베이스라인을 수립하는 프로세스입니다.

### 세부 요소 및 설명

#### 4.1 예산의 구성 요소

**4.1.1 작업 패키지 예산**
- WBS 최하위 수준의 예산
- 통제 계정(Control Account)으로 관리

**4.1.2 우발 예비비 (Contingency Reserve)**
- **정의**: 알려진 리스크 대비 예비비
- **규모**: 프로젝트 예산의 5-10%
- **관리**: PM이 승인 권한
- **사용**: 식별된 리스크 발생 시

**4.1.3 관리 예비비 (Management Reserve)**
- **정의**: 알려지지 않은 리스크 대비 예비비
- **규모**: 프로젝트 예산의 5-10%
- **관리**: 경영진 승인 필요
- **사용**: 범위 변경, 예상치 못한 이슈

**4.1.4 원가 베이스라인 (Cost Baseline)**
- **구성**: 작업 패키지 예산 + 우발 예비비
- **특징**: 시간 단계별(Time-Phased) 예산
- **목적**: 성과 측정 기준

**4.1.5 프로젝트 예산 (Project Budget)**
- **구성**: 원가 베이스라인 + 관리 예비비
- **승인**: 이해관계자 공식 승인

```
프로젝트 예산
├─ 원가 베이스라인
│  ├─ 작업 패키지 예산: 8,000만 원
│  └─ 우발 예비비: 800만 원 (10%)
└─ 관리 예비비: 880만 원 (10%)
─────────────────────────────────
총 프로젝트 예산: 9,680만 원
```

#### 4.2 예산 배분 기법

**4.2.1 자금 한도 조정 (Funding Limit Reconciliation)**
- 가용 자금에 맞춰 예산 조정
- 자금 제약 시 작업 일정 조정

**4.2.2 S-곡선 (S-Curve)**
- 시간에 따른 누적 원가 그래프
- 프로젝트 초기: 완만한 증가
- 프로젝트 중기: 급격한 증가
- 프로젝트 후기: 완만한 증가

#### 4.3 예산 승인 프로세스
1. 원가 산정 완료
2. 예비비 산정
3. 시간별 예산 배분
4. 원가 베이스라인 작성
5. 이해관계자 검토
6. 공식 승인
7. 베이스라인 설정

### 추가 설명

**예비비 관리 원칙:**
1. **분리 관리**: 우발/관리 예비비 구분
2. **투명성**: 사용 내역 명확히 기록
3. **승인 절차**: 권한에 따른 승인
4. **정기 리뷰**: 남은 예비비 확인
5. **미사용 시**: 프로젝트 종료 시 반환

**예산 책정 시 주의사항:**
- 과소 책정: 프로젝트 실패 위험
- 과다 책정: 자원 낭비, 승인 거부
- 현실적 산정: 과거 데이터 기반
- 버퍼 포함: 불확실성 고려

## 5. 원가 통제 (Control Costs)

### 용어
**원가 통제 (Control Costs)**

### 설명
프로젝트 예산 상태를 모니터링하고, 원가 베이스라인 변경을 관리하며, 실제 지출을 통제하는 프로세스입니다.

### 세부 요소 및 설명

#### 5.1 원가 통제 활동
- **원가 성과 측정**: EVM 지표 계산
- **변경 요청 검토**: 원가 영향 분석
- **베이스라인 업데이트**: 승인된 변경 반영
- **예비비 관리**: 사용 내역 추적
- **교훈 문서화**: 원가 관리 개선점

#### 5.2 원가 통제 기법

**5.2.1 EVM (Earned Value Management)**
- CV, SV, CPI, SPI 분석
- EAC, VAC 예측
- 추세 분석

**5.2.2 추세 분석 (Trend Analysis)**
- 그래픽 분석: 시간에 따른 성과 추이
- 회귀 분석: 미래 성과 예측
- 조기 경보: 문제 조기 발견

**5.2.3 편차 분석 (Variance Analysis)**
- 계획 대비 실제 비교
- 원인 분석 (근본 원인 파악)
- 시정 조치 계획

**5.2.4 예측 (Forecasting)**
- EAC 계산: 완료 시 예상 원가
- ETC 산정: 남은 작업 비용
- TCPI 계산: 필요 효율

#### 5.3 원가 통제 보고

**5.3.1 월간 원가 보고서 구성**
- 프로젝트 개요
- 현재 원가 상태 (PV, EV, AC)
- 성과 지표 (CV, SV, CPI, SPI)
- 예측 지표 (EAC, VAC, TCPI)
- 이슈 및 리스크
- 시정 조치 계획

**5.3.2 원가 대시보드**
- 주요 KPI 시각화
- 신호등 표시 (녹색/황색/적색)
- 추세 그래프
- 핵심 메시지

#### 5.4 원가 초과 시 대응 방안
1. **범위 축소**: 필수 기능만 구현
2. **품질 기준 조정**: 골드 플레이팅 제거
3. **자원 최적화**: 저비용 자원 활용
4. **일정 연장**: 자원 평준화
5. **추가 예산 요청**: 이해관계자 협의

### 추가 설명

**효과적인 원가 통제:**
1. **정기 모니터링**: 주간/월간 원가 리뷰
2. **조기 대응**: 편차 발생 즉시 조치
3. **근본 원인 분석**: 표면적 문제만 해결 금지
4. **예방 조치**: 재발 방지 대책
5. **투명한 커뮤니케이션**: 현황 공유

**원가 통제 실패 사례:**
- 늦은 발견: 월말에만 원가 확인
- 낙관적 태도: "나아질 것" 기대
- 근본 원인 미해결: 임시방편만 적용
- 의사소통 부족: 이해관계자 미통보

|------|-------------|----------------|--------------|

| SPI | ≥ 0.95 | 0.90 ~ 0.95 | < 0.90 |

## 6. ROI & 비용 편익 분석 (ROI & Cost-Benefit Analysis)

### 용어
**ROI (Return on Investment, 투자 수익률)**

### 설명
프로젝트 투자 대비 수익을 측정하는 재무 지표로, 프로젝트의 경제적 타당성을 평가하는 데 사용됩니다.

### 세부 요소 및 설명

#### 6.1 ROI 계산
**공식:**
```
ROI = (순이익 / 투자 비용) × 100%
    = [(총 수익 - 총 비용) / 총 비용] × 100%
```

**예시:**
- 투자 비용: 1억 원
- 예상 수익: 1억 5천만 원
- ROI = (1.5억 - 1억) / 1억 × 100% = 50%

**해석:**
- ROI > 0%: 수익 창출
- ROI = 0%: 손익분기
- ROI < 0%: 손실

#### 6.2 NPV (Net Present Value, 순현재가치)
**정의**: 미래 현금흐름을 현재 가치로 환산한 순이익

**공식:**
```
NPV = Σ [CFt / (1+r)^t] - 초기 투자
```
- CFt: t년도 현금흐름
- r: 할인율
- t: 기간

**예시:**
- 초기 투자: 1억 원
- 연간 수익: 3천만 원 (5년)
- 할인율: 10%
- NPV = 3천/(1.1) + 3천/(1.1)^2 + ... - 1억

**해석:**
- NPV > 0: 프로젝트 수용
- NPV = 0: 무차별
- NPV < 0: 프로젝트 거부

#### 6.3 IRR (Internal Rate of Return, 내부수익률)
**정의**: NPV를 0으로 만드는 할인율

**해석:**
- IRR > 요구수익률: 프로젝트 수용
- IRR < 요구수익률: 프로젝트 거부

#### 6.4 회수 기간 (Payback Period)
**정의**: 초기 투자를 회수하는 데 걸리는 시간

**예시:**
- 투자: 1억 원
- 연간 수익: 2천만 원
- 회수 기간: 5년

**단순 회수 vs 할인 회수:**
- 단순: 화폐의 시간 가치 미고려
- 할인: NPV 개념 적용

#### 6.5 비용 편익 비율 (BCR, Benefit-Cost Ratio)
**공식:**
```
BCR = 편익의 현재가치 / 비용의 현재가치
```

**해석:**
- BCR > 1: 편익이 비용 초과
- BCR = 1: 손익분기
- BCR < 1: 비용이 편익 초과

### 추가 설명

**프로젝트 선택 시 고려사항:**
1. **재무적 측면**: ROI, NPV, IRR
2. **전략적 측면**: 비전, 미션 부합
3. **리스크**: 불확실성, 위험도
4. **자원**: 가용 인력, 예산
5. **시장**: 경쟁, 고객 요구

**무형 편익 고려:**
- 브랜드 가치 향상
- 고객 만족도 증가
- 조직 역량 강화
- 시장 지위 확보

### 별첨

#### 프로젝트 선택 예시
| 프로젝트 | 투자 | NPV | IRR | 회수 | 선택 |
|---------|-----|-----|-----|------|-----|
| A | 1억 | 3천만 | 25% | 3년 | ● |
| B | 2억 | 2천만 | 15% | 5년 | - |
| C | 5천만 | 1천만 | 30% | 2년 | ● |
