## 📖 1. 원가 관리의 중요성

### 1.1 프로젝트 실패와 원가
- **예산 초과**: 프로젝트 실패 원인 2위 (45%)
- **원가 관리 실패 사례**:
  - 시드니 오페라 하우스: 예산 14배 초과 (700만 → 1억 달러)
  - 영국 NHS IT 시스템: 120억 파운드 낭비 후 프로젝트 폐기
  - 보잉 787 드림라이너: 개발비 180억 달러 초과

### 1.2 효과적인 원가 관리의 이점
- ✅ 예산 내 프로젝트 완료
- ✅ 이해관계자 신뢰 확보
- ✅ 조기 경보 시스템 (Early Warning)
- ✅ 데이터 기반 의사결정
- ✅ 자원 최적화

### 1.3 원가 관리 vs 재무 관리
| 구분 | 원가 관리 | 재무 관리 |
|-----|----------|----------|
| 범위 | 프로젝트 비용 | 조직 전체 재무 |
| 초점 | 예산 준수 | ROI, 수익성 |
| 기간 | 프로젝트 기간 | 회계 연도 |
| 책임자 | PM | CFO |

## 📊 2. 원가 관리 프로세스 (PMBOK 기반)

### 2.1 4단계 프로세스

```
1. 원가 관리 계획 수립 (Plan Cost Management)
   ↓
2. 원가 산정 (Estimate Costs)
   ↓
3. 예산 결정 (Determine Budget)
   ↓
4. 원가 통제 (Control Costs)
```

### 2.2 각 단계별 상세 설명

#### 2.2.1 원가 관리 계획 수립
**목적**: 원가를 산정, 예산 수립, 모니터링, 통제하는 방법 정의

**주요 내용**:
- 측정 단위 (원, 달러, 인시/인월)
- 정확도 수준 (+/-10%, +/-25%)
- 통제 임계치 (예: ±10% 초과 시 보고)
- 보고 형식 및 주기
- 조직의 절차 연계

**산출물**: 원가 관리 계획서

#### 2.2.2 원가 산정
**목적**: 활동 완료에 필요한 자원의 대략적 비용 산정

**입력 정보**:
- 범위 기준선 (WBS)
- 프로젝트 일정
- 자원 요구사항
- 리스크 등록부

**산출물**: 활동별 원가 산정치, 산정 근거

#### 2.2.3 예산 결정
**목적**: 승인된 원가 산정치를 집계하여 원가 기준선 수립

**주요 활동**:
- 원가 집계 (Bottom-Up)
- 예비비 추가 (우발/관리)
- S-커브 작성
- 자금 조달 요구사항 결정

**산출물**: 원가 기준선, 프로젝트 자금 요구사항

#### 2.2.4 원가 통제
**목적**: 프로젝트 예산 상태 모니터링 및 변경 관리

**주요 활동**:
- EVM 분석
- 예측 업데이트
- 변경 요청 처리
- 시정 조치

**산출물**: 작업 성과 정보, 원가 예측, 변경 요청

## 💰 3. 원가 유형

### 3.1 직접 원가 (Direct Costs)
**정의**: 프로젝트에만 직접 귀속되는 비용

**예시**:
- 프로젝트 팀원 급여
- 전용 장비 구매/임대
- 프로젝트 전용 소프트웨어 라이선스
- 출장비
- 하청 비용

### 3.2 간접 원가 (Indirect Costs)
**정의**: 여러 프로젝트에 공통으로 발생하는 비용

**예시**:
- 사무실 임대료
- 공과금 (전기, 인터넷)
- 관리 부서 인건비 (HR, 재무)
- 공용 장비 감가상각비

**배분 방법**:
- 프로젝트 규모 비율
- 인원 비율
- 직접 원가 비율

### 3.3 고정 원가 (Fixed Costs)
**정의**: 작업량과 무관하게 일정한 비용

**예시**:
- 계약 컨설턴트 비용 (프로젝트당 1,000만원)
- 소프트웨어 라이선스 (연 단위)
- 장비 임대료

### 3.4 변동 원가 (Variable Costs)
**정의**: 작업량에 비례하여 변동하는 비용

**예시**:
- 시간당 인건비
- 클라우드 컴퓨팅 사용료 (사용량 기준)
- 재료비
- 초과 근무 수당

### 3.5 매몰 원가 (Sunk Costs)
**정의**: 이미 지출되어 회수 불가능한 비용

**특징**:
- 미래 의사결정에서 제외
- 과거 투자에 집착하는 오류 주의

**예시**:
```
프로젝트 A에 이미 5,000만원 투자
현재 완료율 30%, 추가 1억원 필요
프로젝트 B (신규): 6,000만원으로 동일 목표 달성 가능

❌ 잘못된 판단: "A에 이미 5,000만원 투자했으니 계속 진행"
✅ 올바른 판단: 총 비용 비교 (A: 1.5억 vs B: 6천만) → B 선택 고려
```

## 📐 4. 원가 산정 기법

### 4.1 유사 산정 (Analogous Estimating)

**정의**: 과거 유사 프로젝트의 실적 데이터 활용

**장점**:
- ✅ 빠르고 저렴
- ✅ 전문가 판단 활용
- ✅ 초기 단계 적합

**단점**:
- ❌ 낮은 정확도
- ❌ 프로젝트 차이 반영 어려움
- ❌ 과거 데이터 필요

**정확도**: ±25% ~ ±75%

**예시**:
```
과거 웹사이트 개발 프로젝트: 3,000만원
새 프로젝트: 유사한 규모/복잡도
→ 산정: 2,500만원 ~ 4,000만원
```

### 4.2 모수 산정 (Parametric Estimating)

**정의**: 통계적 모델 및 단위당 비용 사용

**공식**: 총 원가 = 단위당 원가 × 수량

**장점**:
- ✅ 높은 정확도
- ✅ 객관적 데이터 기반
- ✅ 확장 가능

**단점**:
- ❌ 정확한 파라미터 데이터 필요
- ❌ 모델 개발 비용
- ❌ 비선형 관계 반영 어려움

**정확도**: ±10% ~ ±25%

**예시**:
```
소프트웨어 개발:
- 라인당 비용: 5,000원
- 총 라인 수: 10,000 라인
- 총 원가 = 5,000 × 10,000 = 5,000만원

건설:
- 평당 건축비: 500만원
- 건축 면적: 100평
- 총 원가 = 500 × 100 = 5억원
```

### 4.3 상향식 산정 (Bottom-Up Estimating)

**정의**: 최하위 작업 패키지부터 산정 후 상위로 집계

**프로세스**:
```
1. WBS 최하위 작업 패키지 식별
2. 각 작업 패키지 원가 산정
3. 작업 패키지 → 통제 계정 → WBS 상위로 집계
```

**장점**:
- ✅ 가장 정확
- ✅ 상세한 근거
- ✅ 팀 참여

**단점**:
- ❌ 시간 소요
- ❌ 비용 발생
- ❌ 상세 정보 필요

**정확도**: ±5% ~ ±10%

**예시**:
```
웹사이트 개발 프로젝트:

작업 패키지 1: 요구사항 분석
- 분석가 (5일 × 50만원) = 250만원

작업 패키지 2: UI 디자인
- 디자이너 (10일 × 40만원) = 400만원

작업 패키지 3: 프론트엔드 개발
- 개발자 (20일 × 60만원) = 1,200만원

작업 패키지 4: 백엔드 개발
- 개발자 (25일 × 60만원) = 1,500만원

작업 패키지 5: 테스트
- 테스터 (10일 × 40만원) = 400만원

총 원가 = 3,750만원
```

### 4.4 3점 산정 (Three-Point Estimating)

**정의**: 낙관적, 가능성 높음, 비관적 시나리오 고려

**공식**:
- **삼각 분포**: E = (O + M + P) / 3
- **베타 분포 (PERT)**: E = (O + 4M + P) / 6

여기서:
- O = 낙관적 (Optimistic)
- M = 가능성 높음 (Most Likely)
- P = 비관적 (Pessimistic)
- E = 기대값 (Expected)

**표준편차**: σ = (P - O) / 6

**예시**:
```
백엔드 개발 원가 산정:
- 낙관적 (O): 1,000만원 (모든 것이 순조롭게 진행)
- 가능성 높음 (M): 1,500만원 (일반적인 상황)
- 비관적 (P): 2,500만원 (많은 문제 발생)

베타 분포 (PERT):
E = (1,000 + 4×1,500 + 2,500) / 6
  = (1,000 + 6,000 + 2,500) / 6
  = 9,500 / 6
  = 1,583만원

표준편차:
σ = (2,500 - 1,000) / 6 = 250만원

→ 산정 원가: 1,583만원 ± 250만원
```

### 4.5 예비비 (Reserve)

#### 4.5.1 우발 예비비 (Contingency Reserve)

**목적**: 알려진 리스크 대응

**특징**:
- 원가 기준선에 포함
- PM 권한으로 사용
- 식별된 리스크 대응

**산정 방법**:
- 비율 기반: 총 원가의 5~15%
- 리스크 기반: EMV 합계
- 전문가 판단

**예시**:
```
프로젝트 원가: 1억원
우발 예비비 (10%): 1,000만원
→ 원가 기준선: 1.1억원
```

#### 4.5.2 관리 예비비 (Management Reserve)

**목적**: 미지의 리스크 대응

**특징**:
- 원가 기준선에 미포함
- 경영진 승인 필요
- "Unknown Unknowns" 대응

**산정 방법**:
- 일반적으로 총 예산의 5~10%
- 프로젝트 복잡도/불확실성 고려

**예시**:
```
원가 기준선: 1.1억원
관리 예비비 (10%): 1,100만원
→ 총 프로젝트 예산: 1.21억원
```

## 💵 5. 예산 수립

### 5.1 원가 집계 프로세스

```
작업 패키지 원가
    ↓
통제 계정 (Control Account)
    ↓
WBS 상위 레벨
    ↓
프로젝트 총 원가
    ↓
+ 우발 예비비
    ↓
원가 기준선 (Cost Baseline)
    ↓
+ 관리 예비비
    ↓
프로젝트 예산 (Project Budget)
```

### 5.2 원가 기준선 (Cost Baseline)

**정의**: 시간에 따라 배분된 승인된 예산

**구성**:
- 활동별 원가
- 우발 예비비
- (관리 예비비 제외)

**용도**:
- 성과 측정 기준
- EVM 계산 기준
- 변경 통제 기준

### 5.3 S-커브 (S-Curve)

**정의**: 누적 원가를 시간에 따라 표시한 그래프

**특징**:
```
누적
원가
 ↑
 │           ___---
 │       _--
 │    _-
 │  _/
 │_/
└──────────────────> 시간
```

**형태**:
- 초기: 느린 증가 (계획/설계)
- 중기: 급격한 증가 (실행)
- 후기: 완만한 증가 (마무리)

**활용**:
- 자금 조달 계획
- 현금 흐름 예측
- 진행률 추적

### 5.4 자금 조달 요구사항

**고려사항**:
- 월별/분기별 자금 필요 시기
- 자금 조달 리드 타임
- 최소/최대 자금 보유량
- 지급 조건 (선금, 중도금, 잔금)

**예시**:
```
프로젝트 기간: 12개월
총 예산: 1.2억원

자금 조달 계획:
- 1분기: 2,000만원 (초기 투자)
- 2분기: 4,000만원 (본격 실행)
- 3분기: 4,000만원 (실행 지속)
- 4분기: 2,000만원 (마무리)
```

## 📈 6. EVM (Earned Value Management)

### 6.1 EVM이란?

**정의**: 범위, 일정, 원가를 통합하여 프로젝트 성과를 측정하는 기법

**역사**: 
- 1960년대 미국 국방부 개발
- 현재 전 세계 표준 기법

**장점**:
- ✅ 객관적 성과 측정
- ✅ 조기 경보
- ✅ 완료 시점 예측
- ✅ 트렌드 분석

### 6.2 EVM 핵심 용어

#### 6.2.1 기본 측정값

**1. PV (Planned Value) - 계획 가치**
- **정의**: 특정 시점까지 완료하기로 계획된 작업의 예산
- **다른 이름**: BCWS (Budgeted Cost of Work Scheduled)
- **계산**: 계획된 작업량 × 예산

**2. EV (Earned Value) - 획득 가치**
- **정의**: 실제 완료된 작업의 예산 가치
- **다른 이름**: BCWP (Budgeted Cost of Work Performed)
- **계산**: 실제 작업량 × 예산

**3. AC (Actual Cost) - 실제 원가**
- **정의**: 실제 완료된 작업에 소요된 비용
- **다른 이름**: ACWP (Actual Cost of Work Performed)
- **계산**: 실제 지출 비용

**4. BAC (Budget at Completion) - 완료 시 예산**
- **정의**: 프로젝트 완료 시 총 계획 예산
- **계산**: 전체 PV

### 6.3 EVM 계산 공식

#### 6.3.1 분산 (Variance) 분석

**1. CV (Cost Variance) - 원가 분산**
```
CV = EV - AC

해석:
- CV > 0: 예산 절감 (좋음)
- CV = 0: 예산 준수
- CV < 0: 예산 초과 (나쁨)
```

**2. SV (Schedule Variance) - 일정 분산**
```
SV = EV - PV

해석:
- SV > 0: 일정 앞섬 (좋음)
- SV = 0: 일정 준수
- SV < 0: 일정 지연 (나쁨)
```

#### 6.3.2 성과 지수 (Performance Index)

**1. CPI (Cost Performance Index) - 원가 성과 지수**
```
CPI = EV / AC

해석:
- CPI > 1.0: 효율적 (1원으로 1원 이상의 가치 창출)
- CPI = 1.0: 예산대로
- CPI < 1.0: 비효율적 (1원으로 1원 미만의 가치)

예: CPI = 0.85 → 1원으로 0.85원의 가치만 창출
```

**2. SPI (Schedule Performance Index) - 일정 성과 지수**
```
SPI = EV / PV

해석:
- SPI > 1.0: 일정 앞섬
- SPI = 1.0: 일정대로
- SPI < 1.0: 일정 지연

예: SPI = 0.90 → 계획 대비 90% 진행
```

#### 6.3.3 예측 (Forecasting)

**1. EAC (Estimate at Completion) - 완료 시 예상 원가**

**유형 1: 전형적인 편차가 계속될 경우**
```
EAC = BAC / CPI

예: BAC = 100,000원, CPI = 0.87
→ EAC = 100,000 / 0.87 = 114,943원
```

**유형 2: 일회성 편차인 경우**
```
EAC = AC + (BAC - EV)

예: AC = 52,000원, BAC = 100,000원, EV = 45,000원
→ EAC = 52,000 + (100,000 - 45,000) = 107,000원
```

**유형 3: 초기 계획이 유효하지 않은 경우**
```
EAC = AC + 새로운 산정치
```

**유형 4: 원가와 일정 모두 고려**
```
EAC = AC + [(BAC - EV) / (CPI × SPI)]
```

**2. ETC (Estimate to Complete) - 완료까지 필요한 원가**
```
ETC = EAC - AC

예: EAC = 114,943원, AC = 52,000원
→ ETC = 114,943 - 52,000 = 62,943원
```

**3. VAC (Variance at Completion) - 완료 시 예상 분산**
```
VAC = BAC - EAC

해석:
- VAC > 0: 예산 내 완료 예상
- VAC < 0: 예산 초과 예상

예: BAC = 100,000원, EAC = 114,943원
→ VAC = 100,000 - 114,943 = -14,943원 (초과)
```

**4. TCPI (To-Complete Performance Index) - 목표 달성 필요 성과**

**BAC 달성 시:**
```
TCPI = (BAC - EV) / (BAC - AC)

예: BAC = 100,000원, EV = 45,000원, AC = 52,000원
→ TCPI = (100,000 - 45,000) / (100,000 - 52,000)
       = 55,000 / 48,000 = 1.15

해석: 남은 작업을 1.15의 효율로 수행해야 BAC 달성
```

**EAC 달성 시:**
```
TCPI = (BAC - EV) / (EAC - AC)
```

### 6.4 EVM 종합 예제

#### 예제 1: 웹 개발 프로젝트

**프로젝트 정보**:
- BAC (총 예산): 100,000,000원
- 프로젝트 기간: 10개월
- 현재 시점: 6개월차
- 계획 진행률: 60%
- 실제 진행률: 50%

**데이터**:
- PV = 100,000,000 × 60% = 60,000,000원
- EV = 100,000,000 × 50% = 50,000,000원
- AC = 58,000,000원

**분석**:

1. **분산 분석**:
```
CV = EV - AC = 50,000,000 - 58,000,000 = -8,000,000원
→ 800만원 예산 초과

SV = EV - PV = 50,000,000 - 60,000,000 = -10,000,000원
→ 1,000만원 상당의 작업 지연
```

2. **성과 지수**:
```
CPI = EV / AC = 50,000,000 / 58,000,000 = 0.86
→ 1원당 0.86원의 가치만 창출 (비효율적)

SPI = EV / PV = 50,000,000 / 60,000,000 = 0.83
→ 계획 대비 83% 진행 (지연)
```

3. **예측**:
```
EAC = BAC / CPI = 100,000,000 / 0.86 = 116,279,070원
→ 완료 시 약 1.63억원 예상

ETC = EAC - AC = 116,279,070 - 58,000,000 = 58,279,070원
→ 추가로 약 5,828만원 필요

VAC = BAC - EAC = 100,000,000 - 116,279,070 = -16,279,070원
→ 약 1,628만원 초과 예상

TCPI = (BAC - EV) / (BAC - AC)
     = (100,000,000 - 50,000,000) / (100,000,000 - 58,000,000)
     = 50,000,000 / 42,000,000 = 1.19
→ 남은 작업을 1.19 효율로 수행해야 예산 내 완료
```

4. **결론**:
- ❌ 예산 800만원 초과, 일정 지연
- ❌ 완료 시 약 1,628만원 초과 예상
- ⚠️ 즉각적인 시정 조치 필요
- 💡 남은 작업을 현재보다 38% 효율적으로 수행해야 함

#### 예제 2: 건설 프로젝트

**프로젝트 정보**:
- BAC: 500,000,000원
- 현재 시점: 4개월차 (전체 12개월)
- PV: 150,000,000원
- EV: 160,000,000원
- AC: 155,000,000원

**분석**:
```
CV = 160,000,000 - 155,000,000 = +5,000,000원 (예산 절감)
SV = 160,000,000 - 150,000,000 = +10,000,000원 (일정 앞섬)

CPI = 160,000,000 / 155,000,000 = 1.03 (효율적)
SPI = 160,000,000 / 150,000,000 = 1.07 (일정 앞섬)

EAC = 500,000,000 / 1.03 = 485,436,893원
VAC = 500,000,000 - 485,436,893 = +14,563,107원 (절감)
```

**결론**:
- ✅ 예산 500만원 절감, 일정 앞섬
- ✅ 완료 시 약 1,456만원 절감 예상
- 💡 현재 성과 유지하면 성공적 완료 가능

## 🎯 7. 원가 통제

### 7.1 원가 통제 프로세스

```
1. 성과 측정 (EVM)
   ↓
2. 편차 분석
   ↓
3. 트렌드 분석
   ↓
4. 예측 업데이트
   ↓
5. 시정 조치
   ↓
6. 변경 요청 (필요 시)
```

### 7.2 원가 통제 기법

#### 7.2.1 정기적 성과 검토
- 주간/월간 EVM 보고
- 분산 분석
- 트렌드 그래프 작성

#### 7.2.2 임계치 관리
**예시**:
```
원가 통제 임계치: ±10%

CPI 범위:
- 1.10 이상: 우수
- 0.90 ~ 1.10: 정상
- 0.90 미만: 시정 조치 필요
```

#### 7.2.3 예측 업데이트
- EAC 정기 업데이트
- 완료 일정 재산정
- 자금 조달 계획 조정

### 7.3 원가 절감 방안

#### 7.3.1 범위 관리
- 필수/선택 기능 재분류
- 범위 축소 협상
- 단계별 인도 (Phased Delivery)

#### 7.3.2 자원 최적화
- 자원 레벨링 (Resource Leveling)
- 저비용 자원 활용
- 자원 공유 (Multi-tasking 주의)

#### 7.3.3 프로세스 개선
- 재작업 최소화
- 품질 비용 최적화
- 자동화 도입

#### 7.3.4 일정 압축
- **크래싱 (Crashing)**: 자원 추가 투입
  - 원가 증가, 일정 단축
  - ROI 분석 필요
- **패스트 트래킹 (Fast Tracking)**: 병렬 작업
  - 리스크 증가 주의

#### 7.3.5 외주 vs 내부 개발 재평가
- Make-or-Buy 분석
- 핵심 역량 집중
- 비핵심 업무 외주

### 7.4 변경 통제

#### 7.4.1 변경 요청 처리
```
1. 변경 요청 접수
   ↓
2. 원가 영향 분석
   ↓
3. 통합 변경 통제 위원회 (CCB) 검토
   ↓
4. 승인/거부 결정
   ↓
5. 승인 시: 기준선 업데이트
```

#### 7.4.2 기준선 업데이트
- 승인된 변경 사항 반영
- 재기준선 (Re-baseline)
  - 필요 시에만 수행
  - 이력 보존

## 📚 부록: EVM 계산 체크리스트

### EVM 계산 순서
```
1단계: 기본 값 확인
□ BAC (총 예산)
□ 현재 시점
□ 계획 진행률 → PV 계산
□ 실제 진행률 → EV 계산
□ 실제 지출 → AC 확인

2단계: 분산 계산
□ CV = EV - AC
□ SV = EV - PV

3단계: 성과 지수 계산
□ CPI = EV / AC
□ SPI = EV / PV

4단계: 예측 계산
□ EAC 선택 (편차 유형 판단)
□ ETC = EAC - AC
□ VAC = BAC - EAC
□ TCPI = (BAC - EV) / (BAC - AC)

5단계: 해석 및 시정 조치
□ 프로젝트 상태 분석
□ 트렌드 파악
□ 시정 조치 방안 수립
```

### EVM 해석 가이드
```
[건강한 프로젝트]
✅ CPI ≥ 1.0 (예산 준수/절감)
✅ SPI ≥ 1.0 (일정 준수/앞섬)
✅ CV ≥ 0 (예산 절감)
✅ SV ≥ 0 (일정 앞섬)

[주의 필요]
⚠️ 0.9 ≤ CPI < 1.0 (예산 약간 초과)
⚠️ 0.9 ≤ SPI < 1.0 (일정 약간 지연)

[위험 상태]
❌ CPI < 0.9 (예산 크게 초과)
❌ SPI < 0.9 (일정 크게 지연)
❌ TCPI > 1.1 (목표 달성 어려움)
```
