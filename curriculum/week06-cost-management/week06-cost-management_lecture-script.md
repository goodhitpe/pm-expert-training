### [00:15-01:30] 섹션 1: 원가 관리 기초 (75분)

#### [00:15-00:45] 1.1 원가 관리 프로세스 (30분)

**강의 내용**:
```
📖 원가 관리란?
"원가 관리는 프로젝트를 승인된 예산 내에서 완료하기 위한 
계획, 산정, 예산 수립, 자금 조달, 통제 프로세스입니다."

🔄 원가 관리 4단계 프로세스 (PMBOK 기반)
화이트보드에 그림 그리기:

1. 원가 관리 계획 수립 → "어떻게 관리할 것인가?"
   ↓
2. 원가 산정 → "얼마나 들까?"
   ↓
3. 예산 결정 → "언제 돈이 필요한가?"
   ↓
4. 원가 통제 → "실제는 어떤가? 조치가 필요한가?"

각 단계별 상세 설명:

📍 1단계: 원가 관리 계획 수립
- 측정 단위: 원, 달러, 인시, 인월
- 정확도: ±10%, ±25%
- 통제 임계치: 10% 초과 시 보고
- 예: "우리 프로젝트는 ±10% 초과 시 즉시 보고합니다"

📍 2단계: 원가 산정
- 활동별, 자원별 비용 예측
- 예: 개발자 1명 × 3개월 × 500만원 = 1,500만원

📍 3단계: 예산 결정
- 산정치를 시간별로 배분
- S-커브 작성 (누적 원가 곡선)
- 예비비 추가 (우발 + 관리)

📍 4단계: 원가 통제
- 실제 vs 계획 비교
- EVM 분석
- 시정 조치

💡 실무 팁:
"초기 단계의 정확한 산정이 가장 중요합니다. 
잘못된 산정은 프로젝트 전체를 흔들 수 있습니다."
```

**교수법**:
- 화이트보드에 프로세스 흐름도 그리기
- 각 단계별 간단한 예시 제시
- 학습자 이해도 확인 질문

#### [00:45-01:15] 1.2 원가 유형 (30분)

**강의 내용**:
```
💰 원가 유형 이해하기

1️⃣ 직접 원가 vs 간접 원가

✅ 직접 원가 (Direct Costs)
- 특정 프로젝트에만 귀속
- 예시:
  • 프로젝트 팀원 급여
  • 전용 서버 구매
  • 프로젝트 전용 SW 라이선스
  • 출장비

❌ 간접 원가 (Indirect Costs)
- 여러 프로젝트에 공통 발생
- 예시:
  • 사무실 임대료
  • 공과금 (전기, 인터넷)
  • 관리 부서 인건비
  • 공용 장비

💬 토론: "AWS 비용은 직접 원가일까요, 간접 원가일까요?"
→ 답: 전용 인스턴스면 직접, 공용이면 간접

2️⃣ 고정 원가 vs 변동 원가

📊 고정 원가 (Fixed Costs)
- 작업량과 무관하게 일정
- 예: 계약 컨설턴트 비용 1,000만원 (프로젝트당)
- 그래프: 수평선

📈 변동 원가 (Variable Costs)
- 작업량에 비례
- 예: 시간당 인건비, 클라우드 사용료
- 그래프: 우상향 직선

3️⃣ 매몰 원가 (Sunk Costs)

⚠️ 의사결정에서 제외해야 할 비용
"이미 지출된 비용은 돌려받을 수 없습니다.
미래를 보고 판단해야 합니다."

📌 실습 예제:
```
프로젝트 A: 이미 5,000만원 투자, 추가 1억 필요
프로젝트 B: 신규, 6,000만원으로 동일 목표 달성

❌ 잘못된 판단: "A에 이미 5천만원 넣었으니 계속"
✅ 올바른 판단: 추가 비용만 비교 (A: 1억 vs B: 6천만)
```

💡 실무 사례:
"스타트업에서 3년간 개발한 시스템이 시장 변화로 쓸모없어졌습니다.
이미 투자한 10억은 매몰 원가입니다. 
새로운 방향으로 빠르게 전환하는 것이 맞습니다."
```

**교수법**:
- 표와 그래프로 시각화
- 실제 프로젝트 예시 풍부하게 제시
- 학습자들의 경험 공유 유도

### [01:45-03:15] 섹션 2: 원가 산정 기법 (90분)

#### [01:45-02:30] 2.1 원가 산정 기법 이론 (45분)

**강의 내용**:
```
📐 원가 산정 기법 5가지

1️⃣ 유사 산정 (Analogous Estimating)

📖 정의: 과거 유사 프로젝트 데이터 활용

장점: ✅ 빠르고 저렴, ✅ 초기 단계 적합
단점: ❌ 낮은 정확도 (±25% ~ ±75%)

예시:
"작년에 비슷한 웹사이트 개발에 3,000만원 들었어요.
이번 프로젝트도 비슷하니 2,500만원~4,000만원 정도?"

💬 질문: "언제 이 방법을 사용하면 좋을까요?"
→ 답: 프로젝트 초기, 빠른 견적 필요 시

2️⃣ 모수 산정 (Parametric Estimating)

📖 정의: 통계 모델 및 단위당 비용 사용

공식: 총 원가 = 단위당 원가 × 수량

장점: ✅ 높은 정확도, ✅ 객관적
단점: ❌ 정확한 파라미터 데이터 필요

예시:
"API 개발 1개당 200만원, 총 15개 필요
→ 200만원 × 15 = 3,000만원"

실습:
```
파라미터: 코드 라인당 5,000원
예상 코드: 10,000 라인
→ 계산: 5,000원 × 10,000 = 5,000만원
```

3️⃣ 상향식 산정 (Bottom-Up Estimating)

📖 정의: 최하위 작업부터 산정 후 합산

장점: ✅ 가장 정확 (±5% ~ ±10%)
단점: ❌ 시간 많이 소요, ❌ WBS 필요

예시:
```
작업 패키지 1: UI 설계      = 500만원
작업 패키지 2: API 개발     = 1,500만원
작업 패키지 3: 데이터베이스 = 800만원
작업 패키지 4: 테스트       = 600만원
─────────────────────────────────
총합:                         3,400만원
```

💡 실무 팁: "WBS가 상세할수록 정확도가 높아집니다"

4️⃣ 3점 산정 (Three-Point Estimating)

📖 정의: 낙관/가능성 높음/비관 고려

공식들:
- 삼각 분포: (O + M + P) / 3
- 베타 분포 (PERT): (O + 4M + P) / 6

예시:
```
로그인 기능 개발 시간:
- 낙관적 (O): 3일 (모든 게 순조로움)
- 가능성 높음 (M): 5일 (일반적)
- 비관적 (P): 10일 (많은 문제 발생)

베타 분포 계산:
(3 + 4×5 + 10) / 6 = 33 / 6 = 5.5일

비용: 5.5일 × 50만원/일 = 275만원
```

💬 질문: "왜 베타 분포는 중간값에 4배 가중치를 줄까요?"
→ 답: 가능성이 가장 높은 값을 중요하게 반영

5️⃣ 예비비 (Reserve)

📦 우발 예비비 (Contingency Reserve)
- 알려진 리스크 대응
- PM 권한으로 사용
- 약 10-20% 추가
- 예: 인력 변동, 기술 어려움

📦 관리 예비비 (Management Reserve)
- 미지의 리스크 대응
- 경영진 승인 필요
- 약 5-10% 추가
- 예: 시장 변화, 규제 변경

💰 예산 구조:
```
원가 기준선 (Cost Baseline):
  활동 원가      = 8,000만원
  + 우발 예비비  = 1,200만원 (15%)
  ────────────────────────
  = 9,200만원

프로젝트 예산 (Project Budget):
  원가 기준선    = 9,200만원
  + 관리 예비비  = 800만원 (8%)
  ────────────────────────
  = 1억원
```
```

**교수법**:
- 각 기법마다 명확한 예시 제시
- 화이트보드에 계산 과정 상세히 표시
- 학습자들이 따라 계산하도록 유도
- 언제 어떤 기법을 사용할지 가이드

```
📋 실습 시나리오:
"중소기업을 위한 재고 관리 시스템 개발 프로젝트"

주어진 정보:

- 새 프로젝트: 4개월, 복잡도 중간
→ 산정치를 계산하세요 (±30% 범위)

문제 2: 모수 산정
파라미터:
- 화면 개발: 화면당 100만원

모듈     낙관적  가능성높음  비관적

보고서      200      400      700

→ 베타 분포로 각 모듈의 기대 비용 계산
→ 총 개발 비용 산출

문제 4: 예비비 계산
활동 원가: 5,000만원
- 우발 예비비 15% 추가
- 관리 예비비 10% 추가 (원가 기준선 기준)
→ 원가 기준선과 프로젝트 예산 계산

⏰ 시간: 30분 실습, 15분 풀이

💡 강사 역할:
- 개별 순회하며 질문 답변
- 계산 어려워하는 학습자 도움
- 실습 시트 제공

📊 풀이 시간:
- 각 문제별 정답 설명
- 다양한 접근 방법 공유
- 실무 적용 팁 제공

예시 답안:

문제 1:
중간값: (6000/6 + 3500/3) × 4 = (1000 + 1167) × 4 = 4,334만원
범위: 3,034만원 ~ 5,634만원

총: 2,800만원

주문처리: (350 + 4×550 + 850)/6 = 567만원
보고서: (200 + 4×400 + 700)/6 = 417만원

문제 4:
원가 기준선: 5,000 + 750 = 5,750만원
프로젝트 예산: 5,750 + 575 = 6,325만원
```

### [03:30-05:00] 섹션 3: 예산 수립 및 EVM (90분)

#### [03:30-04:00] 3.1 예산 수립 및 S-커브 (30분)

**강의 내용**:
```
💰 예산 결정 프로세스

1️⃣ 원가 집계 (Cost Aggregation)
"상향식으로 합산합니다"

예시:
```
활동 → 작업 패키지 → 통제 계정 → 프로젝트
```

2️⃣ 시간별 배분
"언제 돈이 필요한지 계획"

월별 예산 배분 예시:
```
월    활동 원가  누적 원가
1월   1,000     1,000
2월   1,500     2,500
3월   2,000     4,500
4월   1,500     6,000
5월   1,000     7,000
6월     500     7,500
```

3️⃣ S-커브 (S-Curve) 작성

📈 S-커브란?
"프로젝트 누적 원가를 시간에 따라 그린 곡선"
"알파벳 S자 모양을 띰"

S-커브의 3단계:
```
시작 단계: 완만한 상승 (계획, 설계)
   ↓
중간 단계: 가파른 상승 (개발, 구현)
   ↓
종료 단계: 완만한 상승 (테스트, 마무리)
```

화이트보드에 S-커브 그리기:
- X축: 시간 (월)
- Y축: 누적 원가 (만원)
- 계획선(PV)과 실제선(AC) 비교

💡 S-커브의 활용:
✅ 자금 조달 계획
✅ 현금 흐름 관리
✅ 진척도 시각화
✅ 예산 초과 조기 발견

실무 예시:
"은행에서 프로젝트 대출을 받을 때 S-커브를 제시합니다.

4️⃣ 자금 조달 요구사항
- 월별 필요 자금
- 지급 조건 (Net 30, Net 60)
- 현금 여유분 (Buffer)

```

#### [04:00-05:00] 3.2 EVM (성과 가치 관리) (60분)

**강의 내용**:
```
📊 EVM (Earned Value Management) 이해하기

🎯 EVM이란?
"프로젝트의 범위, 일정, 원가를 통합하여 
성과를 객관적으로 측정하는 기법"

❓ 왜 필요한가?
"PM: 프로젝트 50% 완료했습니다!"
"경영진: 정말? 예산은 70% 썼는데?"
→ EVM으로 객관적 측정 가능

📌 EVM 핵심 개념 3가지

1️⃣ PV (Planned Value, 계획 가치)
= "지금까지 얼마를 쓸 계획이었나?"
= 예산 × 계획 진척률
= BAC × 계획 % 완료

예: 전체 예산 1억, 계획상 50% 완료 시점
→ PV = 1억 × 50% = 5,000만원

2️⃣ AC (Actual Cost, 실제 원가)
= "지금까지 실제로 얼마를 썼나?"
= 실제 지출 금액

예: 실제로 6,000만원 지출
→ AC = 6,000만원

3️⃣ EV (Earned Value, 성과 가치)
= "완료한 작업의 가치는 얼마인가?"
= 예산 × 실제 진척률
= BAC × 실제 % 완료

예: 전체 예산 1억, 실제 40% 완료
→ EV = 1억 × 40% = 4,000만원

💡 기억법:
"PV는 계획, AC는 실제 돈, EV는 실제 성과"

📊 EVM 성과 지표 계산

1️⃣ 분산 (Variance) 분석

CV (Cost Variance, 원가 차이)
= EV - AC
= 성과 - 실제 비용
• 양수(+): 예산 내 (좋음 ✅)
• 음수(-): 예산 초과 (나쁨 ❌)

예: CV = 4,000 - 6,000 = -2,000만원
→ 2,000만원 초과 지출

SV (Schedule Variance, 일정 차이)
= EV - PV
= 성과 - 계획 가치
• 양수(+): 일정 앞섬 (좋음 ✅)
• 음수(-): 일정 지연 (나쁨 ❌)

예: SV = 4,000 - 5,000 = -1,000만원
→ 계획보다 뒤처짐

2️⃣ 성과 지수 (Performance Index)

CPI (Cost Performance Index, 원가 성과 지수)
= EV / AC
= 투입 비용 대비 성과
• CPI > 1.0: 효율적 (좋음 ✅)
• CPI < 1.0: 비효율적 (나쁨 ❌)
• CPI = 1.0: 계획대로

예: CPI = 4,000 / 6,000 = 0.67
→ 1원 쓸 때마다 67전 가치만 창출 (비효율)

SPI (Schedule Performance Index, 일정 성과 지수)
= EV / PV
= 계획 대비 실제 성과
• SPI > 1.0: 일정 앞섬 (좋음 ✅)
• SPI < 1.0: 일정 지연 (나쁨 ❌)
• SPI = 1.0: 계획대로

예: SPI = 4,000 / 5,000 = 0.80
→ 계획의 80% 속도로 진행 (지연)

3️⃣ 예측 (Forecasting)

EAC (Estimate at Completion, 완료 시 총 원가)
= 프로젝트 완료 시 총 비용 예측

공식 4가지:

1) 전형적인 상황 (CPI 추세 지속)
EAC = BAC / CPI
예: EAC = 1억 / 0.67 = 1.49억

2) 일회성 문제 (앞으로는 계획대로)
EAC = AC + (BAC - EV)
예: EAC = 6,000 + (1억 - 4,000) = 1.2억

3) 일정과 원가 모두 고려
EAC = AC + [(BAC - EV) / (CPI × SPI)]

4) 완전 재산정
EAC = AC + 새로운 하향식 산정

💡 실무에서는 1번 공식을 가장 많이 사용

ETC (Estimate to Complete, 완료까지 남은 원가)
= EAC - AC
= 앞으로 얼마나 더 필요한가?

예: ETC = 1.49억 - 6,000 = 8,900만원

VAC (Variance at Completion, 완료 시 차이)
= BAC - EAC
= 예산 대비 차이 (초과/절감)

예: VAC = 1억 - 1.49억 = -4,900만원
→ 4,900만원 초과 예상

TCPI (To-Complete Performance Index)
= (BAC - EV) / (BAC - AC)
= 남은 작업을 효율적으로 하려면?

예: TCPI = (1억 - 4,000) / (1억 - 6,000)
     = 6,000 / 4,000 = 1.5
→ 남은 기간 동안 CPI 1.5로 일해야 예산 준수
→ 현재 CPI 0.67인데 1.5로 올리기는 거의 불가능!

📊 EVM 종합 예제

프로젝트 상황:
- BAC (총 예산): 1억원
- 프로젝트 기간: 10개월
- 현재: 5개월 경과
- 계획 진척률: 50%
- 실제 진척률: 40%
- 실제 지출: 6,000만원

계산:
PV = 1억 × 50% = 5,000만원
EV = 1억 × 40% = 4,000만원
AC = 6,000만원

CV = 4,000 - 6,000 = -2,000만원 (초과 ❌)
SV = 4,000 - 5,000 = -1,000만원 (지연 ❌)

CPI = 4,000 / 6,000 = 0.67 (비효율 ❌)
SPI = 4,000 / 5,000 = 0.80 (지연 ❌)

EAC = 1억 / 0.67 = 1.49억 (4,900만원 초과 예상 ❌)
ETC = 1.49억 - 6,000 = 8,900만원
VAC = 1억 - 1.49억 = -4,900만원

TCPI = 6,000 / 4,000 = 1.5 (매우 어려움 ❌)

📢 PM의 판단:
"현재 프로젝트는 심각한 상황입니다!
- 예산 2,000만원 초과
- 일정 20% 지연
- 이대로면 4,900만원 초과 예상
- 회복하려면 효율을 2배 이상 올려야 함

즉시 조치 필요:
1. 범위 축소 검토
2. 추가 예산 요청
3. 고효율 인력 투입
4. 불필요한 작업 제거"

💡 EVM의 장점:
✅ 객관적 성과 측정
✅ 조기 경보 시스템
✅ 데이터 기반 의사결정
✅ 이해관계자 보고 자료
✅ 추세 파악 가능

💡 EVM의 한계:
❌ 초기 설정 복잡
❌ 정확한 진척률 필요
❌ 문서 작업 증가
❌ 애자일에는 부적합
```

**교수법**:
- 화이트보드에 공식 정리
- 각 지표마다 해석 방법 강조
- 실제 계산 과정 단계별 시연
- 학습자들이 따라 계산하도록 유도

### 교수법 포인트
- 💰 **돈 얘기는 구체적으로**: 추상적 숫자보다 실제 금액 예시
- 📊 **시각화 중요**: 그래프, 표, 색상 활용
- 🧮 **계산 연습 필수**: EVM은 반복 연습으로 체득
- 🎭 **실무 사례 풍부하게**: 예산 초과 실패 사례, 성공 사례
- 💬 **토론 유도**: 예산 관리 경험 공유

### 자주 하는 질문
1. **Q: "EVM은 모든 프로젝트에 적용하나요?"**
   A: 대형 프로젝트, 공공 프로젝트에 필수. 소규모는 간소화 가능.

2. **Q: "진척률은 어떻게 정확하게 측정하나요?"**
   A: 작업 완료 기준을 명확히 정의. 0-100 규칙 또는 50-50 규칙 사용.

3. **Q: "CPI가 낮으면 어떻게 개선하나요?"**
   A: 불필요한 작업 제거, 고효율 인력 투입, 외주 재협상 등.

4. **Q: "예비비는 누가 통제하나요?"**
   A: 우발 예비비는 PM, 관리 예비비는 경영진.

### 주의사항
- ⚠️ EVM 공식이 많아 혼란스러울 수 있음 → 핵심 공식 먼저 확실히
- ⚠️ 계산 실수 많이 발생 → 단위 통일, 검산 강조
- ⚠️ 이론만 배우면 지루 → 실습 비중 높게
- ⚠️ Excel 실습 시 일부 학습자 뒤처질 수 있음 → 서로 도와주기 유도
