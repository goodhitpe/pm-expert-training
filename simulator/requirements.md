# 프로젝트 라이프사이클 시뮬레이터 요구사항 정의서

## 1. 목표
- PM 커리큘럼(이해관계자, 범위/일정/비용/품질/리스크, 종료 회고)을 통합한 웹 기반 게이미피케이션 학습 도구를 제공한다.
- 학습자는 준비(Planning) 단계에서 커뮤니케이션을 통해 RFP 및 사업 추진안 품질을 끌어올리고,
  실행(Execution) 단계에서 주차별 돌발 이슈에 대응해 프로젝트 성과를 관리하며,
  종료(Closing) 단계에서 최종 성과를 리뷰한다.

## 2. 범위
### 포함
- HTML5 + CSS + Vanilla JS 단일 페이지 앱
- 브라우저 Web Storage(localStorage) 기반 상태 저장
- 3단계 시뮬레이션
  1) 준비: 이해관계자 소통 및 문서 준비도 향상
  2) 실행: 규모 설정 + 주차별 이벤트 의사결정
  3) 종료: KPI 종합 평가 및 재시작
- 의사결정 로그(최근 30개) 제공

### 제외
- 백엔드 서버, 외부 DB
- 멀티 유저 동시 플레이
- 인증/권한

## 3. 핵심 사용자 시나리오
1. 사용자는 시뮬레이터에 접속해 준비 단계에서 이해관계자와 대화를 수행한다.
2. RFP/사업 추진안/정렬 지수가 기준을 넘으면 실행 단계로 전환한다.
3. 실행 시작 시 프로젝트 규모(small/medium/large)를 선택한다.
4. 각 주차 이벤트에서 액션을 하나 선택하고 KPI 변화 결과를 확인한다.
5. 8주차 종료 후 종료 단계에서 종합 점수와 등급을 확인한다.
6. 저장/초기화/재시작 기능으로 반복 학습한다.

## 4. 기능 요구사항
- FR-01: 시스템은 현재 KPI(범위, 비용, 일정, 품질, 이해관계자 정렬, 문서 준비도)를 표시해야 한다.
- FR-02: 준비 단계에서 최소 5명의 이해관계자 소통 액션을 제공해야 한다.
- FR-03: 각 소통 액션은 RFP/사업 추진안/정렬/품질/비용 등의 지표에 영향을 줘야 한다.
- FR-04: 준비 단계 전환 조건(RFP≥70, 사업 추진안≥70, 정렬≥60)을 충족해야 실행으로 이동 가능해야 한다.
- FR-05: 실행 단계 시작 시 프로젝트 규모 선택 기능을 제공해야 한다.
- FR-06: 규모 선택 후 8개 주차 이벤트를 순차적으로 제공해야 한다.
- FR-07: 각 이벤트는 최소 3개 대응 액션을 제공하고, 선택 즉시 KPI를 갱신해야 한다.
- FR-08: 이벤트 선택 기록은 시간과 함께 로그에 저장해야 한다.
- FR-09: 모든 단계 상태는 localStorage 키 `pmLifecycleSimulator.v1`로 저장되어 새로고침 후 복원되어야 한다.
- FR-10: 종료 단계에서 종합 건전성 점수 및 등급(A/B/C/D)을 제공해야 한다.

## 5. 비기능 요구사항
- NFR-01: 최신 크롬 브라우저 기준 데스크톱/태블릿 반응형 레이아웃 지원.
- NFR-02: 외부 라이브러리 의존성 없이 실행 가능해야 한다.
- NFR-03: 사용자 액션 후 100ms 이내 화면 상태가 반영되어야 한다.
- NFR-04: 로컬 단일 파일 정적 호스팅(`python -m http.server`)으로 구동 가능해야 한다.

## 6. 데이터 모델
```text
SimulatorState {
  phase: "preparation" | "execution" | "closure"
  week: number
  configured: boolean
  preparedStakeholders: number[]
  rfp, businessCase, alignment, scope, cost, schedule, quality: number(0~100)
  projectScale: "small" | "medium" | "large" | null
  log: string[] (max 30)
}
```

## 7. 검증 기준
- AC-01: 준비 단계에서 동일 이해관계자를 중복 실행할 수 없다.
- AC-02: 실행 단계에서 각 주차는 1회만 선택 가능하며, 선택 즉시 다음 주차로 이동한다.
- AC-03: KPI 수치는 0~100 범위를 벗어나지 않는다.
- AC-04: 초기화 버튼 클릭 시 기본 상태로 돌아가야 한다.
- AC-05: 저장 버튼 클릭 시 새로고침 후도 직전 상태를 유지해야 한다.
