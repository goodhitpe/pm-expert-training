## 📋 객관식 문제 (10문제)

### 문제 1
**질문**: 데이터베이스 관리 시스템(DBMS)을 사용하는 주요 이점이 **아닌** 것은?

A) 데이터 중복 최소화  
B) 데이터 무결성 보장  
C) 개발 속도 저하  
D) 동시성 제어

**정답**: C

**해설**: DBMS는 데이터 중복 최소화, 무결성 보장, 동시성 제어, 보안 강화 등의 장점을 제공하며, 체계적인 데이터 관리를 통해 오히려 개발 생산성을 향상시킵니다. 개발 속도 저하는 DBMS의 단점이 아닙니다.

**관련 학습 내용**: Week 14 > 데이터베이스 개요 > 데이터베이스란?

### 문제 2
**질문**: 관계형 데이터베이스에서 각 행을 고유하게 식별하는 키는?

A) 외래 키 (Foreign Key)  
B) 기본 키 (Primary Key)  
C) 복합 키 (Composite Key)  
D) 후보 키 (Candidate Key)

**정답**: B

**해설**: 기본 키(Primary Key)는 테이블의 각 행(레코드)을 고유하게 식별하는 컬럼 또는 컬럼의 조합입니다. 기본 키는 NULL 값을 가질 수 없으며, 테이블당 하나만 존재합니다.

**관련 학습 내용**: Week 14 > 관계형 데이터베이스 > 관계형 모델

### 문제 3
**질문**: SQL의 DDL(Data Definition Language)에 해당하는 명령어는?

A) SELECT  
B) INSERT  
C) CREATE  
D) COMMIT

**정답**: C

**해설**: DDL은 데이터베이스 구조를 정의하는 언어로 CREATE, ALTER, DROP이 포함됩니다. SELECT, INSERT는 DML(Data Manipulation Language), COMMIT은 TCL(Transaction Control Language)에 해당합니다.

**관련 학습 내용**: Week 14 > SQL > SQL 종류

### 문제 4
**질문**: 다음 SQL 문의 결과로 **가장 적절한** 것은?
```sql
SELECT * FROM employees WHERE department = 'IT' ORDER BY salary DESC LIMIT 5;
```

A) IT 부서 전체 직원을 급여 오름차순으로 정렬  
B) IT 부서에서 급여가 가장 높은 5명  
C) 전체 직원 중 급여 상위 5명  
D) IT 부서 직원 중 무작위 5명

**정답**: B

**해설**: 이 쿼리는 IT 부서(WHERE) 직원을 급여 내림차순으로 정렬(ORDER BY DESC)하여 상위 5명만 조회(LIMIT 5)합니다. 따라서 IT 부서에서 급여가 가장 높은 5명의 정보를 반환합니다.

**관련 학습 내용**: Week 14 > SQL > 기본 SQL 문법

### 문제 5
**질문**: ER 다이어그램에서 부서와 직원의 관계가 "1:N"일 때의 의미는?

A) 한 부서는 한 명의 직원만 가질 수 있다  
B) 한 부서는 여러 명의 직원을 가질 수 있다  
C) 한 직원은 여러 부서에 소속될 수 있다  
D) 부서와 직원은 관계가 없다

**정답**: B

**해설**: 1:N(일대다) 관계는 한쪽 엔티티(부서)가 여러 개의 다른 엔티티(직원)와 관계를 가질 수 있음을 의미합니다. 한 부서에는 여러 직원이 소속되지만, 한 직원은 하나의 부서에만 소속됩니다.

**관련 학습 내용**: Week 14 > 관계형 데이터베이스 > 관계의 종류

### 문제 6
**질문**: 데이터베이스 정규화(Normalization)의 주요 목적은?

A) 데이터 조회 속도를 최대한 높이기 위해  
B) 데이터 중복을 최소화하고 무결성을 향상시키기 위해  
C) 데이터베이스 크기를 늘리기 위해  
D) 외래 키를 제거하기 위해

**정답**: B

**해설**: 정규화는 데이터 중복을 최소화하고 무결성을 향상시키는 것이 주요 목적입니다. 정규화를 통해 이상 현상(Anomaly)을 방지하고 데이터의 일관성을 유지할 수 있습니다. 경우에 따라 조회 성능을 위해 의도적으로 역정규화를 수행하기도 합니다.

**관련 학습 내용**: Week 14 > 데이터 모델링 > 정규화

### 문제 7
**질문**: ACID 특성 중 "A"가 의미하는 것은?

A) Availability (가용성)  
B) Atomicity (원자성)  
C) Authentication (인증)  
D) Authorization (권한)

**정답**: B

**해설**: ACID는 트랜잭션의 4가지 특성으로 A(Atomicity: 원자성), C(Consistency: 일관성), I(Isolation: 격리성), D(Durability: 지속성)을 의미합니다. 원자성은 트랜잭션의 모든 작업이 완전히 실행되거나 전혀 실행되지 않아야 함을 의미합니다.

**관련 학습 내용**: Week 14 > 트랜잭션 > ACID 특성

### 문제 8
**질문**: NoSQL 데이터베이스가 RDBMS보다 유리한 상황은?

A) 복잡한 조인이 많이 필요한 경우  
B) ACID 트랜잭션이 필수적인 경우  
C) 대용량 데이터와 빠른 읽기/쓰기가 필요한 경우  
D) 고정된 스키마가 필요한 경우

**정답**: C

**해설**: NoSQL은 유연한 스키마, 수평적 확장(Scale-out), 빠른 읽기/쓰기 성능이 장점입니다. 대용량 데이터를 빠르게 처리해야 하는 경우에 적합합니다. 복잡한 조인이나 ACID 트랜잭션이 중요한 경우에는 RDBMS가 더 적합합니다.

**관련 학습 내용**: Week 14 > NoSQL 데이터베이스 > RDBMS vs NoSQL

### 문제 9
**질문**: 인덱스(Index) 사용의 단점으로 **가장 적절한** 것은?

A) 조회 속도가 느려진다  
B) 데이터 무결성이 저하된다  
C) 삽입/수정/삭제 성능이 저하된다  
D) 데이터 중복이 증가한다

**정답**: C

**해설**: 인덱스는 조회 속도를 향상시키지만, 인덱스를 유지하기 위해 추가적인 저장 공간이 필요하고 데이터 삽입, 수정, 삭제 시 인덱스도 함께 업데이트해야 하므로 쓰기 성능이 저하됩니다. 따라서 인덱스는 조회가 많은 컬럼에 선택적으로 생성해야 합니다.

**관련 학습 내용**: Week 14 > 데이터베이스 성능 최적화 > 인덱스

### 문제 10
**질문**: PM이 데이터베이스 프로젝트에서 고려해야 할 사항으로 **가장 거리가 먼** 것은?

A) 용량 계획 및 증가율 예측  
B) 데이터 마이그레이션 계획  
C) 모든 SQL 쿼리의 최적화  
D) 백업 및 복구 전략

**정답**: C

**해설**: PM은 전체적인 데이터베이스 설계, 용량 계획, 마이그레이션 전략, 백업/복구 계획 등을 관리해야 하지만, 모든 SQL 쿼리를 직접 최적화하는 것은 데이터베이스 관리자(DBA)나 개발자의 역할입니다. PM은 성능 목표를 설정하고 모니터링하는 것이 중요합니다.

**관련 학습 내용**: Week 14 > PM이 알아야 할 데이터베이스 지식
