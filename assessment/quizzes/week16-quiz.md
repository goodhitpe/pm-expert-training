## 📋 객관식 문제 (10문제)

### 문제 1
**질문**: OSI 7계층 모델에서 IP 프로토콜이 동작하는 계층은?

A) 데이터 링크 계층 (Layer 2)  
B) 네트워크 계층 (Layer 3)  
C) 전송 계층 (Layer 4)  
D) 응용 계층 (Layer 7)

**정답**: B

**해설**: IP(Internet Protocol)는 OSI 7계층의 네트워크 계층(Layer 3)에서 동작합니다. 네트워크 계층은 패킷 라우팅과 IP 주소 지정을 담당합니다. 전송 계층(Layer 4)에는 TCP/UDP가, 응용 계층(Layer 7)에는 HTTP/FTP 등이 있습니다.

**관련 학습 내용**: Week 16 > OSI 7계층 모델 > 각 계층 설명

### 문제 2
**질문**: TCP와 UDP의 차이점으로 **가장 적절한** 것은?

A) TCP는 비연결 지향, UDP는 연결 지향  
B) TCP는 신뢰성 보장, UDP는 신뢰성 미보장  
C) TCP가 UDP보다 빠르다  
D) TCP는 포트 번호를 사용하지 않는다

**정답**: B

**해설**: TCP는 연결 지향 프로토콜로 데이터의 순서와 무결성을 보장하는 신뢰성 있는 전송을 제공합니다. UDP는 비연결 지향으로 빠르지만 신뢰성을 보장하지 않습니다. TCP는 웹, 이메일 등에, UDP는 스트리밍, VoIP 등에 사용됩니다.

**관련 학습 내용**: Week 16 > TCP/IP 모델 > 주요 프로토콜 > 전송 계층

### 문제 3
**질문**: IPv4 주소 중 사설 IP(Private IP) 주소 범위가 **아닌** 것은?

A) 10.0.0.0 ~ 10.255.255.255  
B) 172.16.0.0 ~ 172.31.255.255  
C) 192.168.0.0 ~ 192.168.255.255  
D) 200.100.0.0 ~ 200.100.255.255

**정답**: D

**해설**: 사설 IP 주소는 인터넷에서 라우팅되지 않으며 내부 네트워크에서만 사용됩니다. Class A(10.0.0.0/8), Class B(172.16.0.0/12), Class C(192.168.0.0/16)가 사설 IP 범위입니다. 200.x.x.x는 공인 IP 주소 범위입니다.

**관련 학습 내용**: Week 16 > IP 주소 > IPv4 > 사설 IP

### 문제 4
**질문**: DNS(Domain Name System)의 주요 역할은?

A) 데이터 암호화  
B) 도메인 이름을 IP 주소로 변환  
C) 네트워크 트래픽 분산  
D) 바이러스 검사

**정답**: B

**해설**: DNS는 사람이 읽을 수 있는 도메인 이름(예: www.example.com)을 컴퓨터가 사용하는 IP 주소(예: 93.184.216.34)로 변환하는 시스템입니다. 인터넷의 전화번호부와 같은 역할을 합니다.

**관련 학습 내용**: Week 16 > DNS > DNS 개요

### 문제 5
**질문**: HTTP 상태 코드 중 "404"가 의미하는 것은?

A) 요청 성공  
B) 서버 내부 오류  
C) 리소스를 찾을 수 없음  
D) 인증 필요

**정답**: C

**해설**: 404 Not Found는 클라이언트가 요청한 리소스를 서버에서 찾을 수 없음을 의미합니다. 200은 성공, 500은 서버 내부 오류, 401은 인증 필요를 나타냅니다. PM은 API 개발 시 적절한 HTTP 상태 코드 사용을 이해해야 합니다.

**관련 학습 내용**: Week 16 > HTTP/HTTPS > HTTP 상태 코드

### 문제 6
**질문**: HTTPS가 HTTP에 비해 제공하는 추가 보안 기능은?

A) 더 빠른 속도  
B) 데이터 암호화  
C) 더 많은 트래픽 처리  
D) 자동 백업

**정답**: B

**해설**: HTTPS는 HTTP에 TLS/SSL 암호화를 추가하여 데이터를 안전하게 전송합니다. 클라이언트와 서버 간의 통신을 암호화하여 중간자 공격을 방지하고, 인증서를 통해 서버의 신원을 확인합니다. 현대 웹에서는 HTTPS 사용이 표준입니다.

**관련 학습 내용**: Week 16 > HTTP/HTTPS > HTTPS

### 문제 7
**질문**: 방화벽(Firewall)의 주요 기능은?

A) 데이터베이스 관리  
B) 네트워크 트래픽 필터링 및 접근 제어  
C) 웹 페이지 호스팅  
D) 이메일 발송

**정답**: B

**해설**: 방화벽은 정의된 보안 규칙에 따라 네트워크 트래픽을 필터링하고 차단하거나 허용하는 보안 장비/소프트웨어입니다. 인바운드/아웃바운드 트래픽을 제어하여 네트워크를 외부 위협으로부터 보호합니다.

**관련 학습 내용**: Week 16 > 네트워크 보안 > 방화벽

### 문제 8
**질문**: 로드 밸런서(Load Balancer)의 주요 목적은?

A) 데이터 백업  
B) 서버 간 트래픽 분산  
C) 도메인 이름 변환  
D) 데이터 암호화

**정답**: B

**해설**: 로드 밸런서는 들어오는 네트워크 트래픽을 여러 서버에 분산시켜 부하를 균등하게 배분합니다. 이를 통해 단일 서버의 과부하를 방지하고, 고가용성과 확장성을 제공하며, 서버 장애 시 자동으로 다른 서버로 트래픽을 전환합니다.

**관련 학습 내용**: Week 16 > 네트워크 장비 > 로드 밸런서

### 문제 9
**질문**: VPN(Virtual Private Network)의 주요 용도로 **가장 적절한** 것은?

A) 웹사이트 속도 향상  
B) 암호화된 원격 접속  
C) 데이터베이스 백업  
D) 이메일 필터링

**정답**: B

**해설**: VPN은 공용 네트워크(인터넷)를 통해 암호화된 터널을 생성하여 안전한 원격 접속을 제공합니다. 재택근무자가 회사 네트워크에 안전하게 접속하거나, 지사 간 네트워크를 연결할 때 사용됩니다.

**관련 학습 내용**: Week 16 > 네트워크 보안 > VPN

### 문제 10
**질문**: PM이 네트워크 프로젝트에서 이해해야 하는 주요 지표로 **가장 거리가 먼** 것은?

A) 대역폭 (Bandwidth)  
B) 지연 시간 (Latency)  
C) 각 패킷의 비트 순서  
D) SLA (Service Level Agreement)

**정답**: C

**해설**: PM은 대역폭, 지연 시간, SLA 등 네트워크 성능과 관련된 주요 지표를 이해해야 합니다. 그러나 각 패킷의 비트 순서와 같은 저수준의 기술적 세부사항은 네트워크 엔지니어의 영역이며, PM이 직접 관리할 필요는 없습니다.

**관련 학습 내용**: Week 16 > PM이 알아야 할 네트워크 지식 > 네트워크 프로젝트 관리
