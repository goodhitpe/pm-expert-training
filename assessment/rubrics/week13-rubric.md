# Week 13: 소프트웨어 공학 - 평가 루브릭

> **평가 대상**: SDLC 모델 비교 보고서 + Git 실습 + CI/CD 파이프라인 설계  
> **제출 형식**: 보고서 (A4 1페이지) + Git 스크린샷 + 파이프라인 다이어그램  
> **제출 기한**: Week 13 종료 시점

---

## 📋 실습 과제 설명

**과제명**: PM 관점의 소프트웨어 공학 이해

**과제 내용**:
PM으로서 소프트웨어 프로젝트를 관리하기 위해 필요한 기본 기술 지식을 습득합니다. 깊은 기술 전문성보다는 개발자와 소통하고, 프로젝트 계획을 수립하며, 리스크를 관리하는 데 필요한 개념 이해에 집중합니다.

**요구사항**:
1. **과제 1: SDLC 모델 비교** (A4 1페이지)
   - 폭포수, 애자일, DevOps 모델의 장단점 비교
   - 각 모델이 적합한 프로젝트 유형
   - PM 관점의 선택 기준

2. **과제 2: Git 실습**
   - GitHub 저장소 생성
   - 기본 Git 명령어 실습 (init, add, commit, push)
   - 브랜치 생성 및 병합
   - 스크린샷 제출 (커밋 히스토리, 브랜치 구조)

3. **과제 3: CI/CD 파이프라인 설계**
   - 선정한 프로젝트의 CI/CD 파이프라인 설계
   - 각 단계별 작업 정의
   - 다이어그램 작성

---

## 📊 평가 항목 (총 100점)

### 1. SDLC 모델 이해 및 비교 (40점)
- 폭포수, 애자일, DevOps의 특성 정확한 이해
- 각 모델의 장단점을 프로젝트 관리 관점에서 분석
- 적합한 프로젝트 유형 제시의 현실성
- PM의 선택 기준과 의사결정 근거

### 2. Git 실습 및 버전 관리 이해 (30점)
- Git 기본 명령어의 올바른 사용
- 브랜치 전략의 이해
- 커밋 히스토리의 적절성
- 버전 관리의 필요성 이해

### 3. CI/CD 파이프라인 설계 (30점)
- 파이프라인 구조의 논리성
- 각 단계의 명확성 및 완성도
- PM 관점의 실용성
- 다이어그램의 명확성 및 전문성

## 💡 피드백 예시

### 탁월한 답변 예시 (90-100점)
**강점**:
- SDLC 모델 비교에서 PM의 역할 변화를 명확히 분석했습니다. 폭포수에서는 초기 계획이 중요하고, 애자일에서는 스프린트 관리와 이해관계자 참여가 핵심임을 잘 설명했습니다.
- Git 실습에서 feature 브랜치를 생성하여 기능 개발을 시뮬레이션하고, main 브랜치에 병합하는 전체 흐름을 실습했습니다.
- CI/CD 파이프라인 설계에서 품질 게이트와 PM의 승인 단계를 포함하여, PM 관점을 잘 반영했습니다.
- 각 단계별로 SonarQube(정적 분석), JUnit(단위 테스트), Selenium(E2E 테스트) 등 구체적인 도구를 명시했습니다.

**추가 학습 제안**:
- 실제 오픈소스 프로젝트의 GitHub를 탐색하여 브랜치 전략(Git Flow, GitHub Flow)을 관찰해보세요.
- Jenkins, GitHub Actions, GitLab CI 등 CI/CD 도구의 무료 버전을 직접 체험해보세요.

---

### 우수한 답변 예시 (75-89점)
**강점**:
- SDLC 모델의 특성을 잘 이해하고 있습니다.
- Git 기본 실습을 성공적으로 완료했습니다.
- CI/CD 파이프라인의 기본 구조를 적절히 설계했습니다.

**개선 필요 사항**:
- SDLC 모델 비교에서 "애자일이 더 유연하다"는 언급만 있고, 구체적으로 어떤 상황에서 왜 유연한지 설명이 부족합니다. 
  - 예: "요구사항 변경 시 폭포수는 전체 계획 재수립이 필요하지만, 애자일은 다음 스프린트에 반영 가능"
- Git 실습에서 브랜치명이 "test", "branch1" 등 의미 없는 이름입니다. 실무에서는 "feature/user-login", "bugfix/issue-123" 같은 명명 규칙을 사용하세요.
- CI/CD 파이프라인에 테스트 단계는 있으나, 테스트 실패 시 어떻게 처리할지(롤백, 알림 등) 설명이 없습니다.

---

### 보통 수준 답변 예시 (60-74점)
**강점**:
- 기본적인 과제 요구사항을 충족했습니다.
- Git 저장소를 생성하고 기본 명령어를 실습했습니다.

**개선 필요 사항**:
- SDLC 모델 설명이 교과서적입니다. Week 13 강의의 "PM이 알아야 할 소프트웨어 개발 지식" 섹션을 다시 읽고, PM 관점을 추가하세요.
  - 예: "폭포수는 초기 계획 수립에 시간이 많이 소요되므로, 프로젝트 헌장과 범위 기술서 작성이 매우 중요합니다."
- Git 실습에서 브랜치 병합을 수행하지 않았습니다. Week 13 강의의 Git 명령어를 참고하여 `git merge`를 실습하세요.
- CI/CD 파이프라인이 너무 단순합니다(코드 커밋 → 배포). 최소한 빌드, 테스트 단계를 추가하세요.

**도움이 될 자료**:
- Week 13 강의 자료: Git 브랜치 전략, CI/CD 파이프라인
- GitHub Learning Lab: https://lab.github.com/
- Git 공식 문서: https://git-scm.com/book/ko/v2

### 부족한 답변 예시 (0-59점)
**주요 문제점**:
- SDLC 모델에 대한 이해가 부족합니다. 폭포수, 애자일, DevOps의 차이를 명확히 설명하지 못했습니다.
- Git 실습이 미완료되었습니다. 저장소 생성만 하고 커밋이나 브랜치 작업이 없습니다.
- CI/CD 파이프라인이 개념적으로만 언급되고, 구체적인 설계가 없습니다.

**즉시 해야 할 일**:
1. Week 13 강의 자료를 처음부터 끝까지 정독하세요.
2. SDLC 각 모델의 특성을 비교표로 작성하세요.
3. Git 튜토리얼을 따라하며 기본 명령어를 익히세요:
   - Try Git: https://try.github.io/
4. 간단한 웹 애플리케이션 프로젝트를 예로 들어 CI/CD 파이프라인을 다시 설계하세요.
5. 강사와 1:1 미팅을 통해 부족한 개념을 보충하세요.

**재제출 권장**:
현재 수준으로는 소프트웨어 프로젝트 관리에 어려움이 있을 수 있습니다. 위 학습을 완료한 후 과제를 재제출하시기 바랍니다.
