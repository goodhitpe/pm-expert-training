# Week 8: 자원 & 조달 관리 - 평가 루브릭

> **평가 대상**: RACI 매트릭스 및 조달 계획 보고서  
> **제출 형식**: A4 2-3페이지 + RACI 매트릭스 (별도)  
> **제출 기한**: Week 8 종료 시점

## 📋 실습 과제 설명

**과제명**: 프로젝트 자원 관리 및 조달 계획 수립

**과제 내용**:
선정한 프로젝트의 주요 활동에 대한 RACI 매트릭스를 작성하고, 외주가 필요한 항목을 식별하여 적절한 계약 유형을 제안하며, 팀 갈등 시나리오에 대한 해결 방안을 제시하는 보고서를 작성하세요.

**요구사항**:
1. 프로젝트 주요 활동 10개 이상에 대한 RACI 매트릭스
2. 외주 필요 항목 식별 (최소 3개)
3. 각 외주 항목에 대한 적절한 계약 유형(고정가/원가정산/T&M) 제안 및 근거
4. 갈등 시나리오 2가지와 해결 기법 적용
5. 자원 평준화 또는 평탄화 필요성 분석 (선택 사항)

## 📊 평가 항목 (총 100점)

### 1. RACI 매트릭스 정확성 (30점)
- RACI 원칙의 올바른 적용
- 역할 배정의 적절성
- A(Accountable)가 각 활동당 1명인지 확인

### 2. 조달 계획 적절성 (25점)
- Make vs Buy 분석의 타당성
- 계약 유형 선택의 적절성
- 공급업체 선정 기준 제시

### 3. 갈등 해결 능력 (25점)
- 갈등 시나리오의 현실성
- 5가지 해결 기법의 적절한 적용
- 해결 방안의 구체성 및 실행 가능성

### 4. 보고서 완성도 (20점)
- 논리적 구성 및 가독성
- 시각 자료의 효과적 활용
- 전문 용어의 정확한 사용

## 💡 피드백 예시

### 탁월한 답변 예시 (90-100점)
**강점**:
- RACI 매트릭스에서 "요구사항 분석" 활동에 PM을 A(최종 책임), 개발 리드를 R(실행), 고객을 C(사전 협의), 경영진을 I(사후 통보)로 명확히 배정했습니다.
- "클라우드 인프라 구축" 외주에 고정가 계약(FP)을 제안하고, "범위가 명확하고 기술이 검증되어 가격 예측 가능하며, 리스크를 공급업체에 전가"라는 타당한 근거를 제시했습니다.
- 갈등 시나리오 "개발팀과 디자인팀의 일정 충돌"에 협력(Collaborate) 기법을 적용하여 "공동 워크샵을 통해 양측의 제약을 공유하고, 스프린트 목표를 재조정하여 Win-Win 해결"이라는 구체적 방안을 제시했습니다.
- 자원 평준화 분석에서 "개발자 3명이 Week 5에 동시 배정되어 과부하 → 일정을 1주 연장하여 자원 분산"하는 실무적 해결책을 포함했습니다.

**추가 학습 제안**:
- Tuckman의 팀 개발 5단계와 연결하여 각 단계별 갈등 유형과 해결 기법을 매핑해보세요.
- 협상 기법(BATNA)을 적용한 조달 시나리오를 추가로 작성해보세요.

### 우수한 답변 예시 (75-89점)
**강점**:
- RACI 매트릭스를 체계적으로 작성했습니다.
- 조달 항목과 계약 유형을 적절히 제안했습니다.
- 갈등 해결 기법을 올바르게 적용했습니다.

**개선 필요 사항**:
- RACI에서 "프로젝트 킥오프" 활동에 A가 2명(PM, 스폰서)입니다. A는 항상 1명이어야 합니다. 스폰서는 I(통보)가 더 적절합니다.
- "UI/UX 디자인" 외주에 T&M 계약을 제안했는데, 범위가 명확하다면 고정가(FP) 계약이 더 적절할 수 있습니다. 계약 유형 선택 근거를 강화하세요.
- 갈등 시나리오가 다소 일반적입니다. "팀원 A와 B의 의견 차이"보다는 "프론트엔드 개발자가 API 일정 지연으로 백엔드 팀에 불만 표출"처럼 구체적으로 작성하세요.

### 보통 수준 답변 예시 (60-74점)
**강점**:
- RACI 매트릭스와 조달 계획의 기본 구조를 갖췄습니다.
- 갈등 해결 기법을 시도했습니다.

**개선 필요 사항**:
- RACI에서 R(Responsible)과 A(Accountable)의 차이를 명확히 이해하세요. Week 8 강의 자료를 다시 학습하세요:
  - R: 실제 작업 수행 (여러 명 가능)
  - A: 최종 책임 및 승인 (반드시 1명)
- 계약 유형 선택이 임의적입니다. 고정가(FP), 원가정산(CR), T&M의 차이를 명확히 하세요:
  - FP: 범위 명확, 판매자 리스크
  - CR: 범위 불명확, 구매자 리스크
  - T&M: 중간, 범위 변동 가능
- 갈등 해결 시 왜 특정 기법을 선택했는지 근거를 제시하세요. "협력이 가장 효과적이므로"보다는 "양측 모두 만족할 수 있는 Win-Win 해결이 필요하고, 장기 협력 관계 유지가 중요하므로 협력 기법 선택"이 더 설득력 있습니다.

**도움이 될 자료**:
- Week 8 강의 자료: RACI 예시, 계약 유형 비교표
- PMBOK Guide Chapter 9 (Resource Management), Chapter 12 (Procurement)
- "Getting to Yes" by Roger Fisher (협상 기법)

### 부족한 답변 예시 (0-59점)
**주요 문제점**:
- RACI 원칙을 이해하지 못했습니다.
- 계약 유형을 잘못 적용했습니다.
- 갈등 해결 기법이 부적절합니다.

**즉시 해야 할 일**:
1. Week 8 강의 자료를 처음부터 끝까지 다시 학습하세요.
2. RACI 매트릭스 예제를 따라 작성해보세요:
   ```
   활동: 요구사항 분석
   PM: A (최종 책임)
   개발 리드: R (실행)
   고객: C (협의)
   경영진: I (통보)
   ```
3. 계약 유형 3가지를 명확히 구분하세요:
   - 고정가(FP): 총액 고정, "웹사이트 개발: 5,000만원"
   - 원가정산(CR): 실비 + 수수료, "연구개발: 실비 + 15% 수수료"
   - T&M: 시간당 단가, "컨설팅: 시간당 20만원"
4. 갈등 해결 기법 5가지를 이해하고 효과적 순서를 기억하세요:
   협력 (Win-Win) > 타협 (부분 Win-Win) > 강제 (Win-Lose) > 완화 (일시적) > 회피 (미해결)
5. 강사와 1:1 미팅을 예약하여 개념을 다시 확인하세요.

**재제출 권장**:
자원 관리는 모든 프로젝트의 핵심입니다. RACI는 실무에서 매우 자주 사용되는 도구이므로 확실히 이해해야 합니다. 위 자료를 학습한 후 보고서를 수정하여 재제출하시기 바랍니다.

## ✅ 자가 점검 체크리스트

제출 전에 다음 항목을 확인하세요:

### RACI 매트릭스
- [ ] 활동을 10개 이상 정의했는가?
- [ ] 각 활동에 A(Accountable)가 정확히 1명인가?
- [ ] R(Responsible)이 실제 작업 수행자로 적절히 배정되었는가?
- [ ] C(Consulted)가 사전 협의가 필요한 사람으로 배정되었는가?
- [ ] I(Informed)가 사후 통보 대상으로 적절한가?
- [ ] 역할(PM, 개발자, 디자이너 등)이 구체적으로 정의되었는가?
- [ ] 매트릭스가 표 형태로 보기 쉽게 작성되었는가?

### 조달 계획
- [ ] 외주가 필요한 항목을 3개 이상 식별했는가?
- [ ] 각 항목에 Make vs Buy 분석을 수행했는가?
- [ ] 계약 유형(FP, CR, T&M)을 제안했는가?
- [ ] 계약 유형 선택의 근거를 명시했는가? (범위 명확성, 리스크 등)
- [ ] 공급업체 선정 기준을 3개 이상 제시했는가?

### 갈등 해결
- [ ] 현실적인 갈등 시나리오를 2개 제시했는가?
- [ ] 각 시나리오에 적절한 해결 기법을 적용했는가?
- [ ] 5가지 기법(협력/타협/강제/완화/회피) 중 2개 이상을 활용했는가?
- [ ] 해결 방안이 구체적이고 단계별로 설명되었는가?
- [ ] 해결 기법 선택의 이유를 명시했는가?

### 형식
- [ ] A4 2-3페이지 + RACI 매트릭스 별도인가?
- [ ] 전문 용어(RACI, FP, CR, T&M 등)를 정확히 사용했는가?
- [ ] 맞춤법과 문법을 검토했는가?

### 자주 묻는 질문 (FAQ)

**Q1: RACI에서 A(Accountable)가 2명이면 안 되나요?**
A: 안 됩니다. A는 최종 책임자로 반드시 1명이어야 합니다. "책임자가 많으면 책임자가 없는 것"이라는 원칙입니다. 여러 명이 협력해야 한다면 R(Responsible)로 배정하세요.

**Q2: 모든 역할이 모든 활동에 참여해야 하나요?**
A: 아니요. 관련 없는 역할은 비워두면 됩니다. 예를 들어, "인프라 구축" 활동에 디자이너는 참여하지 않을 수 있습니다.

**Q3: 고정가(FP) 계약과 T&M 계약을 어떻게 구분하나요?**
A: 
- FP: 범위가 명확하고 변경 가능성이 낮을 때. 예: "로고 디자인 10개: 500만원"
- T&M: 범위가 불명확하거나 변동 가능할 때. 예: "UI 개선 컨설팅: 시간당 15만원"

**Q4: 갈등 해결 기법 중 어떤 것이 가장 좋나요?**
A: 상황에 따라 다르지만, 일반적으로 협력(Collaborate)이 가장 효과적입니다:
- 협력: 시간이 있고 장기 관계가 중요할 때 (Win-Win)
- 강제: 긴급하고 PM 권한이 필요할 때 (Win-Lose)
- 회피: 사소하거나 시간이 해결할 문제일 때

**Q5: 자원 평준화(Resource Leveling)와 평탄화(Smoothing)의 차이는?**
A:
- 평준화: 자원 제약이 우선, 일정 연장 가능
- 평탄화: 일정 제약이 우선, 여유 시간 내에서만 조정

**Q6: Make vs Buy를 어떻게 결정하나요?**
A: 다음을 고려하세요:
- Make: 핵심 역량, 장기 필요, 기밀 유지, 비용 효율적
- Buy: 비핵심, 단기 필요, 전문성 부족, 시간 절약
