# Week 10: PM 도구 활용 - 평가 루브릭

> **평가 대상**: PM 도구 실습 및 비교 분석 보고서  
> **제출 형식**: A4 2-3페이지 + 스크린샷/차트 (별도)  
> **제출 기한**: Week 10 종료 시점

---

## 📋 실습 과제 설명

**과제명**: Jira 프로젝트 생성 및 PM 도구 비교 분석

**과제 내용**:
Jira(또는 GitHub Projects)를 사용하여 선정한 프로젝트의 스크럼 보드를 실제로 생성하고, 2개 이상의 PM 도구를 비교 분석하여 프로젝트 특성에 맞는 도구 선택 근거를 제시하는 보고서를 작성하세요.

**요구사항**:
1. Jira 또는 GitHub Projects에서 프로젝트 생성 (스크린샷 필수)
   - Epic 2개 이상 생성
   - 사용자 스토리 10개 이상 생성
   - 스프린트 1개 시작
   - 백로그, 보드, 번다운 차트 캡처
2. PM 도구 2개 이상 비교 분석 (Jira, Trello, MS Project, GitHub Projects, Asana 등)
   - 기능 비교 (애자일 지원, Gantt 차트, 협업 기능 등)
   - 장단점 분석
   - 프로젝트 유형별 적합성
3. 자신의 프로젝트에 가장 적합한 도구 선택 및 근거 제시

---

## 📊 평가 항목 (총 100점)

### 1. 도구 실습 완성도 (35점)
- Jira/GitHub Projects 프로젝트 설정의 정확성
- Epic, Story, Sprint 구성의 적절성
- 스크린샷의 명확성 및 완성도

### 2. 도구 비교 분석 (30점)
- 비교 기준의 적절성 및 객관성
- 장단점 분석의 균형성
- 실무 관점의 통찰력

### 3. 도구 선택 근거 (20점)
- 프로젝트 특성 고려
- 선택 근거의 논리성
- 트레이드오프 이해

### 4. 보고서 완성도 (15점)
- 스크린샷 및 비교표의 효과적 활용
- 논리적 구성 및 가독성
- 전문 용어의 정확한 사용

## 💡 피드백 예시

### 탁월한 답변 예시 (90-100점)
**강점**:
- Jira에서 "모바일 앱 개발" 프로젝트를 생성하고, Epic "사용자 인증"과 "상품 검색"에 각각 5개씩 Story를 배치한 백로그 스크린샷이 명확합니다.
- 스프린트 보드에서 3개 Story가 "In Progress", 2개가 "Done"으로 이동한 실제 작업 흔적이 확인됩니다.
- Jira, Trello, GitHub Projects를 "애자일 지원", "가격", "통합", "학습 곡선", "보고 기능" 5개 기준으로 비교한 표가 객관적이고 명확합니다.
- "소규모 소프트웨어 팀(5명)이므로 GitHub Projects를 선택. 이유: (1) 무료, (2) GitHub 코드 저장소와 완벽 통합, (3) 간단한 칸반 보드로 학습 곡선 낮음. Jira는 대규모 팀에 적합하나 월 7달러×5명 비용이 부담"이라는 구체적 근거를 제시했습니다.

**추가 학습 제안**:
- Week 11-12 모의 프로젝트에서 선택한 도구를 실제로 활용해보세요.
- JQL(Jira Query Language)을 학습하여 고급 필터를 활용해보세요.
- Jira와 Confluence, Slack 통합 방법을 실습해보세요.

### 우수한 답변 예시 (75-89점)
**강점**:
- Jira 프로젝트를 적절히 생성하고 스크린샷을 포함했습니다.
- 2개 도구를 비교하고 선택 근거를 제시했습니다.

**개선 필요 사항**:

- 도구 비교 시 가격 정보를 구체적으로 추가하세요. "Jira: 월 7달러/사용자 (10명까지)", "Trello: 무료 (Power-Up 제한 있음)"처럼 작성하세요.
- 트레이드오프를 명시하세요. "GitHub Projects를 선택했지만, Jira에 비해 보고 기능이 약하다는 단점이 있음. 이는 별도 대시보드로 보완 계획"처럼 설명하세요.

### 보통 수준 답변 예시 (60-74점)
**강점**:
- Jira 프로젝트를 생성했습니다.
- 도구 비교를 시도했습니다.

**개선 필요 사항**:
- Jira에서 스프린트를 시작하지 않았습니다. Week 10 강의 자료의 "Jira 실습: 스크럼 프로젝트" 섹션을 따라 스프린트를 시작하고, Story를 "To Do → In Progress → Done"으로 이동해보세요.
- 번다운 차트 스크린샷이 없습니다. Jira에서 Reports → Burndown Chart로 접근하여 캡처하세요.
- 도구 비교가 "Jira는 좋다", "Trello는 간단하다" 같이 추상적입니다. 구체적인 기준으로 비교하세요:
  - 애자일 지원: Jira는 스프린트, 백로그, 번다운 차트 제공. Trello는 기본 칸반만 지원.
  - 가격: Jira 월 7달러, Trello 무료 (기본 플랜)
  - 학습 곡선: Jira 복잡, Trello 간단

**도움이 될 자료**:
- Week 10 강의 자료: Jira 실습 가이드
- Jira Documentation: https://support.atlassian.com/jira-software-cloud/
- Trello Guide: https://trello.com/guide

### 부족한 답변 예시 (0-59점)
**주요 문제점**:
- Jira 프로젝트를 생성하지 않았거나 스크린샷이 없습니다.
- 도구 비교가 없거나 1개만 분석했습니다.
- 선택 근거가 비논리적입니다.

**즉시 해야 할 일**:
1. Week 10 강의 자료를 처음부터 끝까지 다시 학습하세요.
2. Jira 또는 GitHub Projects에 무료 계정을 만드세요:
   - Jira: https://www.atlassian.com/software/jira/free (10명까지 무료)
   - GitHub Projects: GitHub 계정으로 무료 사용
3. Week 10 강의 자료의 "Jira 실습: 스크럼 프로젝트" 7단계를 따라하세요:
   - 프로젝트 생성 → Epic 생성 → Story 생성 → 백로그 관리 → 스프린트 시작 → 보드 작업 → 스프린트 종료
4. 각 단계마다 스크린샷을 찍으세요.
5. 최소 2개 도구를 직접 사용해보고 차이를 경험하세요.
6. 강사와 1:1 미팅을 예약하여 도구 사용법을 확인하세요.

**재제출 권장**:
PM 도구는 실무에서 매일 사용하는 필수 스킬입니다. Week 11-12 모의 프로젝트에서도 도구를 사용해야 하므로 지금 반드시 익혀야 합니다. 위 단계를 완료한 후 보고서를 수정하여 재제출하시기 바랍니다.

## ✅ 자가 점검 체크리스트

제출 전에 다음 항목을 확인하세요:

### Jira/GitHub Projects 실습
- [ ] 프로젝트를 생성했는가? (Scrum 또는 Kanban)
- [ ] Epic을 2개 이상 생성했는가?
- [ ] 사용자 스토리를 10개 이상 생성했는가? (Summary, Description, Story Points 포함)
- [ ] 스프린트를 시작했는가?
- [ ] Story를 보드에서 이동시켰는가? (To Do → In Progress → Done)
- [ ] 다음 스크린샷을 포함했는가?
  - [ ] 백로그 (Epic과 Story 목록)
  - [ ] 스크럼 보드 (To Do, In Progress, Done 칼럼)
  - [ ] 번다운 차트 (선택 사항)
  - [ ] 프로젝트 설정 화면 (선택 사항)

### 도구 비교 분석
- [ ] PM 도구 2개 이상을 비교했는가?
- [ ] 비교 기준을 3개 이상 정의했는가? (기능, 가격, 학습 곡선, 통합, 확장성 등)
- [ ] 비교표를 작성했는가?
- [ ] 각 도구의 장점 3개 이상, 단점 3개 이상을 제시했는가?
- [ ] 프로젝트 유형별 적합성을 분석했는가? (애자일/전통적, 소규모/대규모)

### 도구 선택 근거
- [ ] 자신의 프로젝트 특성을 명확히 정의했는가? (방법론, 팀 규모, 예산 등)
- [ ] 선택한 도구가 프로젝트 특성에 적합한 이유를 설명했는가?
- [ ] 다른 도구를 선택하지 않은 이유(트레이드오프)를 설명했는가?
- [ ] 도구 도입 계획을 제시했는가? (선택 사항)

### 형식
- [ ] A4 2-3페이지 + 스크린샷 별도인가?
- [ ] 스크린샷이 명확하고 주요 부분이 표시되었는가?
- [ ] 전문 용어(Epic, Story, Sprint, Backlog, Burndown 등)를 정확히 사용했는가?
- [ ] 맞춤법과 문법을 검토했는가?

### 자주 묻는 질문 (FAQ)

**Q1: Jira 무료 버전으로 충분한가요?**
A: 네, Jira Free 플랜(10명까지)으로 과제를 완료할 수 있습니다. Backlog, Sprint, Board, Burndown 차트 모두 무료로 사용 가능합니다.

**Q2: Jira 대신 다른 도구를 사용해도 되나요?**
A: 네, GitHub Projects, Trello, Azure DevOps 등을 사용해도 됩니다. 단, 애자일 기능(Epic, Story, Sprint)을 지원하는 도구여야 합니다.

**Q3: 스프린트를 시작하지 않고 백로그만 있으면 안 되나요?**
A: 스프린트를 시작해야 합니다. 스프린트를 시작해야 번다운 차트, 스프린트 보드 등 실제 스크럼 프로세스를 경험할 수 있습니다.

**Q4: 어떤 도구를 비교해야 하나요?**
A: 자유롭게 선택하세요. 추천 조합:
- 애자일 중심: Jira vs GitHub Projects vs Trello
- 전통적 중심: MS Project vs ProjectLibre
- 하이브리드: Jira vs Monday.com vs Asana

**Q5: 도구 비교 시 실제로 모든 도구를 사용해봐야 하나요?**
A: 가능하면 직접 사용해보는 것이 좋지만, 무료 체험이 어렵다면 공식 문서, 리뷰, 비교 사이트를 참고해도 됩니다. 단, 출처를 명시하세요.

**Q6: 번다운 차트가 보이지 않습니다.**
A: 
1. Jira에서 Reports 메뉴 → Burndown Chart 선택
2. 스프린트를 시작해야 번다운 차트가 생성됩니다
3. Story에 Story Points가 있어야 차트가 정확합니다

**Q7: 도구 통합이란 무엇인가요?**
A: 다른 시스템과의 연동입니다. 예:
- Jira + Confluence (문서화)
- Jira + Slack (알림)
- Jira + GitHub (코드 연동)
- Trello + Google Drive (파일 공유)

**Q8: 가산점은 어떻게 받나요?**
A: 다음을 추가하면 가산점:
- 워크플로우 커스터마이징
- 팀원 추가 및 역할 배정
- 도구 통합 전략 제시
- 팀 교육 및 도입 계획
