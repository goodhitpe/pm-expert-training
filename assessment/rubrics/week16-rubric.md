# Week 16: 네트워크 - 평가 루브릭

> **평가 대상**: 네트워크 진단 + OSI 7계층 설명 + 네트워크 보안 계획  
> **제출 형식**: 진단 보고서 + OSI 설명서 (A4 1페이지) + 보안 계획서  
> **제출 기한**: Week 16 종료 시점

---

## 📋 실습 과제 설명

**과제명**: PM을 위한 네트워크 기초

**과제 내용**:
PM으로서 네트워크 인프라 프로젝트를 관리하거나 네트워크 요구사항이 있는 시스템 개발 프로젝트를 수행하기 위한 기본 지식을 습득합니다. 네트워크 엔지니어 수준이 아닌, PM이 네트워크 개념을 이해하고 요구사항을 정의하며 개발팀 및 인프라팀과 소통하기 위한 실용적 지식에 집중합니다.

**요구사항**:
1. **과제 1: 네트워크 진단**
   - 본인의 네트워크 환경 진단
   - IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버 확인
   - 특정 사이트(예: google.com, naver.com)에 대한 ping 및 traceroute 실행
   - 결과 스크린샷 및 분석

2. **과제 2: OSI 7계층 이해**
   - 특정 웹 페이지 접속 시 OSI 7계층에서 일어나는 일을 단계별로 설명
   - 각 계층의 역할과 사용되는 프로토콜/기술 명시
   - A4 1페이지

3. **과제 3: 네트워크 보안 계획**
   - 소규모 사무실(직원 20명)의 네트워크 보안 계획 수립
   - 방화벽 규칙
   - VPN 설정 (원격 근무자용)
   - 접근 제어
   - 백업 및 복구 계획

### 2. OSI 7계층 이해 (35점)

| 등급 | 점수 | 기준 |
|------|------|------|
| **탁월** | 32-35 | • 웹 페이지 접속 과정을 OSI 7계층으로 명확히 설명<br>  **예: www.example.com 접속**<br>  **계층 7 (응용)**: 브라우저가 HTTP GET 요청 생성<br>  **계층 6 (표현)**: HTTPS인 경우 TLS/SSL로 데이터 암호화<br>  **계층 5 (세션)**: TCP 세션 관리, 쿠키/세션 처리<br>  **계층 4 (전송)**: TCP 3-way 핸드셰이크, 포트 80/443<br>  **계층 3 (네트워크)**: IP 패킷 생성, 라우팅, DNS 조회로 IP 주소 확인<br>  **계층 2 (데이터 링크)**: 프레임 생성, MAC 주소 사용, ARP<br>  **계층 1 (물리)**: 전기 신호 또는 광 신호로 전송<br>• 각 계층에서 사용되는 프로토콜/기술 명시<br>  - 응용: HTTP, HTTPS, DNS<br>  - 전송: TCP, UDP<br>  - 네트워크: IP, ICMP, ARP<br>  - 데이터 링크: Ethernet, Wi-Fi<br>  - 물리: 케이블, 전파<br>• 데이터 흐름을 다이어그램으로 표현<br>• 각 계층의 헤더/트레일러 추가 과정 설명<br>• PM 관점에서 계층별 문제가 프로젝트에 미치는 영향 언급<br>  (예: "전송 계층의 TCP 재전송은 응답 시간 증가") |
| **우수** | 28-31 | • 7계층 모두 설명<br>• 웹 페이지 접속 과정 단계별 설명<br>• 주요 프로토콜 언급<br>  (HTTP, TCP, IP, Ethernet)<br>• 기본 흐름 이해<br>• 논리적 구성<br>• A4 1페이지 |
| **보통** | 22-27 | • 7계층 설명하나 일부 부정확<br>• 접속 과정 간략히 설명<br>• 프로토콜 일부만 언급<br>• 흐름 이해 부족<br>• 구성 미흡 |
| **부족** | 0-21 | • 7계층 이해 부족<br>• 접속 과정 설명 불충분<br>• 프로토콜 언급 없음<br>• 논리적 흐름 없음<br>• 분량 부족 |

## 💡 피드백 예시

### 탁월한 답변 예시 (90-100점)
**강점**:
- 네트워크 진단에서 IP 주소, 서브넷 마스크, 게이트웨이를 정확히 확인했고, 네트워크 주소(192.168.1.0)와 브로드캐스트 주소(192.168.1.255)까지 계산했습니다.
- traceroute 결과를 분석하여 "3번째 홉에서 지연 시간이 증가하는 것은 ISP 라우터의 혼잡"이라고 해석한 것이 우수합니다.
- OSI 7계층 설명에서 각 계층의 역할을 명확히 했고, TCP 3-way 핸드셰이크, DNS 조회, TLS 핸드셰이크 등 구체적인 프로토콜 동작을 설명했습니다.
- 네트워크 보안 계획에서 방화벽 규칙을 인바운드/아웃바운드로 구분하고, 최소 권한 원칙을 적용했습니다.
- VPN 설정에서 인증서 기반 인증을 사용하고, 접근 가능한 리소스를 명시한 것이 현실적입니다.

**추가 학습 제안**:
- Wireshark 같은 패킷 캡처 도구를 사용하여 실제 네트워크 트래픽을 분석해보세요.
- 네트워크 보안 인증(CompTIA Network+, Security+)을 고려해보시면 좋습니다.

### 우수한 답변 예시 (75-89점)
**강점**:
- 네트워크 진단을 성공적으로 수행했고, ping과 traceroute 결과를 제시했습니다.
- OSI 7계층을 이해하고 있으며, 웹 페이지 접속 과정을 설명했습니다.
- 네트워크 보안 계획의 주요 요소를 포함했습니다.

**개선 필요 사항**:
- 네트워크 진단에서 DNS 서버 정보가 누락되었습니다. `ipconfig /all` (Windows) 또는 `cat /etc/resolv.conf` (Linux)로 확인하세요.
- ping 결과에서 RTT(왕복 시간)가 평균 50ms인데, 이것이 양호한지 나쁜지 평가가 없습니다. 일반적으로 국내는 30ms 이하, 해외는 100ms 이하가 양호합니다.
- OSI 7계층 설명에서 계층 6(표현)과 계층 5(세션)의 역할이 모호합니다. 
  - 계층 6: 데이터 형식 변환, 암호화(SSL/TLS)
  - 계층 5: 세션 관리, 연결 유지
- 네트워크 보안 계획에서 VPN 프로토콜이 명시되지 않았습니다. IPsec, OpenVPN, L2TP 등 구체적인 프로토콜을 선택하고 이유를 설명하세요.

### 보통 수준 답변 예시 (60-74점)
**강점**:
- 네트워크 진단을 시도했습니다.
- OSI 7계층의 이름을 알고 있습니다.
- 네트워크 보안의 필요성을 이해하고 있습니다.

**개선 필요 사항**:
- 네트워크 진단에서 IP 주소만 제시하고, 서브넷 마스크와 게이트웨이가 누락되었습니다. Week 16 강의의 "IP 주소" 섹션을 다시 읽고, 네트워크 환경 정보를 완전히 수집하세요.
- traceroute를 실행하지 않았습니다. 다음 명령어로 실습하세요:
  ```bash
  tracert google.com  # Windows
  traceroute google.com  # Linux/Mac
  ```
- OSI 7계층 설명이 각 계층의 이름만 나열했습니다. 웹 페이지 접속 시 각 계층에서 구체적으로 무슨 일이 일어나는지 설명하세요.
  - 예: "계층 4에서 TCP 3-way 핸드셰이크로 연결을 설정한다."
- 네트워크 보안 계획에서 방화벽 규칙이 "필요한 포트만 연다"라고만 표현되어 불명확합니다. 구체적으로 어떤 포트(80, 443, 22 등)를 허용할지 명시하세요.

**도움이 될 자료**:
- Week 16 강의 자료: OSI 7계층, 네트워크 보안
- Cisco Networking Academy: https://www.netacad.com/
- 네트워크 진단 도구 사용법: 구글에서 "ping 사용법", "traceroute 사용법" 검색

### 부족한 답변 예시 (0-59점)
**주요 문제점**:
- 네트워크 진단이 미완료되었거나 결과가 부정확합니다.
- OSI 7계층에 대한 이해가 부족합니다.
- 네트워크 보안 계획이 매우 간략하거나 누락되었습니다.

**즉시 해야 할 일**:
1. Week 16 강의 자료를 처음부터 끝까지 정독하세요.
2. OSI 7계층을 암기하지 말고 이해하세요. 각 계층의 역할과 프로토콜을 정리하세요:
   | 계층 | 이름 | 역할 | 프로토콜 |
   |------|------|------|----------|
   | 7 | 응용 | 사용자 서비스 | HTTP, FTP, SMTP |
   | 4 | 전송 | 종단 간 통신 | TCP, UDP |
   | 3 | 네트워크 | 패킷 라우팅 | IP, ICMP |
   | 2 | 데이터 링크 | 프레임 전송 | Ethernet |
   | 1 | 물리 | 비트 전송 | 케이블 |
3. 네트워크 진단 실습을 다시 수행하세요:
   - Windows: `ipconfig /all`, `ping google.com`, `tracert google.com`
   - Mac/Linux: `ifconfig`, `ping google.com`, `traceroute google.com`
4. 네트워크 보안 기본 개념을 학습하세요: 방화벽, VPN, 암호화
5. 강사와 1:1 미팅을 예약하여 네트워크 기초 개념을 재학습하세요.

**재제출 권장**:
현재 수준으로는 네트워크 인프라 프로젝트 관리에 어려움이 있을 수 있습니다. 위 학습을 완료한 후 과제를 재제출하시기 바랍니다.
